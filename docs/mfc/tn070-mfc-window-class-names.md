# <a name="tn070-mfc-window-class-names"></a>TN070. Имена классов окна MFC

> [!NOTE]
> Следующее техническое примечание не было обновлено, поскольку сначала оно было включено в электронную документацию. В результате некоторые процедуры и разделы могут быть устаревшими или неверными. Для получения последних сведений рекомендуется выполнить поиск интересующей темы в алфавитном указателе документации в Интернете.

Окна MFC используют динамически созданное название класса, отражающее особенности окна. MFC генерирует имена классов динамически для окон кадра, представлений и всплывающих окон, производимых приложением. Коробки dialog и элементы управления, производимые приложением MFC, имеют предоставленное Windows название класса оконного проема.

Вы можете заменить динамически предоставленное название класса, зарегистрировав свой собственный класс окон и используя его в переопределение [PreCreateWindow.](../mfc/reference/cwnd-class.md#precreatewindow) Их MFC-поставленные имена типа приспосабливают одну из 2 следующих форм:

```
Afx:%x:%x
Afx:%x:%x:%x:%x:%x
```

Цифры шестиугольники, `%x` заменяющие символы, заполняются на данных из структуры [WNDCLASS.](/windows/win32/api/winuser/ns-winuser-wndclassw) MFC использует этот метод, так что несколько классов СЗ, требующих идентичных структур **WNDCLASS,** могут совместно классифицировать сяочку. В отличие от большинства простых приложений Win32, приложения MFC имеют только один **WNDPROC,** так что вы можете легко обмениваться **структурами WNDCLASS,** чтобы сэкономить время и память. Заменяемые значения `%x` для персонажей, показанных выше, являются следующими:

- **WNDCLASS.hInstance**

- **WNDCLASS.style**

- **WNDCLASS.hCursor**

- **WNDCLASS.hbrBackground**

- **WNDCLASS.hIcon**

Первая форма`Afx:%x:%x`() используется, когда **hCursor,** **hbrBackground**, и **hIcon** все **NULL**.
