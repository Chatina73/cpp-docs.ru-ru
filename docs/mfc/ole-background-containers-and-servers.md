# <a name="ole-background-containers-and-servers"></a>Поддержка OLE. Контейнеры и серверы

Приложения-контейнера — это приложение, которое может включать внедренные и связанные элементы в свои документы. Документы, управляемые приложения-контейнера должен иметь возможность хранения и отображения компонентов документа OLE, а также данные, созданные самим приложением. Приложения-контейнера необходимо также разрешить пользователям вставлять новые элементы или изменять существующие элементы, активировав серверных приложений, при необходимости. В этой статье перечислены требования пользовательского интерфейса для приложения контейнера [контейнеры: проблемы пользовательского интерфейса](../mfc/containers-user-interface-issues.md).

Серверное приложение или компонент приложение — это приложение, можно создавать компоненты документа OLE для использования приложением-контейнером. Серверные приложения обычно поддерживает перетаскивание или копировании свои данные в буфер обмена, чтобы приложения-контейнера можно вставить данные в виде внедренного или связанного элемента. Приложение может быть и контейнером, так и сервера.

Большинство серверов — автономного приложения или полных серверов; они могут выполняться либо как самостоятельные приложения или могут запускать приложения-контейнера. Miniserver — это специальный тип серверного приложения, которые могут запускаться только в контейнере. Он не может выполняться как отдельное приложение. Примерами miniservers являются серверы Microsoft Draw и Microsoft Graph.

Контейнеры и серверы не взаимодействуют напрямую. Вместо этого они обмениваются данными через OLE системных библиотек динамической компоновки (DLL). Эти библиотеки DLL обеспечивают функции, вызывающие контейнеры и серверы, а контейнеры и серверы также функции обратного вызова, которые вызывают библиотеки DLL.

С помощью этого средства коммуникации, контейнер не должны знать детали реализации серверного приложения. Она позволяет контейнер для принятия элементов, созданных любым сервером без необходимости определять типы серверов, с которыми можно работать. Таким образом пользователь приложения-контейнера можно воспользоваться преимуществами приложений будущего и форматов данных. Если они являются компонентами OLE, составной документ будет иметь возможность включать элементы, созданные при работе приложений.
