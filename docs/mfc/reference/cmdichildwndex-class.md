# <a name="cmdichildwndex-class"></a>Класс CMDIChildWndEx

`CMDIChildWndEx` Класс предоставляет функциональные возможности Windows дочернего окна многодокументного интерфейса (MDI). Он расширяет функциональные возможности [класс CMDIChildWnd](../../mfc/reference/cmdichildwnd-class.md). Платформа требует этот класс, если приложение MDI использует определенные классы MFC.

Для получения дополнительных сведений см. в разделе исходном коде, расположенном в **VC\\atlmfc\\src\\mfc** папке установки Visual Studio.

## <a name="syntax"></a>Синтаксис

```
class CMDIChildWndEx : public CMDIChildWnd
```

##  <a name="canshowonwindowslist"></a>  CMDIChildWndEx::CanShowOnWindowsList

Указывает, может ли отображаться имя дочернего окна интерфейса MDI в [класс CMFCWindowsManagerDialog](../../mfc/reference/cmfcwindowsmanagerdialog-class.md) диалоговое окно.

```
virtual BOOL CanShowOnWindowsList();
```

### <a name="return-value"></a>Возвращаемое значение

Значение TRUE, если окно может быть отображено в **Windows** диалоговое окно; в противном случае — значение FALSE.

### <a name="remarks"></a>Примечания

Переопределите этот метод в производном классе и возвращает значение FALSE, если окно не должен отображаться в **Windows** диалоговое окно. Эта функция вызывается из `CMFCWindowsManagerDialog`.

##  <a name="getdocumentname"></a>  CMDIChildWndEx::GetDocumentName

Возвращает имя документа, который отображается в дочернего окна интерфейса MDI.

```
virtual LPCTSTR GetDocumentName(CObject** pObj);
```

### <a name="return-value"></a>Возвращаемое значение

Указатель на строку, содержащую имя документа.

### <a name="remarks"></a>Примечания

Документ является отображает дочернего окна интерфейса MDI. Как правило в окне отображаются данные, загруженные из или в файл. Таким образом имя документа является имя файла. Реализация по умолчанию `GetDocumentName` возвращает string, полученный из `CDocument::GetPathName`.

Если в окне отображается документ, который не загружается из файла, переопределите этот метод в производном классе и возвращают уникальный идентификатор документа.

`GetDocumentName` вызывается платформой при сохранении состояния всех открытых документов. Возвращаемая строка записывается в реестр.

Когда платформа восстанавливает состояние позже, имя документа считывается из реестра и передается [CMDIFrameWndEx::CreateDocumentWindow](../../mfc/reference/cmdiframewndex-class.md#createdocumentwindow). Переопределите этот метод в [CMDIFrameWndEx](../../mfc/reference/cmdiframewndex-class.md)-производного класса и создания или открытия документа с таким именем и считать содержимое файла с таким именем. Если документ не основан на файл, создайте на основе идентификатора документа сам документ. Предыдущие действия следует делать только в том случае, если вы планируете сохранить и восстановить документы.

##  <a name="ongeticoniclivepreviewbitmap"></a>  CMDIChildWndEx::OnGetIconicLivePreviewBitmap

Вызвано структурой при необходимости растровое изображение для динамического просмотра элемента дочерней MDI-формы.

```
virtual HBITMAP OnGetIconicLivePreviewBitmap(
    BOOL bIsMDIChildActive,
    CPoint& ptLocation);
```

### <a name="parameters"></a>Параметры

*bIsMDIChildActive*<br/>
Этот параметр имеет значение TRUE, если точечный рисунок запрашивается для дочерней MDI-формы, который в данный момент активна и не сворачивается главного окна. В данном случае обработку по умолчанию создает моментальный снимок главного окна.

*ptLocation*<br/>
Указывает расположение растрового изображения в main (верхнего уровня) окна координатах клиентской области окна. Эта точка должны предоставляться вызываемым объектом.

### <a name="return-value"></a>Возвращаемое значение

При обработке возвращает дескриптор к растровому изображению, допустимым глубиной цвета 32 бита, в противном случае — значение NULL.

### <a name="remarks"></a>Примечания

Переопределите этот метод в производном классе и возвращать допустимый глубиной цвета 32 бита растрового изображения для динамического просмотра элемента дочерней MDI-формы. Этот метод вызывается только в том случае, если дочерней MDI-формы отображаются на вкладках панели задач Windows 7. Если возвращается значение NULL, MFC вызывает обработчики по умолчанию и получает точечные рисунки с помощью `PrintClient` или `PrintWindow`.
