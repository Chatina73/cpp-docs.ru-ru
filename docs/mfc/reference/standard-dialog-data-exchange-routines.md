## <a name="ddx_cbindex"></a><a name="ddx_cbindex"></a>DDX_CBIndex

Функция `DDX_CBIndex` управляет передачей данных **Int** между управлением комбо-коробкой в диалоговом поле, представлением формы или **объектом** представления управления и членом инди-данных в диалоговом поле, представлением формы или объектом управления.

```cpp
void AFXAPI DDX_CBIndex(
    CDataExchange* pDX,
    int nIDC,
    int& index);
```

### <a name="parameters"></a>Параметры

*Pdx*<br/>
Указатель на объект `CDataExchange`. Структура предоставляет этот объект для формирования контекста обмена данными, включая его направление.

*nIDC*<br/>
Идентификатор ресурса управления комбо-коробкой, связанный с свойством управления.

*Индекс*<br/>
Ссылка на переменную члена диалогового окна, представления формы или объекта управления, с которым обмениваются данными.

### <a name="remarks"></a>Remarks

При `DDX_CBIndex` вызове *индекс* устанавливается к индексу текущего выбора комбо-бокса. Если элемент не выбран, *индекс* устанавливается до 0.

Дополнительные сведения об DDX см [обмен данными окон и проверка](../../mfc/dialog-data-exchange-and-validation.md).

### <a name="requirements"></a>Требования

  **Заголовок** afxdd_.h

## <a name="ddx_datetimectrl"></a><a name="ddx_datetimectrl"></a>DDX_DateTimeCtrl

Функция `DDX_DateTimeCtrl` управляет передачей данных о дате и/или времени между управлением сборщиком даты и времени [(CDateTimeCtrl](../../mfc/reference/cdatetimectrl-class.md)) в диалоговом поле или объекте представления формы и либо [CTime,](../../atl-mfc-shared/reference/ctime-class.md) либо членом данных [COleDateTime](../../atl-mfc-shared/reference/coledatetime-class.md) в диалоговом поле или объекте представления формы.

```cpp
void AFXAPI DDX_DateTimeCtrl(
    CDataExchange* pDX,
    int nIDC,
    CTime& value);

void AFXAPI DDX_DateTimeCtrl(
    CDataExchange* pDX,
    int nIDC,
    COleDateTime& value);

void AFXAPI DDX_DateTimeCtrl(
    CDataExchange* pDX,
    int nIDC,
    CString& value);
```

### <a name="parameters"></a>Параметры

*Pdx*<br/>
Указатель на объект [CDataExchange.](../../mfc/reference/cdataexchange-class.md) Структура предоставляет этот объект для формирования контекста обмена данными, включая его направление. Вам не нужно удалять этот объект.

*nIDC*<br/>
Идентификатор ресурса управления сборщиком даты и времени, связанный с переменной элемента.

*value*<br/>
В первых двух версиях ссылка `COleDateTime` на переменную или переменную `CTime` члена, диалоговую коробку, представление формы или объект представления управления, с которым обмениваются данными. В третьей версии ссылка на объект представления представления элемента `CString` данных.

### <a name="remarks"></a>Remarks

Когда `DDX_DateTimeCtrl` вызывается, *значение* устанавливается в текущее состояние управления сборщика даты и времени, или элемент управления устанавливается для *значения,* в зависимости от направления обмена.

В третьей версии выше `DDX_DateTimeCtrl` управление `CString` передачей данных между контролем времени даты и членом данных [CString](../../atl-mfc-shared/reference/cstringt-class.md) объекта представления управления. Строка отформатирована с использованием правил текущего локализации для форматирования дат и времени.

Дополнительные сведения об DDX см [обмен данными окон и проверка](../../mfc/dialog-data-exchange-and-validation.md).

```cpp

void DDX_ManagedControl(
   CDataExchange* pDX,
   int nIDC,
   CWinFormsControl<T>& control );
```

*Pdx*<br/>
Указатель на объект [класса CDataExchange.](cdataexchange-class.md) Структура предоставляет этот объект для формирования контекста обмена данными, включая его направление.

*nIDC*<br/>
Идентификатор ресурса элемента управления, связанный с свойством управления.

*Управления*<br/>
Ссылка на объект [класса CWinFormsControl.](cwinformscontrol-class.md)

### <a name="remarks"></a>Remarks

`DDX_ManagedControl`вызывает [CWinFormsControl::CreateManagedControl](cwinformscontrol-class.md#createmanagedcontrol) для создания элемента управления, соответствующий идентификатору управления ресурсами. Используйте `DDX_ManagedControl` для создания элементов управления из идорождений ресурсов в [CDialog::OnInitDialog](cdialog-class.md#oninitdialog). Для обмена данными не требуется использовать функции DDX/DDV с элементами управления Windows Forms.

Для получения дополнительной [How to: Do DDX/DDV Data Binding with Windows Forms](../../dotnet/how-to-do-ddx-ddv-data-binding-with-windows-forms.md)информации см.

## <a name="ddx_ipaddress"></a><a name="ddx_ipaddress"></a>DDX_IPAddress

Функция `DDX_IPAddress` управляет передачей данных между контролем IP-адреса и членом данных объекта представления управления.

```cpp
void AFXAPI DDX_IPAddress(
    CDataExchange* pDX,
    int nIDC,
    DWORD& value);
```

### <a name="parameters"></a>Параметры

*Pdx*<br/>
Указатель на объект `CDataExchange`. Структура предоставляет этот объект для формирования контекста обмена данными, включая его направление.

*nIDC*<br/>
Идентификатор ресурса управления IP-адресом, связанный с свойством управления.

*value*<br/>
Ссылка на DWORD, содержащую четырехполевое значение управления IP-адресом. Поля заполняются или читаются следующим образом.

|Поле|Биты, содержащие значение поля|
|-----------|-------------------------------------|
|3|От 0 до 7|
|2|от 8 до 15|
|1|от 16 до 23|
|0|от 24 до 31|

Используйте [IPM_GETADDRESS](/windows/win32/Controls/ipm-getaddress) Win32 для чтения значения или [IPM_SETADDRESS](/windows/win32/Controls/ipm-setaddress) для заполнения значения. Эти сообщения описаны в Windows SDK.

### <a name="remarks"></a>Remarks

Когда `DDX_IPAddress` вызывается, *значение* либо считывается из управления IP-адресом, либо *значение* записывается в элемент управления, в зависимости от направления обмена.

Указатель на объект `CDataExchange`. Структура предоставляет этот объект для формирования контекста обмена данными, включая его направление.

*nIDC*<br/>
Идентификатор ресурса управления полем списка, связанный с свойством управления.

*value*<br/>
Ссылка на переменную члена диалогового окна, представления формы или объекта управления, с которым обмениваются данными.

### <a name="remarks"></a>Remarks

При `DDX_LBStringExact` вызове для передачи данных в элемент управления полем списка выбирается первый элемент элемента управления, который соответствует *значению.* (Чтобы соответствовать только приставке, а не всему элементу, используйте [DDX_LBString](#ddx_lbstring).) Если нет совпадений, не выбираются элементы. Соответствие является нечувствительным.

При `DDX_CBStringExact` вызове для передачи данных из управления полем списка *значение* устанавливается для текущего выбора поля списка. Если элемент не выбран, *значение* устанавливается строке нулевой длины.

> [!NOTE]
> Если поле списка представляет собой поле списка, обменное значение ограничено 255 символами.

## <a name="ddx_radio"></a><a name="ddx_radio"></a>DDX_Radio

Функция `DDX_Radio` управляет передачей данных **int** между группой управления радио в диалоговом поле, объектом представления формы или **элементом** управления и членом инди-данных в диалоговом поле, представлением формы или объектом управления. Значение участника **int** data определяется в соответствии с тем, какая кнопка радио в группе выбрана.

```cpp
void AFXAPI DDX_Radio(
    CDataExchange* pDX,
    int nIDC,
    int& value);
```

### <a name="parameters"></a>Параметры

*Pdx*<br/>
Указатель на объект `CDataExchange`. Структура предоставляет этот объект для формирования контекста обмена данными, включая его направление.

*nIDC*<br/>
Идентификатор ресурса первого радиоуправления в группе.

*value*<br/>
Ссылка на переменную члена диалогового окна, представления формы или объекта управления, с которым обмениваются данными.

### <a name="remarks"></a>Remarks

При `DDX_Radio` вызове *значение* устанавливается к текущему состоянию группы радиоконтроля. Значение устанавливается как 0-основанный индекс радиоконтроля, который в настоящее время проверяется, или -1, если радиоэлементы не проверены.

Например, в случае проверки первой радиокнопки в группе (кнопка с WS_GROUP стилем) значение участника **int** составляет 0 и так далее.
