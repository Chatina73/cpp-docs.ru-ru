# <a name="styles-used-by-mfc"></a>Стили, используемые MFC

Используйте следующие флаги стиля для указания внешнего вида окна или элемента управления и поведения при создании соответствующего объекта MFC. В большинстве случаев эти стили задаются в *dwStyle* параметр класса `Create` функции.

## <a name="button-styles"></a> Стили кнопок

Стили кнопок применяются к [класс CButton](../../mfc/reference/cbutton-class.md) проверьте объекты, такие как переключатели, поля и кнопок. Укажите сочетание стилей в *dwStyle* параметр [CButton::Create](../../mfc/reference/cbutton-class.md#create). Дополнительные сведения о стилях кнопки в Windows, см. в разделе [стили кнопок (Windows)](/windows/desktop/Controls/button-styles).

### <a name="button-types"></a>Типы кнопок

В следующей таблице перечислены типы кнопок. При необходимости можно выбрать одно из следующих. Если вы не укажете тип кнопки, по умолчанию используется BS_PUSHBUTTON.

|Тип|Описание:|
|----------|-----------------|
|BS_3STATE|Создает флажок с тремя состояниями: BST_CHECKED BST_INDETERMINATE и BST_UNCHECKED. При нажатии кнопки отправляет уведомление BN_CLICKED окну-владельцу, но не изменяет состояние кнопки. По умолчанию связанный текст отображается справа от флажка. Для отображения текста слева от флажка используйте стиль BS_LEFTTEXT или BS_RIGHTBUTTON.|
|BS_AUTO3STATE|Создает флажок с тремя состояниями: BST_CHECKED BST_INDETERMINATE и BST_UNCHECKED. При нажатии кнопки отправляет уведомление BN_CLICKED окну-владельцу и изменяет состояние кнопки. Кнопка состояний цикл порядке BST_CHECKED BST_INDETERMINATE и BST_UNCHECKED. По умолчанию связанный текст отображается справа от флажка. Для отображения текста слева от флажка используйте стиль BS_LEFTTEXT или BS_RIGHTBUTTON.|
|BS_AUTOCHECKBOX|Создает флажок с двумя состояниями: BST_CHECKED и BST_UNCHECKED. При нажатии кнопки отправляет уведомление BN_CLICKED окну-владельцу и изменяет состояние кнопки. По умолчанию связанный текст отображается справа от флажка. Для отображения текста слева от флажка используйте стиль BS_LEFTTEXT или BS_RIGHTBUTTON.|
|BS_AUTORADIOBUTTON|Создает переключатель с двумя состояниями: BST_CHECKED и BST_UNCHECKED. Переключатели обычно используются в группах, с каждой группой, более одного параметра проверенного одновременно. При нажатии кнопки отправляет окну-владельцу уведомление BN_CLICKED, устанавливает состояние переключателю BST_CHECKED и задает состояния всех переключателей в группе переключателей для BST_UNCHECKED. По умолчанию связанный текст отображается справа от переключателя. Для отображения текста слева от переключателя используйте стиль BS_LEFTTEXT или BS_RIGHTBUTTON.|
|BS_CHECKBOX|Создает флажок с двумя состояниями: BST_CHECKED и BST_UNCHECKED. При нажатии кнопки отправляет уведомление BN_CLICKED окну-владельцу, но не изменяет состояние кнопки. По умолчанию связанный текст отображается справа от флажка. Для отображения текста слева от флажка используйте стиль BS_LEFTTEXT или BS_RIGHTBUTTON.|
|BS_COMMANDLINK|Создает кнопку command link. Кнопка command link — это Командная кнопка, относящиеся к Windows Vista, которая отображает зеленую стрелку слева от основного текста и Примечание под основным текстом. Можно задать текст примечания с помощью [CButton::SetNote](../../mfc/reference/cbutton-class.md#setnote).|
|BS_DEFCOMMANDLINK|Создает кнопку command link. Кнопка command link — это Командная кнопка, относящиеся к Windows Vista, которая отображает зеленую стрелку слева от основного текста и Примечание под основным текстом. Можно задать текст примечания с помощью [CButton::SetNote](../../mfc/reference/cbutton-class.md#setnote). Если кнопка в диалоговом окне, нажатие клавиши ВВОД отправляет уведомление BN_CLICKED в диалоговое окно, даже в том случае, если кнопка не имеет фокус ввода.|
|BS_DEFPUSHBUTTON|Создает командную кнопку с толстой черной границей. Если кнопка в диалоговом окне, нажатие клавиши ВВОД отправляет уведомление BN_CLICKED в диалоговое окно, даже в том случае, если кнопка не имеет фокус ввода.|
|BS_DEFSPLITBUTTON|Создает разворачивающуюся кнопку. Разворачивающаяся кнопка — это Командная кнопка, относящиеся к Windows Vista, которая содержит кнопку рядом стрелку раскрывающегося списка. При нажатии кнопки выполняется команда по умолчанию. Если щелкнуть стрелку раскрывающегося списка, появится меню дополнительных команд. Если разворачивающуюся кнопку в диалоговом окне, нажатие клавиши ВВОД отправляет уведомление BN_CLICKED в диалоговое окно даже в том случае, если кнопка не имеет фокус ввода|
|BS_GROUPBOX|Создает прямоугольник, в котором можно группировать другие кнопки. Текст, связанный с этим стилем отображается в левом верхнем углу прямоугольника.|
|BS_OWNERDRAW|Создает рисуемую владельцем кнопку. Платформа вызывает `DrawItem` метод, когда визуальный аспект кнопки был изменен. Этот подход следует установить при использовании `CBitmapButton` класса.|
|BS_PUSHBUTTON|Создает командную кнопку, которая отправляет окну-владельцу уведомление BN_CLICKED, когда пользователь нажимает кнопку.|
|BS_RADIOBUTTON|Создает переключатель с двумя состояниями: BST_CHECKED и BST_UNCHECKED. Переключатели обычно используются в группах, с каждой группой, более одного параметра проверенного одновременно. При нажатии кнопки отправляет уведомление BN_CLICKED окну-владельцу, но не автоматически изменяет состояние переключателей в группе. По умолчанию связанный текст отображается справа от переключателя. Для отображения текста слева от переключателя используйте стиль BS_LEFTTEXT или BS_RIGHTBUTTON.|
|BS_SPLITBUTTON|Создает разворачивающуюся кнопку. Разворачивающаяся кнопка — это Командная кнопка, относящиеся к Windows Vista, которая содержит кнопку рядом стрелку раскрывающегося списка. При нажатии кнопки выполняется команда по умолчанию. Если щелкнуть стрелку раскрывающегося списка, появится меню дополнительных команд.|
|BS_USERBUTTON|Является устаревшим, но указывается для совместимости с 16-разрядных версиях Windows. Приложения Win32 должны использовать BS_OWNERDRAW.|

### <a name="radio-button-and-check-box-styles"></a>Стили переключателей и флажков

В следующей таблице перечислены примеры, относящиеся к переключатели и флажки. Эти стили игнорируются во всех других типах кнопки. При необходимости можно выбрать один или несколько из следующих.

|Стиль|Описание:|
|-----------|-----------------|
|BS_LEFTTEXT|В сочетании со стилем переключателя кнопку или флажок текст отображается в левой части переключателя или флажка.|
|BS_RIGHTBUTTON|В сочетании со стилем переключателя кнопку или флажок текст отображается в левой части переключателя или флажка. Этот стиль идентичен стилю BS_LEFTTEXT.|
|BS_PUSHLIKE|Делает "флажок" или "переключатель" так же, как кнопки. Кнопка выглядит нажатой, когда он находится в состоянии BST_CHECKED, нажата и затемняется, когда он находится в состоянии BST_INDETERMINATE снимаются, когда он находится в состоянии BST_UNCHECKED.|

### <a name="button-text-alignment-styles"></a>Стили выравнивания текста кнопки

В следующей таблице перечислены горизонтальные и вертикальные параметры выравнивания. При необходимости можно выбрать одно из следующих.

|Стиль|Описание:|
|-----------|-----------------|
|BS_LEFT|По левому краю в прямоугольнике кнопки. Тем не менее, если кнопка "флажок" или "переключатель", не поддерживает стиль BS_RIGHTBUTTON, текст остается выравниваются по правой части флажка или переключателя.|
|BS_RIGHT|По правому текста в прямоугольнике кнопки. Тем не менее если кнопка "флажок" или "переключатель", не поддерживает стиль BS_RIGHTBUTTON, текст выравнивается по правому краю в правой части флажка или переключателя.|
|BS_CENTER|Центрирует текст по горизонтали в прямоугольнике кнопки.|
|BS_TOP|Размещает текст в верхней части прямоугольника кнопки.|
|BS_BOTTOM|Размещает текст в нижней части прямоугольника кнопки.|
|BS_VCENTER|Центрирует текст по вертикали в прямоугольнике кнопки.|

### <a name="button-content-options"></a>Параметры содержимого кнопок

Ниже перечислены параметры, которые указывают, что отображается на кнопке. Эти стили игнорируют типы кнопок, которые только отображают текст. При необходимости можно выбрать одно из следующих.

|Стиль|Описание:|
|-----------|-----------------|
|BS_BITMAP|Указывает, что кнопка отображает точечный рисунок.|
|BS_ICON|Указывает, что кнопка отображает значок.|
|BS_TEXT|Указывает, что кнопка отображает текст.|

### <a name="other-button-options"></a>Другие параметры кнопки

В следующей таблице перечислены дополнительные параметры, которые можно использовать с любым типом кнопок. При необходимости можно выбрать один или несколько из следующих.

|Стиль|Описание:|
|-----------|-----------------|
|BS_FLAT|Указывает, что кнопка двухмерна и не рисуется с затенением по умолчанию для создания трехмерный изображение.|
|BS_MULTILINE|Переносит текст кнопки на несколько строк, если текстовая строка умещается на одной строке в прямоугольнике кнопки.|
|BS_NOTIFY|Позволяет кнопке отправлять BN_DBLCLK BN_KILLFOCUS и BN_SETFOCUS сообщения уведомления своему родительскому окну. Обратите внимание на то, что кнопки отправки уведомления BN_CLICKED независимо от того, указан ли данный стиль.|

## <a name="combo-box-styles"></a>  Поле со списком стилей

В MFC доступны перечисленные далее стили полей со списками. Дополнительные сведения о стилях поле со списком в Windows см. в разделе [стили поля со списком (Windows)](/windows/desktop/Controls/combo-box-styles).

|Стиль|Описание:|
|-----------|-----------------|
|CBS_AUTOHSCROLL|Автоматическая прокрутка текста в элементе управления вправо при вводе символа в конце строки. Если этот стиль не задан, допускается только текст, который умещается в прямоугольной области.|
|CBS_DISABLENOSCROLL|Отображается отключенная вертикальная полоса прокрутки при списке содержится недостаточно элементов для прокрутки. Если этот стиль не задан, полоса прокрутки будет скрыта при недостаточном количестве элементов в списке.|
|CBS_DROPDOWN|Аналогично CBS_SIMPLE, за исключением того, что поле со списком отображается только в том случае, если пользователь выбирает значок рядом с элемента управления редактирования.|
|CBS_DROPDOWNLIST|Аналогично CBS_DROPDOWN, за исключением того, что поле ввода заменяется элементом статического текста, отображающий текущее выделение в поле со списком.|
|CBS_HASSTRINGS|Рисуемый владельцем поле со списком содержит элементы, состоящий из строк. Поле со списком поддерживает память и указатели для строк, поэтому приложение может использовать функцию-член `GetText` для получения текста для определенного элемента.|
|CBS_LOWERCASE|Преобразует в нижний регистр весь текст в поле выбора и в списке.|
|CBS_NOINTEGRALHEIGHT|Указывает, что размер поля со списком точно размер, заданному приложением при создании поля со списком. Как правило, в Windows размер поля со списком определяется так, чтобы в избежать частичного отображения элементов.|
|CBS_OEMCONVERT|Текст, введенный в поле со списком управления преобразуется из набора символов ANSI в набор символов OEM и затем обратно в ANSI. |SS_BLACKRECT|Указывает прямоугольник, закрашенный цветом, используемым для рисования рамки окон. Значением по умолчанию является Black.|
|SS_CENTER|Обозначает простой прямоугольник и отображает заданный текст по центру прямоугольника. Текст форматируется перед его отображением. Слова, которые бы расширить после конца строки, автоматически переносятся в начало следующей строки по центру.|
|SS_CENTERIMAGE|Указывает, что, в случае, если точечный рисунок или значок меньше клиентской области статического элемента управления, остальная часть клиентской области заполняется цвет пикселя в левом верхнем углу точечный рисунок или значок. Если статический элемент управления содержит одну строку текста, текст выравнивается по центру по вертикали в клиентской области элемента управления.|
|SS_ENDELLIPSIS|SS_PATHELLIPSIS заменяет часть или заданную строку с многоточием, при необходимости, таким образом, чтобы результат помещается в указанный прямоугольник. <br /><br />Можно указать SS_END_ELLIPSIS для замены символов в конце строки или SS_PATHELLIPSIS для замены символов в середине строки. Если строка содержит обратную косую черту (\\) символов, SS_PATHELLIPSIS сохраняет как можно большую часть текста после последней обратной косой черты можно.|
|SS_ENHMETAFILE|Указывает, что расширенный метафайл для отображения в статического элемента управления. Данный текст является имя метафайла. Расширенный метафайл статический элемент управления имеет фиксированный размер; метафайл масштабируется по размерам клиентской области статического элемента управления.|
|SS_ETCHEDFRAME|Рисует кадр статического элемента управления с использованием стиля EDGE_ETCHED edge.|
|SS_ETCHEDHORZ|Рисует верхней и нижней границ статического элемента управления с использованием стиля EDGE_ETCHED edge.|
|SS_ETCHEDVERT|Рисует левого и правого краев статического элемента управления с использованием стиля EDGE_ETCHED edge.|
|SS_GRAYFRAME|Указывает окно с кадром, нарисованных при помощи цвет фона экрана (desktop). По умолчанию задано значение Gray.|
|SS_GRAYRECT|Указывает прямоугольник, закрашенный цветом, используемым для заливки фона экрана. По умолчанию задано значение Gray.|
|SS_ICON|Задает значок, отображаемый в диалоговом окне. Данный текст является имя значка (не имя файла), определенный в другом месте в файле ресурсов. *NWidth* и *nHeight* игнорируются; значок автоматически изменяет свой размер.|
|SS_LEFT|Обозначает простой прямоугольник и отображает заданный текст flush в левого угла прямоугольника. Текст форматируется перед его отображением. Слова, которые бы расширить после конца строки, автоматически переносятся в начало следующей строки левого flush.|
|SS_LEFTNOWORDWRAP|Обозначает простой прямоугольник и отображает заданный текст flush в левого угла прямоугольника. Вкладки будут развернуты, но слова не заключены в оболочку. Обрезается текст, который выходит за пределы строки.|
|SS_NOPREFIX|Если этот стиль не задан, Windows будет интерпретировать любой амперсанд (&) символов в тексте элемента управления быть символы префикса сочетаний клавиш. В этом случае амперсанд удаляется и следующий символ в строке подчеркивается. Если статический элемент управления должен содержать текст где эта функция нежелательна, SS_NOPREFIX могут быть добавлены. Этот стиль элемента управления статическим могут быть включены в любой из определенных статических элементов управления. SS_NOPREFIX можно объединить с другими стилями, с помощью битового оператора или. Чаще всего это используется, когда имена файлов или другие строки, которые могут содержать знак должны отображаться в статический элемент управления в диалоговом окне.|
|SS_NOTIFY|Отправляет родительского окна STN_CLICKED, STN_DBLCLK, STN_DISABLE и STN_ENABLE сообщения уведомления, когда пользователь нажимает кнопку или дважды щелкает элемент управления.|
|SS_OWNERDRAW|Указывает, что владельцем статический элемент управления отвечает за отрисовку элемента управления. Окно-владелец получает сообщение WM_DRAWITEM всякий раз, когда элемент управления должен быть нарисован.|
|SS_REALSIZEIMAGE|Предотвращает изменение размера при загрузке или отображается статическим управления значка или точечного рисунка (то есть статические элементы управления со стилем SS_ICON или SS_BITMAP). Если значок или растровое изображение больше, чем область назначения, рисунок обрезается.|
|SS_RIGHT|Обозначает простой прямоугольник и отображает заданный текст правом flush в прямоугольнике. Текст форматируется перед его отображением. Слова, которые бы расширить после конца строки, автоматически переносятся в начало следующей строки flush справа.|
|SS_RIGHTJUST|Указывает, что правый нижний угол статический элемент управления со стилем SS_BITMAP или SS_ICON оставаться устранена при изменении размера элемента управления. Только верхняя и левая стороны корректируются в соответствии с новой точечный рисунок или значок.|
|SS_SIMPLE|Обозначает простой прямоугольник и отображается одна строка текста flush в левого угла прямоугольника. Строка текста не может укорачивать или изменен каким-либо образом. (Окно или диалоговое окно родительского элемента управления не должен обработать wm_ctlcolor-сообщение.)|
|SS_SUNKEN|Рисует утопленная граница вокруг статический элемент управления.|
|SS_USERITEM|Задает определяемые пользователем элементы.|
|SS_WHITEFRAME|Указывает окно с кадром, нарисованных при помощи цвет фона окна. По умолчанию цвет белый.|
|SS_WHITERECT|Указывает прямоугольник, закрашенный цветом, используемым для заполнения фона окна. По умолчанию цвет белый.|
|SS_WORDELLIPSIS|Усекает текст, который не умещается и добавляет кнопку с многоточием.|

## <a name="window-styles"></a> Стили окна

Стили окна применяются к [класс CWnd](../../mfc/reference/cwnd-class.md) объектов. Укажите сочетание стилей в *dwStyle* параметр [CWnd::Create](../../mfc/reference/cwnd-class.md#create) или [CWnd::CreateEx](../../mfc/reference/cwnd-class.md#createex). Дополнительные сведения о стили окна в Windows, см. в разделе [стили окна (Windows)](/windows/desktop/winmsg/window-styles).

|Стиль|Описание:|
|-----------|-----------------|
|WS_BORDER|Создает окно с границей вокруг него.|
|WS_CAPTION|Создает окно, которое имеет заголовок окна (подразумевает стиль WS_BORDER). Не может использоваться со стилем WS_DLGFRAME.|
|WS_CHILD|Создает дочернее окно. Не может использоваться со стилем WS_POPUP.|
|WS_CHILDWINDOW|То же, как стиль WS_CHILD.|
|WS_CLIPCHILDREN|Исключает область занятую дочерними окнами при рисовании внутри родительского окна. Используется при создании родительского окна.|
|WS_CLIPSIBLINGS|Относительно друг друга; то есть когда конкретного дочернего окна получает сообщения рисования, стиль WS_CLIPSIBLINGS отсекает все остальные перекрывающиеся дочерние окна вне области дочернего окна, чтобы обновить. (Если не имеет WS_CLIPSIBLINGS и дочерние окна перекрываются, при рисовании в клиентской области дочернего окна, это можно рисовать в клиентской области соседнего дочернего окна.) Для использования только в стиле WS_CHILD.|
|WS_DISABLED|Создает окно, которое изначально было отключено.|
|WS_DLGFRAME|Создает окно с двойной границей, но без заголовка.|
|WS_GROUP|Указывает первый элемент управления из группы элементов управления, в которых пользователь может перемещаться от одного элемента управления к другому с помощью клавиш со стрелками. Все элементы управления, определенные с помощью стиля WS_GROUP FALSE после первого элемента управления принадлежать к той же группе. Следующий элемент управления с помощью стиля WS_GROUP запускается следующая команда (то есть одна команда заканчивается где начинается следующее).|
|WS_HSCROLL|Создает окно с горизонтальной полосы прокрутки.|
|WS_ICONIC|Создает окно, которое изначально было свернуто. Совпадает со значением WS_MINIMIZE стиль.|
|WS_MAXIMIZE|Создает окно максимального размера.|
|WS_MAXIMIZEBOX|Создает окно, которое имеет кнопку развертывания.|
|WS_MINIMIZE|Создает окно, которое изначально было свернуто. Для использования только в стиле WS_OVERLAPPED.|
|WS_MINIMIZEBOX|Создает окно, которое имеет кнопку свертывания.|
|WS_OVERLAPPED|Создает перекрывающееся окно. Перекрытое окно обычно имеет заголовок и границу.|
|WS_OVERLAPPEDWINDOW|Создает перекрывающиеся окно со стилями WS_OVERLAPPED, WS_CAPTION, WS_SYSMENU, WS_THICKFRAME, WS_MINIMIZEBOX и WS_MAXIMIZEBOX.|
|WS_POPUP|Создает всплывающее окно. Не может использоваться со стилем WS_CHILD.|
|WS_POPUPWINDOW|Создает всплывающее окно со стилями WS_BORDER WS_POPUP и WS_SYSMENU. Стиль WS_CAPTION должны объединяться со стилем WS_POPUPWINDOW, чтобы отобразить меню элемента управления.|
|WS_SIZEBOX|Создает окно с рамкой для изменения размера. Совпадает со значением WS_THICKFRAME стиль.|
|WS_SYSMENU|Создает окно, которое содержит поле меню для элемента управления в заголовке окна. Используется только для окон с заголовками окна.|
|WS_TABSTOP|Задает один из произвольного числа элементов управления, которые пользователь может перемещать с помощью клавиши TAB. Клавиша TAB перемещает пользователя к следующему элементу управления, указанного стилем WS_TABSTOP.|
|WS_THICKFRAME|Создает окно с утолщенной, который может использоваться для изменения размера окна.|
|WS_TILED|Создает перекрывающееся окно. Перекрытое окно имеет заголовок и границу. Совпадает со значением WS_OVERLAPPED стиль.|
|WS_TILEDWINDOW|Создает перекрывающиеся окно со стилями WS_OVERLAPPED, WS_CAPTION, WS_SYSMENU, WS_THICKFRAME, WS_MINIMIZEBOX и WS_MAXIMIZEBOX. Совпадает со значением WS_OVERLAPPEDWINDOW стиль.|
|WS_VISIBLE|Создает окно, которое изначально является видимым.|
|WS_VSCROLL|Создает окно, которое имеет вертикальную полосу прокрутки.|

## <a name="extended-window-styles"></a> Расширенные стили окна

Расширенные стили окна применяются к [класс CWnd](../../mfc/reference/cwnd-class.md) объектов. Укажите сочетание стилей в *dwExStyle* параметр [CWnd::CreateEx](../../mfc/reference/cwnd-class.md#createex). Дополнительные сведения о расширенные стили окна в Windows, см. в разделе [расширенные стили окна (Windows)](/windows/desktop/winmsg/extended-window-styles).

|Стиль|Описание:|
|-----------|-----------------|
|WS_EX_ACCEPTFILES|Указывает, что с окном, созданным при использовании этого стиля принимает и перетаскивать файлы.|
|WS_EX_APPWINDOW|Заставляет окна верхнего уровня на панели задач, если окно является видимым.|
|WS_EX_CLIENTEDGE|Указывает, что окно имеет трехмерных — то есть граница имеет утопленную границу.|
|WS_EX_CONTEXTHELP|Включает в себя вопросительного знака в строке заголовка окна. Когда пользователь щелкает вопросительный знак, курсор изменяется с вопросительным знаком с указателем. Если пользователь щелкает дочернее окно, это окно получает сообщение WM_HELP.|
|WS_EX_CONTROLPARENT|Позволяет пользователю переходить от одного дочерних окон окна с помощью клавиши TAB.|
|WS_EX_DLGMODALFRAME|Назначает окно с двойной границей, (при необходимости) могут быть созданы с помощью заголовка окна при указании флага стиль WS_CAPTION в *dwStyle* параметра.|
|WS_EX_LAYERED|Окно [многоуровневых окон](/windows/desktop/winmsg/window-features). Этот стиль не может использоваться, если окно имеет [стиль класса](/windows/desktop/winmsg/about-window-classes) CS_OWNDC или CS_CLASSDC. Тем не менее Microsoft Windows 8 поддерживает WS_EX_LAYERED стиль для дочерних окон, где предыдущие версии Windows поддерживают его только для окон верхнего уровня.|
|WS_EX_LEFT|Предоставляет свойства универсального по левому краю окна. Это значение по умолчанию.|
|WS_EX_LEFTSCROLLBAR|Помещает вертикальная полоса прокрутки слева от клиентской области.|
|WS_EX_LTRREADING|Отображает окно текст, использующий слева направо считывание свойств заказа. Это значение по умолчанию.|
|WS_EX_MDICHILD|Создает дочерние окна интерфейса MDI.|
|WS_EX_NOPARENTNOTIFY|Указывает, что дочернего окна, созданные с помощью этот стиль не будет отправлять сообщения WM_PARENTNOTIFY своему родительскому окну при создании или уничтожении дочернего окна.|
|WS_EX_OVERLAPPEDWINDOW|Сочетание стилей WS_EX_CLIENTEDGE и WS_EX_WINDOWEDGE|
|WS_EX_PALETTEWINDOW|Сочетание стилей WS_EX_WINDOWEDGE и WS_EX_TOPMOST.|
|WS_EX_RIGHT|Предоставляет окно универсальные свойства, по правому краю. Это зависит от класса окна.|
|WS_EX_RIGHTSCROLLBAR|Помещает вертикальной полосы прокрутки (при его наличии) справа от клиентской области. Это значение по умолчанию.|
|WS_EX_RTLREADING|Отображает текст окна, с помощью чтения справа налево упорядочить свойства.|
|WS_EX_STATICEDGE|Создает окно со стилем трехмерная граница, предназначен для использования для элементов, которые не принимают пользовательский ввод.|
|WS_EX_TOOLWINDOW|Создает окно инструментов, который представляет собой окно, предназначен для использования плавающую панель инструментов. Окно инструментов имеет заголовок, короче обычного заголовка и выводится более мелким шрифтом. Окно инструментов не отображается на панели задач или в окне, появляющемся при нажатии клавиши ALT + TAB.|
|WS_EX_TOPMOST|Указывает, что с окном, созданным с помощью этот стиль следует поместить над всеми nontopmost windows и оставаться более высокого уровня, даже в том случае, когда окно деактивировано. Приложение может использовать `SetWindowPos` функция-член для добавления или удаления этого атрибута.|
|WS_EX_TRANSPARENT|Указывает, что с окном, созданным при использовании этого стиля прозрачным. То есть всех окон, которые находятся ниже окна не занятую окна. С окном, созданным при использовании этого стиля получает сообщения WM_PAINT, только в том случае, после обновления все окна того же уровня под ним.|
|WS_EX_WINDOWEDGE|Указывает, что окно имеет приподнятую границу.|
