# <a name="styles-used-by-mfc"></a>Стили, используемые MFC

Используйте следующие флаги стиля, чтобы задать внешний вид окна или элемента управления и поведение при создании соответствующего объекта MFC. В большинстве случаев эти стили задаются в параметре *двстиле* функции класса `Create` .

## <a name="button-styles"></a>Стили кнопок

Стили кнопок применяются к объектам [класса кбуттон](../../mfc/reference/cbutton-class.md) , таким как переключатели, флажки и кнопки. Укажите сочетание стилей в параметре *Двстиле* [Кбуттон:: Create](../../mfc/reference/cbutton-class.md#create). Дополнительные сведения о стилях кнопок в Windows см. в разделе [стили кнопок (Windows)](/windows/win32/Controls/button-styles).

### <a name="button-types"></a>Типы кнопок

В следующей таблице перечислены типы кнопок. При необходимости можно выбрать один из следующих вариантов. Если не указать тип кнопки, по умолчанию используется значение BS_PUSHBUTTON.

|Тип|Описание|
|----------|-----------------|
|BS_3STATE|Создает кнопку флажка с тремя состояниями: BST_CHECKED, BST_INDETERMINATE и BST_UNCHECKED. Нажатие кнопки отправляет уведомление BN_CLICKED в окно владельца, но не изменяет состояние кнопки. По умолчанию соответствующий текст отображается справа от флажка. Чтобы отобразить текст слева от флажка, используйте стиль BS_LEFTTEXT или BS_RIGHTBUTTON.|
|BS_AUTO3STATE|Создает кнопку флажка с тремя состояниями: BST_CHECKED, BST_INDETERMINATE и BST_UNCHECKED. Нажатие кнопки отправляет уведомление BN_CLICKED в окно владельца и изменяет состояние кнопки. Состояние кнопки цикла находится в порядке BST_CHECKED, BST_INDETERMINATE и BST_UNCHECKED. По умолчанию соответствующий текст отображается справа от флажка. Чтобы отобразить текст слева от флажка, используйте стиль BS_LEFTTEXT или BS_RIGHTBUTTON.|
|BS_AUTOCHECKBOX|Создает кнопку флажка с двумя состояниями: BST_CHECKED и BST_UNCHECKED. Нажатие кнопки отправляет уведомление BN_CLICKED в окно владельца и изменяет состояние кнопки. По умолчанию соответствующий текст отображается справа от флажка. Чтобы отобразить текст слева от флажка, используйте стиль BS_LEFTTEXT или BS_RIGHTBUTTON.|
|BS_AUTORADIOBUTTON|Создает переключатель с двумя состояниями: BST_CHECKED и BST_UNCHECKED. Переключатели обычно используются в группах, при этом каждая группа имеет максимум один параметр проверки за раз. Нажимая кнопку, вы отправляете уведомление BN_CLICKED в окно владельца, устанавливаете состояние переключателя в положение BST_CHECKED и устанавливает состояния всех остальных переключателей в группе кнопок в значение BST_UNCHECKED. По умолчанию соответствующий текст отображается справа от переключателя. Чтобы отобразить текст слева от переключателя, используйте стиль BS_LEFTTEXT или BS_RIGHTBUTTON.|
|BS_CHECKBOX|Создает кнопку флажка с двумя состояниями: BST_CHECKED и BST_UNCHECKED. Нажатие кнопки отправляет уведомление BN_CLICKED в окно владельца, но не изменяет состояние кнопки. По умолчанию соответствующий текст отображается справа от флажка. Чтобы отобразить текст слева от флажка, используйте стиль BS_LEFTTEXT или BS_RIGHTBUTTON.|
|BS_COMMANDLINK|Создает кнопку ссылки на команду. Кнопка «ссылка на команду» — это кнопка, относящаяся к Windows Vista, которая отображает зеленую стрелку слева от основного текста и Примечание под основным текстом. Текст заметки можно задать с помощью [кбуттон:: сетноте](../../mfc/reference/cbutton-class.md#setnote).|
|BS_DEFCOMMANDLINK|Создает кнопку ссылки на команду. Кнопка «ссылка на команду» — это кнопка, относящаяся к Windows Vista, которая отображает зеленую стрелку слева от основного текста и Примечание под основным текстом. Текст заметки можно задать с помощью [кбуттон:: сетноте](../../mfc/reference/cbutton-class.md#setnote). Если кнопка находится в диалоговом окне, нажатие клавиши ВВОД отправляет BN_CLICKED уведомление в диалоговое окно, даже если кнопка не имеет фокуса ввода.|
|BS_DEFPUSHBUTTON|Создает кнопку команды с толстой черной границей. Если кнопка находится в диалоговом окне, нажатие клавиши ВВОД отправляет BN_CLICKED уведомление в диалоговое окно, даже если кнопка не имеет фокуса ввода.|
|BS_DEFSPLITBUTTON|Создает разворачивающуюся кнопку. Разворачивающаяся кнопка — это кнопка, относящаяся к Windows Vista, которая содержит кнопку, смежную со стрелкой раскрывающегося списка. При нажатии кнопки выполняется команда по умолчанию. Если щелкнуть стрелку раскрывающегося списка, появится меню дополнительных команд. Если кнопка разделить находится в диалоговом окне, нажатие клавиши ВВОД отправляет BN_CLICKED уведомление в диалоговое окно, даже если кнопка не имеет фокуса ввода.|
|BS_GROUPBOX|Создает прямоугольник, в котором можно группировать другие кнопки. Текст, связанный с этим стилем, отображается в левом верхнем углу прямоугольника.|
|BS_OWNERDRAW|Создает кнопку, рисуемую владельцем. Платформа вызывает `DrawItem` метод при изменении визуального аспекта кнопки. Этот стиль должен быть задан при использовании `CBitmapButton` класса.|
|BS_PUSHBUTTON|Создает кнопку команды, которая отправляет уведомление BN_CLICKED в окно владельца, когда пользователь нажимает кнопку.|
|BS_RADIOBUTTON|Создает переключатель с двумя состояниями: BST_CHECKED и BST_UNCHECKED. Переключатели обычно используются в группах, при этом каждая группа имеет максимум один параметр проверки за раз. Нажатие кнопки отправляет уведомление BN_CLICKED в окно владельца, но не изменяет состояние каких-либо кнопок в группе автоматически. По умолчанию соответствующий текст отображается справа от переключателя. Чтобы отобразить текст слева от переключателя, используйте стиль BS_LEFTTEXT или BS_RIGHTBUTTON.|
|BS_SPLITBUTTON|Создает разворачивающуюся кнопку. Разворачивающаяся кнопка — это кнопка, относящаяся к Windows Vista, которая содержит кнопку, смежную со стрелкой раскрывающегося списка. При нажатии кнопки выполняется команда по умолчанию. Если щелкнуть стрелку раскрывающегося списка, появится меню дополнительных команд.|
|BS_USERBUTTON|Устаревший, но они предоставляются для обеспечения совместимости с 16-разрядными версиями Windows. В приложениях на основе Win32 вместо этого следует использовать BS_OWNERDRAW.|

### <a name="radio-button-and-check-box-styles"></a>Стили переключателей и флажков

В следующей таблице перечислены стили, характерные для переключателей и флажков. Эти стили игнорируются во всех остальных типах кнопок. При необходимости можно выбрать один или несколько из следующих вариантов.

|Стиль|Описание|
|-----------|-----------------|
|BS_LEFTTEXT|При объединении с переключателем или стилем флажка текст отображается в левой части переключателя или флажка.|
|BS_RIGHTBUTTON|При объединении с переключателем или стилем флажка текст отображается в левой части переключателя или флажка. Этот стиль идентичен стилю BS_LEFTTEXT.|
|BS_PUSHLIKE|Устанавливает флажок или вид переключателя и ведут себя как кнопка команды. Кнопка отображается нажатой, когда ее состояние — BST_CHECKED, нажата и затемнена, когда состояние BST_INDETERMINATE, и освобождается, когда его состояние — BST_UNCHECKED.|

### <a name="button-text-alignment-styles"></a>Стили выравнивания текста кнопки

В следующей таблице перечислены горизонтальные и вертикальные параметры выравнивания текста. При необходимости можно выбрать один из следующих вариантов.

|Стиль|Описание|
|-----------|-----------------|
|BS_LEFT|По левому краю текст в прямоугольнике кнопки. Однако, если кнопка является флажком или переключателем, у которых нет стиля BS_RIGHTBUTTON, то текст слева от флажка или переключателя выровняйтеся по левому краю.|
|BS_RIGHT|По правому краю текст в прямоугольнике кнопки. Однако если кнопка является флажком или переключателем, у которых нет стиля BS_RIGHTBUTTON, то текст в правой части поля флажка или переключателя выровняйтеся по правому краю.|
|BS_CENTER|Центрирование текста по горизонтали в прямоугольнике кнопки.|
|BS_TOP|Размещает текст в верхней части прямоугольника кнопки.|
|BS_BOTTOM|Размещает текст в нижней части прямоугольника кнопки.|
|BS_VCENTER|Центрирование текста по вертикали в прямоугольнике кнопки.|

### <a name="button-content-options"></a>Параметры содержимого кнопки

В следующей таблице перечислены параметры, которые указывают, что отображается на кнопке. Типы кнопок, которые отображают только текст, игнорируют эти стили. При необходимости можно выбрать один из следующих вариантов.

|Стиль|Описание|
|-----------|-----------------|
|BS_BITMAP|Указывает, что кнопка отображает точечный рисунок.|
|BS_ICON|Указывает, что кнопка отображает значок.|
|BS_TEXT|Указывает, что кнопка отображает текст.|

### <a name="other-button-options"></a>Другие параметры кнопки

В следующей таблице перечислены дополнительные параметры, которые можно использовать с любым типом кнопки. При необходимости можно выбрать один или несколько из следующих вариантов.

|Стиль|Описание|
|-----------|-----------------|
|BS_FLAT|Указывает, что кнопка является двумерным и не рисуется с помощью заливки по умолчанию для создания трехмерного изображения.|
|BS_MULTILINE|Заключает текст кнопки в несколько строк, если текстовая строка слишком длинна, чтобы уместиться в одной строке в прямоугольнике кнопки.|
|BS_NOTIFY|Позволяет кнопке Отправить сообщения уведомления BN_DBLCLK, BN_KILLFOCUS и BN_SETFOCUS в родительское окно. Обратите внимание, что кнопки отправляют уведомление BN_CLICKED независимо от того, указан ли этот стиль.|

## <a name="combo-box-styles"></a>Стили полей со списками

В MFC доступны перечисленные далее стили полей со списками. Дополнительные сведения о стилях полей со списками в Windows см. в разделе [стили полей со списками (Windows)](/windows/win32/Controls/combo-box-styles).

|Стиль|Описание|
|-----------|-----------------|
|CBS_AUTOHSCROLL|Автоматически прокручивает текст в элементе управления "поле ввода" вправо, когда пользователь вводит символ в конце строки. Если этот стиль не задан, допускается только текст, который умещается в прямоугольной области.|
|CBS_DISABLENOSCROLL|В списке отображается отключенная вертикальная полоса прокрутки, если окно списка не содержит достаточного количества элементов для прокрутки. Если этот стиль не задан, полоса прокрутки будет скрыта при недостаточном количестве элементов в списке.|
|CBS_DROPDOWN|Аналогично CBS_SIMPLE, за исключением того, что список не отображается, если только пользователь не выберет значок рядом с элементом управления редактирования.|
|CBS_DROPDOWNLIST|Аналогично CBS_DROPDOWN, за исключением того, что элемент управления "поле ввода" заменяется статическим текстовым элементом, который отображает текущее выделение в списке.|
|CBS_HASSTRINGS|Поле со списком, рисуемое владельцем, содержит элементы, состоящие из строк. Поле со списком поддерживает память и указатели для строк, поэтому приложение может использовать функцию-член `GetText` для получения текста для определенного элемента.|
|CBS_LOWERCASE|Преобразует в нижний регистр весь текст как в поле выбора, так и в списке.|

## <a name="static-styles"></a>Статические стили

Статические стили применяются к объектам [класса кстатик](../../mfc/reference/cstatic-class.md) . Укажите сочетание стилей в параметре *Двстиле* [Кстатик:: Create](../../mfc/reference/cstatic-class.md#create). Дополнительные сведения о стилях статических элементов управления в Windows см. в разделе [стили статических элементов управления (Windows)](/windows/win32/Controls/static-control-styles).

|Стиль|Описание|
|-----------|-----------------|
|SS_BITMAP|Указывает точечный рисунок, отображаемый в статическом элементе управления. Заданный текст представляет собой имя точечного рисунка (а не имени файла), определенного в файле ресурсов. Стиль игнорирует параметры Нвидс и Нхеигхт. элемент управления автоматически изменяется в соответствии с размером растрового изображения.|
|SS_BLACKFRAME|Задает поле с фреймом, нарисованным с тем же цветом, что и рамки окна. Значением по умолчанию является Black.|
|SS_BLACKRECT|Задает прямоугольник, заполненный цветом, используемым для рисования кадров окна. Значением по умолчанию является Black.|
|SS_CENTER|Обозначает простой прямоугольник и отображает заданный текст в центре прямоугольника. Перед отображением текст форматируется. Слова, которые будут расширяться после конца строки, автоматически переносятся в начало следующей выровненной строки.|
|SS_CENTERIMAGE|Указывает, что, если битовая карта или значок меньше клиентской области статического элемента управления, оставшаяся часть клиентской области заполняется цветом пикселя в левом верхнем углу точечного рисунка или значка. Если статический элемент управления содержит одну строку текста, текст выравнивается по центру в клиентской области элемента управления.|
|SS_ENDELLIPSIS|или SS_PATHELLIPSIS заменяет часть заданной строки многоточием, если это необходимо, чтобы результат соответствовал указанному прямоугольнику. <br /><br />Можно указать SS_END_ELLIPSIS, чтобы заменить символы в конце строки, или SS_PATHELLIPSIS, чтобы заменить символы в середине строки. Если строка содержит символы обратной\\косой черты (), SS_PATHELLIPSIS сохраняет как можно больше текста после последней обратной косой черты.|
|SS_ENHMETAFILE|Указывает, что расширенный метафайл должен отображаться в статическом элементе управления. Заданный текст представляет собой имя метафайла. Статический элемент управления "расширенный метафайл" имеет фиксированный размер; метафайл масштабируется в соответствии с клиентской областью статического элемента управления.|
|SS_ETCHEDFRAME|Рисует рамку статического элемента управления с помощью стиля EDGE_ETCHED ребра.|
|SS_ETCHEDHORZ|Рисование верхнего и нижнего краев статического элемента управления с помощью стиля EDGE_ETCHED Edge.|
|SS_ETCHEDVERT|Рисует левый и правый края статического элемента управления с помощью стиля EDGE_ETCHED Edge.|
|SS_GRAYFRAME|Задает рамку с тем же цветом, что и фон экрана (Рабочий стол). По умолчанию задано значение Gray.|
|SS_GRAYRECT|Задает прямоугольник, заполненный цветом, используемым для заполнения фона экрана. По умолчанию задано значение Gray.|
|SS_ICON|Задает значок, отображаемый в диалоговом окне. Заданный текст представляет собой имя значка (а не имени файла), определенного в файле ресурсов в других местах. Параметры *нвидс* и *нхеигхт* игнорируются; Размер значка автоматически изменяется.|
|SS_LEFT|Обозначает простой прямоугольник и отображает заданный текст, расположенный слева в прямоугольнике. Перед отображением текст форматируется. Слова, которые будут превышены до конца строки, автоматически переносятся в начало следующей строки, слева от нуля.|
|SS_LEFTNOWORDWRAP|Обозначает простой прямоугольник и отображает заданный текст, расположенный слева в прямоугольнике. Вкладки разворачиваются, но слова не упаковываются. Текст, который выходит за пределы строки, обрезается.|
|SS_NOPREFIX|Если этот стиль не указан, Windows интерпретирует все символы амперсанда (&) в тексте элемента управления, чтобы они были символами префикса ускорителя. В этом случае амперсанд удаляется, а следующий символ в строке подчеркиванием. Если статический элемент управления должен содержать текст, в котором не требуется эта функция, можно добавить SS_NOPREFIX. Этот стиль статического элемента управления может включаться в любой из определенных статических элементов управления. SS_NOPREFIX можно объединить с другими стилями с помощью побитового оператора или. Чаще всего это используется, когда имена файлов или другие строки, которые могут содержать амперсанд, должны отображаться в статическом элементе управления в диалоговом окне.|
|SS_NOTIFY|Отправляет сообщения уведомления родительского окна STN_CLICKED, STN_DBLCLK, STN_DISABLE и STN_ENABLE, когда пользователь щелкает или дважды щелкает элемент управления.|
|SS_OWNERDRAW|Указывает, что владелец статического элемента управления отвечает за рисование элемента управления. Окно-владелец получает сообщение WM_DRAWITEM каждый раз, когда требуется прорисовка элемента управления.|
|SS_REALSIZEIMAGE|Предотвращает изменение размера элемента управления "статический значок" или "точечный рисунок" (т. е. статических элементов управления, имеющих стиль SS_ICON или SS_BITMAP) по мере их загрузки или прорисовки. Если значок или точечный рисунок больше области назначения, изображение будет обрезано.|
|SS_RIGHT|Назначает простой прямоугольник и отображает заданный текст справа от записи в прямоугольнике. Перед отображением текст форматируется. Слова, которые будут превышены до конца строки, автоматически переносятся в начало следующей строки с правом записи на диск.|
|SS_RIGHTJUST|Указывает, что правый нижний угол статического элемента управления с стилем SS_BITMAP или SS_ICON должен оставаться фиксированным при изменении размера элемента управления. Только верхняя и левая стороны настраиваются для размещения нового точечного рисунка или значка.|
|SS_SIMPLE|Обозначает простой прямоугольник и отображает в прямоугольнике одну строку с очисткой текста слева направо. Текстовая строка не может быть сокращена или изменена каким бы то ни было образом. (Родительское окно или диалоговое окно элемента управления не должно обрабатывать сообщение WM_CTLCOLOR.)|
|SS_SUNKEN|Рисует утопленную границу вокруг статического элемента управления.|
|SS_USERITEM|Задает определяемый пользователем элемент.|
|SS_WHITEFRAME|Задает рамку, нарисованную с тем же цветом, что и фон окна. По умолчанию цвет белый.|
|SS_WHITERECT|Задает прямоугольник, заполненный цветом, используемым для заполнения фона окна. По умолчанию цвет белый.|
|SS_WORDELLIPSIS|Усекает текст, который не умещается и добавляет многоточие.|

## <a name="window-styles"></a>Стили окна

Стили окна применяются к объектам [класса CWnd](../../mfc/reference/cwnd-class.md) . Укажите сочетание стилей в параметре *двстиле* для [CWnd:: Create](../../mfc/reference/cwnd-class.md#create) или [CWnd:: креатикс](../../mfc/reference/cwnd-class.md#createex). Дополнительные сведения о стилях окна в Windows см. в разделе [стили окон (Windows)](/windows/win32/winmsg/window-styles).

|Стиль|Описание|
|-----------|-----------------|
|WS_BORDER|Создает окно с границей.|
|WS_CAPTION|Создает окно с заголовком окна (подразумевает стиль WS_BORDER). Не может использоваться с WS_DLGFRAME стилем.|
|WS_CHILD|Создает дочернее окно. Не может использоваться с WS_POPUP стилем.|
|WS_CHILDWINDOW|То же, что и стиль WS_CHILD.|
|WS_CLIPCHILDREN|Исключает область, занятую дочерними окнами при рисовании внутри родительского окна. Используется при создании родительского окна.|
|WS_CLIPSIBLINGS|Обрезает дочерние окна относительно друг друга; то есть, когда конкретное дочернее окно получает сообщение Paint, стиль WS_CLIPSIBLINGS обрезает все остальные перекрывающиеся дочерние окна из региона дочернего Window, который будет обновлен. (Если WS_CLIPSIBLINGS не задано и дочерние окна перекрываются, при рисовании в клиентской области дочернего окна можно нарисовать в клиентской области соседнего дочернего окна.) Для использования только с стилем WS_CHILD.|
|WS_DISABLED|Создает окно, которое изначально отключено.|
|WS_DLGFRAME|Создает окно с двойной границей, но без заголовка.|
|WS_GROUP|Задает первый элемент управления группы элементов управления, в которой пользователь может перемещаться от одного элемента управления к другому с помощью клавиш со стрелками. Все элементы управления, определенные с WS_GROUP стилем "ложь" после первого элемента управления, принадлежат к одной группе. Следующий элемент управления с стилем WS_GROUP запускает следующую группу (то есть одна группа заканчивается, где начинается следующая).|
|WS_HSCROLL|Создает окно с горизонтальной полосой прокрутки.|
|WS_ICONIC|Создает окно, которое изначально свернется. То же, что и стиль WS_MINIMIZE.|
|WS_MAXIMIZE|Создает окно с максимальным размером.|
|WS_MAXIMIZEBOX|Создает окно, в котором есть кнопка Развернуть.|
|WS_MINIMIZE|Создает окно, которое изначально свернется. Для использования только с стилем WS_OVERLAPPED.|
|WS_MINIMIZEBOX|Создает окно с кнопкой сворачивания.|
|WS_OVERLAPPED|Создает перекрывающееся окно. Перекрывающиеся окна обычно имеют заголовок и границу.|
|WS_OVERLAPPEDWINDOW|Создает перекрывающееся окно с стилями WS_OVERLAPPED, WS_CAPTION, WS_SYSMENU, WS_THICKFRAME, WS_MINIMIZEBOX и WS_MAXIMIZEBOX.|
|WS_POPUP|Создает всплывающее окно. Не может использоваться с WS_CHILD стилем.|
|WS_POPUPWINDOW|Создает всплывающее окно с стилями WS_BORDER, WS_POPUP и WS_SYSMENU. Стиль WS_CAPTION необходимо сочетать с стилем WS_POPUPWINDOW, чтобы сделать меню управления видимым.|
|WS_SIZEBOX|Создает окно с границей размера. То же, что и стиль WS_THICKFRAME.|
|WS_SYSMENU|Создает окно с полем оконного меню в строке заголовка. Используется только для окон со строками заголовка.|
|WS_TABSTOP|Указывает одно из нескольких элементов управления, через которое пользователь может перемещаться с помощью клавиши TAB. Клавиша TAB перемещает пользователя к следующему элементу управления, заданному стилем WS_TABSTOP.|
|WS_THICKFRAME|Создает окно с толстой рамкой, которую можно использовать для изменения размера окна.|
|WS_TILED|Создает перекрывающееся окно. Перекрывающиеся окна имеют строку заголовка и границу. То же, что и стиль WS_OVERLAPPED.|
|WS_TILEDWINDOW|Создает перекрывающееся окно с стилями WS_OVERLAPPED, WS_CAPTION, WS_SYSMENU, WS_THICKFRAME, WS_MINIMIZEBOX и WS_MAXIMIZEBOX. То же, что и стиль WS_OVERLAPPEDWINDOW.|
|WS_VISIBLE|Создает окно, которое изначально видимо.|
|WS_VSCROLL|Создает окно с вертикальной полосой прокрутки.|

## <a name="extended-window-styles"></a>Расширенные стили окна

Расширенные стили окна применяются к объектам [класса CWnd](../../mfc/reference/cwnd-class.md) . Укажите сочетание стилей в параметре *Двексстиле* [CWnd:: креатикс](../../mfc/reference/cwnd-class.md#createex). Дополнительные сведения о расширенных стилях окон в Windows см. в разделе [Расширенные стили окон (Windows)](/windows/win32/winmsg/extended-window-styles).

|Стиль|Описание|
|-----------|-----------------|
|WS_EX_ACCEPTFILES|Указывает, что окно, созданное с помощью этого стиля, допускает перетаскивание файлов.|
|WS_EX_APPWINDOW|Принудительно запускает окно верхнего уровня на панели задач, когда окно отображается.|
|WS_EX_CLIENTEDGE|Указывает, что окно имеет трехмерный вид, то есть граница с утопленной границей.|
|WS_EX_CONTEXTHELP|Содержит вопросительный знак в заголовке окна. Когда пользователь нажимает на вопросительный знак, курсор превращается в вопросительный знак с указателем. Если пользователь щелкает дочернее окно, дочерний элемент получает сообщение WM_HELP.|
|WS_EX_CONTROLPARENT|Позволяет пользователю перемещаться между дочерними окнами окна с помощью клавиши TAB.|
|WS_EX_DLGMODALFRAME|Определяет окно с двойной границей, которую можно (при необходимости) создать с помощью строки заголовка при указании флага стиля WS_CAPTION в параметре *двстиле* .|
|WS_EX_LAYERED|Окно является слоеным [окном](/windows/win32/winmsg/window-features). Этот стиль нельзя использовать, если окно имеет [стиль класса](/windows/win32/winmsg/about-window-classes) либо CS_OWNDC, либо CS_CLASSDC. Однако Microsoft Windows 8 поддерживает стиль WS_EX_LAYERED для дочерних окон, где предыдущие версии Windows поддерживают его только для окон верхнего уровня.|
|WS_EX_LEFT|Предоставляет окно универсальных свойств, выровняйте по левому краю. Это значение по умолчанию.|
|WS_EX_LEFTSCROLLBAR|Помещает вертикальную полосу прокрутки слева от клиентской области.|
|WS_EX_LTRREADING|Отображает текст окна, используя свойства порядка чтения слева направо. Это значение по умолчанию.|
|WS_EX_MDICHILD|Создает дочернее окно MDI.|
|WS_EX_NOPARENTNOTIFY|Указывает, что дочернее окно, созданное с помощью этого стиля, не будет отсылать сообщение WM_PARENTNOTIFY в родительское окно при создании или уничтожении дочернего окна.|
|WS_EX_OVERLAPPEDWINDOW|Объединяет стили WS_EX_CLIENTEDGE и WS_EX_WINDOWEDGE|
|WS_EX_PALETTEWINDOW|Объединяет стили WS_EX_WINDOWEDGE и WS_EX_TOPMOST.|
|WS_EX_RIGHT|Предоставляет окно с универсальными свойствами по правому краю окна. Это зависит от класса Window.|
|WS_EX_RIGHTSCROLLBAR|Размещает вертикальную полосу прокрутки (если она есть) справа от клиентской области. Это значение по умолчанию.|
|WS_EX_RTLREADING|Отображает текст окна, используя свойства порядка чтения справа налево.|
|WS_EX_STATICEDGE|Создает окно с трехмерным стилем границы, предназначенным для элементов, которые не принимают ввод пользователем.|
|WS_EX_TOOLWINDOW|Создает окно инструментов, которое является окном, предназначенным для использования в качестве плавающей панели инструментов. Окно инструментов содержит строку заголовка, меньшую, чем обычная строка заголовка, и заголовок окна отображается с использованием меньшего шрифта. Окно инструментов не отображается на панели задач или в окне, которое появляется, когда пользователь нажимает клавишу ALT + TAB.|
|WS_EX_TOPMOST|Указывает, что окно, созданное с помощью этого стиля, должно располагаться над всеми неверхними окнами и остаться над ним даже после отключения окна. Приложение может использовать `SetWindowPos` функцию члена для добавления или удаления этого атрибута.|
|WS_EX_TRANSPARENT|Указывает, что окно, созданное с помощью этого стиля, должно быть прозрачным. То есть все окна, находящиеся под окном, не скрываются окном. Окно, созданное с помощью этого стиля, получает сообщения WM_PAINT только после обновления всех одноуровневого окна под ним.|
|WS_EX_WINDOWEDGE|Указывает, что окно имеет границу с порожденным краем.|
