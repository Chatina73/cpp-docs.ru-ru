# <a name="styles-used-by-mfc"></a>Стили, используемые MFC

Используйте следующие флаги стиля, чтобы указать окно или контролировать внешний вид и поведение при создании соответствующего объекта MFC. В большинстве случаев эти стили устанавливаются в `Create` параметре *dwStyle* функции класса.

## <a name="button-styles"></a><a name="button-styles"></a>Стили стонок

Стили кнопок применяются к объектам [класса CButton,](../../mfc/reference/cbutton-class.md) таким как радиокнопки, флажки и кнопки. Укажите комбинацию стилей в параметре *dwStyle* [CButton::Create](../../mfc/reference/cbutton-class.md#create). Для получения дополнительной информации о [Button Styles (Windows)](/windows/win32/Controls/button-styles)стилях кнопок в Windows см.

### <a name="button-types"></a>Типы кнопок

В следующей таблице перечислены типы кнопок. Вы можете по желанию выбрать один из следующих. Если вы не укажете тип кнопки, по умолчанию BS_PUSHBUTTON.

|Тип|Описание|
|----------|-----------------|
|BS_3STATE|Создает кнопку флажка с тремя состояниями: BST_CHECKED, BST_INDETERMINATE и BST_UNCHECKED. Нажатие на кнопку отправляет BN_CLICKED уведомление в окно владельца, но не изменяет состояние кнопки. По умолчанию связанный текст отображается справа от флажка. Для отображения текста слева от флажка используйте BS_LEFTTEXT или BS_RIGHTBUTTON стиле.|
|BS_AUTO3STATE|Создает кнопку флажка с тремя состояниями: BST_CHECKED, BST_INDETERMINATE и BST_UNCHECKED. Нажатие на кнопку отправляет BN_CLICKED уведомление в окно владельца и изменяет состояние кнопки. Кнопка указывает цикл в порядке BST_CHECKED, BST_INDETERMINATE и BST_UNCHECKED. По умолчанию связанный текст отображается справа от флажка. Для отображения текста слева от флажка используйте BS_LEFTTEXT или BS_RIGHTBUTTON стиле.|
|BS_AUTOCHECKBOX|Создает кнопку флажка с двумя состояниями: BST_CHECKED и BST_UNCHECKED. Нажатие на кнопку отправляет BN_CLICKED уведомление в окно владельца и изменяет состояние кнопки. По умолчанию связанный текст отображается справа от флажка. Для отображения текста слева от флажка используйте BS_LEFTTEXT или BS_RIGHTBUTTON стиле.|
|BS_AUTORADIOBUTTON|Создает кнопку радио с двумя состояниями: BST_CHECKED и BST_UNCHECKED. Кнопки радио обычно используются в группах, при этом каждая группа имеет максимум один проверенный вариант за один раз. Нажатие на кнопку отправляет BN_CLICKED уведомление в окно владельца, устанавливает состояние нажатой кнопки радио на BST_CHECKED и устанавливает состояния всех других радиокнопок в группе кнопок для BST_UNCHECKED. По умолчанию связанный текст отображается справа от кнопки радио. Для отображения текста слева от кнопки радио используйте BS_LEFTTEXT или BS_RIGHTBUTTON стиле.|
|BS_CHECKBOX|Создает кнопку флажка с двумя состояниями: BST_CHECKED и BST_UNCHECKED. Нажатие на кнопку отправляет BN_CLICKED уведомление в окно владельца, но не изменяет состояние кнопки. По умолчанию связанный текст отображается справа от флажка. Для отображения текста слева от флажка используйте BS_LEFTTEXT или BS_RIGHTBUTTON стиле.|
|BS_COMMANDLINK|Создает кнопку ссылки команды. Кнопка ссылки команды — это кнопка команд, специфичная для Windows Vista, которая отображает зеленую стрелку слева от основного текста и заметку под основным текстом. Вы можете установить текст заметки с помощью [CButton::SetNote](../../mfc/reference/cbutton-class.md#setnote).|
|BS_DEFCOMMANDLINK|Создает кнопку ссылки команды. Кнопка ссылки команды — это кнопка команд, специфичная для Windows Vista, которая отображает зеленую стрелку слева от основного текста и заметку под основным текстом. Вы можете установить текст заметки с помощью [CButton::SetNote](../../mfc/reference/cbutton-class.md#setnote). Если кнопка находится в диалоговом поле, нажатие клавиши ENTER отправляет BN_CLICKED уведомление в диалоговую будку, даже если кнопка не имеет ввода фокусировки.|
|BS_DEFPUSHBUTTON|Создает кнопку команды, которая имеет тяжелую черную границу. Если кнопка находится в диалоговом поле, нажатие клавиши ENTER отправляет BN_CLICKED уведомление в диалоговую будку, даже если кнопка не имеет ввода фокусировки.|
|BS_DEFSPLITBUTTON|Создает кнопку разделения. Кнопка сплит-кнопки — это кнопка команды, предназначенная для Windows Vista, которая содержит кнопку, прилегающую к стрелке выпадения. При нажатии кнопки выполняется команда по умолчанию. При нажатии на стрелку выпадения появляется меню дополнительных команд. Если сплит-кнопка находится в диалоговом поле, нажатие клавиши ENTER отправляет BN_CLICKED уведомление в диалоговую будку, даже если кнопка не имеет ввода фокуса|
|BS_GROUPBOX|Создает прямоугольник, в котором другие кнопки могут быть сгруппированы. Текст, связанный с этим стилем, отображается в верхнем левом углу прямоугольника.|
|BS_OWNERDRAW|Создает нарисованную владельцем кнопку. Фрейм `DrawItem` называет метод при изменении визуального аспекта кнопки. Этот стиль должен быть установлен `CBitmapButton` при использовании класса.|
|BS_PUSHBUTTON|Создает кнопку команды, которая отправляет BN_CLICKED уведомление в окно владельца, когда пользователь нажимает на кнопку.|
|BS_RADIOBUTTON|Создает кнопку радио с двумя состояниями: BST_CHECKED и BST_UNCHECKED. Кнопки радио обычно используются в группах, при этом каждая группа имеет максимум один проверенный вариант за один раз. Нажатие на кнопку отправляет BN_CLICKED уведомление в окно владельца, но не изменяет автоматически состояние любой кнопки в группе. По умолчанию связанный текст отображается справа от кнопки радио. Для отображения текста слева от кнопки радио используйте BS_LEFTTEXT или BS_RIGHTBUTTON стиле.|
|BS_SPLITBUTTON|Создает кнопку разделения. Кнопка сплит-кнопки — это кнопка команды, предназначенная для Windows Vista, которая содержит кнопку, прилегающую к стрелке выпадения. При нажатии кнопки выполняется команда по умолчанию. При нажатии на стрелку выпадения появляется меню дополнительных команд.|
|BS_USERBUTTON|Устаревшие, но предусмотренные для совместимости с 16-битной версии Windows. Вместо этого приложения на основе Win32 должны использовать BS_OWNERDRAW.|

### <a name="radio-button-and-check-box-styles"></a>Кнопка радио и флажок стилей

В следующей таблице перечислены стили, характерные для радиокнопок и флажков. Эти стили игнорируются во всех других типах кнопок. Вы можете по желанию выбрать один или несколько из следующих.

|Стиль|Описание|
|-----------|-----------------|
|BS_LEFTTEXT|В сочетании с радиокнопкой или стилем флажка текст отображается на левой стороне кнопки радио или флажка.|
|BS_RIGHTBUTTON|В сочетании с радиокнопкой или стилем флажка текст отображается на левой стороне кнопки радио или флажка. Этот стиль идентичен BS_LEFTTEXT стилю.|
|BS_PUSHLIKE|Делает флажок или кнопку радио выглядеть и вести себя как кнопка команды. Кнопка отображается нажатой, когда ее состояние BST_CHECKED, нажата и затемнена, когда ее состояние BST_INDETERMINATE, и освобождается, когда ее состояние BST_UNCHECKED.|

### <a name="button-text-alignment-styles"></a>Стили выравнивания текста кнопки

В следующей таблице перечислены горизонтальные и вертикальные варианты выравнивания текста. Вы можете по желанию выбрать один из следующих.

|Стиль|Описание|
|-----------|-----------------|
|BS_LEFT|Слева выравнивает текст в прямоугольнике кнопки. Однако, если кнопка является флажком или радиокнопкой, которая не имеет BS_RIGHTBUTTON стиля, текст остается выровнен на правой стороне флажка или кнопки радио.|
|BS_RIGHT|Справа выравнивает текст в прямоугольнике кнопки. Однако, если кнопка является флажком или радиокнопкой, которая не имеет BS_RIGHTBUTTON стиля, текст правильно выровнен на правой стороне флажка или кнопки радио.|
|BS_CENTER|Центры текста горизонтально в прямоугольнике кнопки.|
|BS_TOP|Места текста в верхней части прямоугольника кнопки.|
|BS_BOTTOM|Места текста в нижней части прямоугольника кнопки.|
|BS_VCENTER|Центры текста вертикально в прямоугольнике кнопки.|

### <a name="button-content-options"></a>Параметры содержимого кнопки

В следующей таблице перечислены параметры, указывающие на то, что отображается в кнопке. Типы кнопк, которые отображают только текст, игнорируют эти стили. Вы можете по желанию выбрать один из следующих.

|Стиль|Описание|
|-----------|-----------------|
|BS_BITMAP|Упомянет, что кнопка отображает битную карту.|
|BS_ICON|Упомянет, что кнопка отображает значок.|
|BS_TEXT|Упомянет, что кнопка отображает текст.|

### <a name="other-button-options"></a>Другие параметры кнопки

В следующей таблице перечислены дополнительные опции, которые можно использовать с любым типом кнопки. Вы можете по желанию выбрать один или несколько из следующих.

|Стиль|Описание|
|-----------|-----------------|
|BS_FLAT|Уточняется, что кнопка является двумерной и не нарисована с затенением по умолчанию для создания трехмерного изображения.|
|BS_MULTILINE|Оберните текст кнопки в несколько строк, если текстовая строка слишком длинная, чтобы поместиться на одной строке в прямоугольнике кнопки.|
|BS_NOTIFY|Позволяет кнопке отправлять сообщения BN_DBLCLK, BN_KILLFOCUS и BN_SETFOCUS уведомления в родительское окно. Обратите внимание, что кнопки отправляют уведомление BN_CLICKED независимо от того, указан ли этот стиль.|

## <a name="combo-box-styles"></a><a name="combo-box-styles"></a>Комбо-бокс стили

В MFC доступны перечисленные далее стили полей со списками. Для получения дополнительной информации о стилях комбо-боксов в Windows [см.](/windows/win32/Controls/combo-box-styles)

|Стиль|Описание|
|-----------|-----------------|
|CBS_AUTOHSCROLL|Автоматически прокручивает текст в элементе управления редактированием вправо, когда пользователь вводит символ в конце строки. Если этот стиль не задан, допускается только текст, который умещается в прямоугольной области.|
|CBS_DISABLENOSCROLL|В поле списка отображается отключенная вертикальная панель прокрутки, когда поле списка не содержит достаточного количества элементов для прокрутки. Если этот стиль не задан, полоса прокрутки будет скрыта при недостаточном количестве элементов в списке.|
|CBS_DROPDOWN|Как и CBS_SIMPLE, за исключением того, что поле списка не отображается, если пользователь не выбирает значок рядом с элементом управления отсеиваний.|
|CBS_DROPDOWNLIST|Как и CBS_DROPDOWN, за исключением того, что элемент управления редактированием заменяется элементом статического текста, отображая текущий выбор в поле списка.|
|CBS_HASSTRINGS|Коробка комбо-владельца содержит элементы, состоящие из строк. Поле со списком поддерживает память и указатели для строк, поэтому приложение может использовать функцию-член `GetText` для получения текста для определенного элемента.|
|CBS_LOWERCASE|Преобразует сяпоние всего текста как в поле выбора, так и в списке.|
|CBS_NOINTEGRALHEIGHT|Уточняется, что размер комбо-коробки точно такого размера, указанного приложением при создании комбо-коробки. Как правило, в Windows размер поля со списком определяется так, чтобы в избежать частичного отображения элементов.|

|SS_BITMAP|Определяет битную карту, которая должна отображаться в статическом элементе управления. Данный текст — это имя битовой карты (не имя файла), определенное в другом месте файла ресурса. Стиль игнорирует параметры nWidth и nHeight; элемент управления автоматически размерирует сядки, чтобы приспособить биткарту.|
|SS_BLACKFRAME|Определяет поле с рамой, нарисованной с тем же цветом, что и оконные рамы. Значением по умолчанию является Black.|
|SS_BLACKRECT|Определяет прямоугольник, наполненный цветом, используемым для рисования оконных рам. Значением по умолчанию является Black.|
|SS_CENTER|Обозначает простой прямоугольник и отображает данный текст, центре в прямоугольнике. Текст отформатирован перед отображением. Слова, которые будут распространяться мимо конца строки, автоматически обернуты к началу следующей по центру линии.|
|SS_CENTERIMAGE|Украшиваю, что, если биткарта или значок меньше, чем область клиента статического элемента управления, остальная часть клиентской области заполнена цветом пикселя в верхнем левом углу битной карты или значка. Если статическое управление содержит одну строку текста, текст централись вертикально в клиентской области элемента управления.|
|SS_ENDELLIPSIS|или SS_PATHELLIPSIS Заменяет часть данной строки эллипсами, если это необходимо, так что результат помещается в указанном прямоугольнике. <br /><br />Можно указать SS_END_ELLIPSIS заменить символы в конце строки или SS_PATHELLIPSIS заменить символы в середине строки. Если строка содержит\\backslash ( ) символы, SS_PATHELLIPSIS сохраняет как можно больше текста после последнего backslash насколько это возможно.|
|SS_ENHMETAFILE|Определяет увеличенный метафайл, который должен отображаться в статическом элементе управления. Данный текст — это название метафайла. Улучшенный метафилет статический элемент управления имеет фиксированный размер; метафайл масштабируется в соответствии с клиентской зоной элемента элемента управления.|
|SS_ETCHEDFRAME|Рисует рамку статического управления с помощью стиля EDGE_ETCHED края.|
|SS_ETCHEDHORZ|Рисует верхние и нижние края статического управления, используя стиль EDGE_ETCHED края.|
|SS_ETCHEDVERT|Рисует левый и правый края статического управления с помощью стиля EDGE_ETCHED края.|
|SS_GRAYFRAME|Определяет поле с рамкой, нарисованной с тем же цветом, что и фон экрана (рабочий стол). По умолчанию задано значение Gray.|
|SS_GRAYRECT|Определяет прямоугольник, заполненный цветом, используемым для заполнения фона экрана. По умолчанию задано значение Gray.|
|SS_ICON|Обозначает значок, отображаемый в диалоговом окне. Данный текст — это имя значка (не имя файла), определенное в другом месте файла ресурса. Параметры *nWidth* и *nHeight* игнорируются; значок автоматически размерирует ся.|
|SS_LEFT|Обозначает простой прямоугольник и отображает данный текст, флеш-левый в прямоугольнике. Текст отформатирован перед отображением. Слова, которые будут распространяться мимо конца строки, автоматически обернуты к началу следующей флеш-левой линии.|
|SS_LEFTNOWORDWRAP|Обозначает простой прямоугольник и отображает данный текст, флеш-левый в прямоугольнике. Вкладки расширяются, но слова не обернуты. Текст, который простирается мимо конца строки, обрезается.|
|SS_NOPREFIX|Если этот стиль не указан, Windows будет интерпретировать любые символы амперсанда (&) в тексте элемента управления, чтобы быть символами приставки акселератора. В этом случае усилитель удаляется и подчеркивается следующий символ в строке. Если статический элемент управления содержит текст, в котором эта функция не нужна, могут быть добавлены SS_NOPREFIX. Этот статический стиль управления может быть включен в любой из определенных статических элементов управления. Вы можете объединить SS_NOPREFIX с другими стилями с помощью bitwise или оператора. Это чаще всего используется, когда имена файлов или другие строки, которые могут содержать амперсанд должны быть отображены в статическом элементе управления в диалоговом поле.|
|SS_NOTIFY|Отправляет родительское окно STN_CLICKED, STN_DBLCLK, STN_DISABLE и STN_ENABLE сообщения уведомлений, когда пользователь нажимает или дважды щелкает элемент управления.|
|SS_OWNERDRAW|Уточняется, что владелец статического элемента управления отвечает за рисование элемента управления. Окно владельца получает WM_DRAWITEM сообщение всякий раз, когда необходимо нарисовать элемент управления.|
|SS_REALSIZEIMAGE|Предотвращает изменение размера статического значка или элемента управления биткартой (т.е. статических элементов управления, которые имеют SS_ICON или SS_BITMAP стиль) при загрузке или нарисованном виде. Если значок или битная карта больше, чем область назначения, изображение обрезано.|
|SS_RIGHT|Обозначает простой прямоугольник и отображает данный текст флеш-право в прямоугольнике. Текст отформатирован перед отображением. Слова, которые будут распространяться мимо конца строки, автоматически обернуты к началу следующей флеш-правой линии.|
|SS_RIGHTJUST|Уточняется, что нижний правый угол элемента управления с SS_BITMAP или SS_ICON стилем должен оставаться фиксированным при повторном размере элемента управления. Только верхняя и левая стороны корректируются для размещения новой бит-карты или значка.|
|SS_SIMPLE|Обозначает простой прямоугольник и отображает одну строку текста, флеш-левой в прямоугольнике. Строка текста не может быть сокращена или изменена каким-либо образом. (Родительское окно или поле диалога элемента управления не должны обрабатывать WM_CTLCOLOR сообщение.)|
|SS_SUNKEN|Рисует полузатонувшую границу вокруг статического элемента управления.|
|SS_USERITEM|Определяет элемент, определяемый пользователем.|
|SS_WHITEFRAME|Определяет поле с рамкой, нарисованной с тем же цветом, что и фон окна. По умолчанию цвет белый.|
|SS_WHITERECT|Определяет прямоугольник, заполненный цветом, используемым для заполнения фона окна. По умолчанию цвет белый.|
|SS_WORDELLIPSIS|Truncates текст, который не подходит и добавляет эллипсы.|

## <a name="window-styles"></a><a name="window-styles"></a>Стили окна

Стили окон применяются к объектам [класса CWnd.](../../mfc/reference/cwnd-class.md) Укажите комбинацию стилей в параметре *dwStyle* [CWnd::Create](../../mfc/reference/cwnd-class.md#create) или [CWnd::CreateEx](../../mfc/reference/cwnd-class.md#createex). Для получения дополнительной информации о [Window Styles (Windows)](/windows/win32/winmsg/window-styles)стилях окон в Windows см.

|Стиль|Описание|
|-----------|-----------------|
|WS_BORDER|Создает окно, имеющее границу.|
|WS_CAPTION|Создает окно с заголовком (подразумевает WS_BORDER стиль). Не может быть использован с WS_DLGFRAME стилем.|
|WS_CHILD|Создает детское окно. Не может быть использован с WS_POPUP стилем.|
|WS_CHILDWINDOW|Так же, как и WS_CHILD стиль.|
|WS_CLIPCHILDREN|Исключает область, занимаемую детскими окнами, когда вы рисуете в родительском окне. Используется при создании родительского окна.|
|WS_CLIPSIBLINGS|Клипы детских окон относительно друг друга; то есть, когда определенное окно ребенка получает сообщение о краске, WS_CLIPSIBLINGS зажимах стиля все другие перекрытые окна ребенка из области окна ребенка, которые будут обновлены. (Если WS_CLIPSIBLINGS не дается и детские окна перекрываются, когда вы рисуете в клиентской области окна ребенка, можно нарисовать в клиентской области соседнего окна ребенка.) Для использования только в WS_CHILD стиле.|
|WS_DISABLED|Создает окно, которое изначально отключено.|
|WS_DLGFRAME|Создает окно с двойной границей, но без названия.|
|WS_GROUP|Определяет первый элемент управления группой элементов управления, в котором пользователь может перемещаться от одного элемента управления к другому со клавишами стрелки. Все элементы управления, определяемые с WS_GROUP стилем FALSE после первого элемента управления, принадлежат одной и той же группе. Следующий элемент управления со стилем WS_GROUP запускает следующую группу (т.е. одна группа заканчивается там, где начинается следующая).|
|WS_HSCROLL|Создает окно с горизонтальной панелью прокрутки.|
|WS_ICONIC|Создает окно, которое изначально сведено к минимуму. То же, что и WS_MINIMIZE стиль.|
|WS_MAXIMIZE|Создает окно максимального размера.|
|WS_MAXIMIZEBOX|Создает окно с кнопкой «Максимизация».|
|WS_MINIMIZE|Создает окно, которое изначально сведено к минимуму. Для использования только в WS_OVERLAPPED стиле.|
|WS_MINIMIZEBOX|Создает окно с кнопкой «Минимизация».|
|WS_OVERLAPPED|Создает перекрытое окно. Перекрытое окно обычно имеет заголовок и границу.|
|WS_OVERLAPPEDWINDOW|Создает перекрываемые окна с WS_OVERLAPPED, WS_CAPTION, WS_SYSMENU, WS_THICKFRAME, WS_MINIMIZEBOX и WS_MAXIMIZEBOX стилями.|
|WS_POPUP|Создает всплывающее окно. Не может быть использован с WS_CHILD стилем.|
|WS_POPUPWINDOW|Создает всплывающее окно с WS_BORDER, WS_POPUP и WS_SYSMENU стилями. Стиль WS_CAPTION должен сочетаться с WS_POPUPWINDOW стилем, чтобы сделать меню Управления видимым.|
|WS_SIZEBOX|Создает окно с границей размеров. Так же, как и WS_THICKFRAME стиль.|
|WS_SYSMENU|Создает окно, которое имеет поле Control-menu в заголовке бара. Используется только для окон с заголовком баров.|
|WS_TABSTOP|Упоняет один из любого количества элементов управления, с помощью которых пользователь может перемещаться с помощью ключа TAB. Ключ TAB перемещает пользователя к следующему элементу управления, указанному WS_TABSTOP стилем.|
|WS_THICKFRAME|Создает окно с толстой рамкой, которая может быть использована для размера окна.|
|WS_TILED|Создает перекрытое окно. Перекрытое окно имеет заголовок бара и границы. То же, что и WS_OVERLAPPED стиль.|
|WS_TILEDWINDOW|Создает перекрываемые окна с WS_OVERLAPPED, WS_CAPTION, WS_SYSMENU, WS_THICKFRAME, WS_MINIMIZEBOX и WS_MAXIMIZEBOX стилями. То же самое, что и WS_OVERLAPPEDWINDOW стиль.|
|WS_VISIBLE|Создает окно, которое изначально видно.|
|Ws_vscroll|Создает окно с вертикальным баром прокрутки.|

## <a name="extended-window-styles"></a><a name="extended-window-styles"></a>Расширенные стили окон

Расширенные стили окон применяются к объектам [класса CWnd.](../../mfc/reference/cwnd-class.md) Укажите комбинацию стилей в параметре *dwExStyle* [CWnd::CreateEx](../../mfc/reference/cwnd-class.md#createex). Для получения дополнительной информации о расширенных стилях окон в Windows [см.](/windows/win32/winmsg/extended-window-styles)

|Стиль|Описание|
|-----------|-----------------|
|WS_EX_ACCEPTFILES|Упомянет, что окно, созданное с помощью этого стиля, принимает файлы перетаскивания.|
|WS_EX_APPWINDOW|Привизить окно верхнего уровня на панель задач, когда окно видно.|
|WS_EX_CLIENTEDGE|Уточняется, что окно имеет 3D вид, то есть границу с затонувшим краем.|
|WS_EX_CONTEXTHELP|Включает вопросительный знак в заглавной панели окна. Когда пользователь нажимает на вопросительный знак, курсор меняется на вопросительный знак с указателем. Если пользователь щелкает окно ребенка, ребенок получает WM_HELP сообщение.|
|WS_EX_CONTROLPARENT|Позволяет пользователю перемещаться среди детских окон окне, используя ключ TAB.|
|WS_EX_DLGMODALFRAME|Обозначает окно с двойной границей, которая может (по желанию) быть создана с заголовком, когда вы указываете флаг WS_CAPTION стиля в параметре *dwStyle.*|
|WS_EX_LAYERED|Окно представляет собой [многослойное окно.](/windows/win32/winmsg/window-features) Этот стиль не может быть использован, если в окне есть [стиль класса](/windows/win32/winmsg/about-window-classes) CS_OWNDC или CS_CLASSDC. Тем не менее, Microsoft Windows 8 поддерживает WS_EX_LAYERED стиле для детских окон, где предыдущие версии Windows поддерживают его только для окон верхнего уровня.|
|WS_EX_LEFT|Придает оконным общим левым выровненным свойствам. Это значение по умолчанию.|
|WS_EX_LEFTSCROLLBAR|Места вертикальной панели прокрутки слева от области клиента.|
|WS_EX_LTRREADING|Отображает текст окна с помощью свойств ордера на чтение слева направо. Это значение по умолчанию.|
|WS_EX_MDICHILD|Создает окно ребенка MDI.|
|WS_EX_NOPARENTNOTIFY|Упцирует, что окно ребенка, созданное с помощью этого стиля, не будет отправлять WM_PARENTNOTIFY сообщение в родительское окно при создании или уничтожении окна ребенка.|
|WS_EX_OVERLAPPEDWINDOW|Сочетает в себе WS_EX_CLIENTEDGE и WS_EX_WINDOWEDGE стилей|
|WS_EX_PALETTEWINDOW|Сочетает в себе WS_EX_WINDOWEDGE и WS_EX_TOPMOST стили.|
|WS_EX_RIGHT|Дает окону общие правильные выровненные свойства. Это зависит от класса окон.|
|WS_EX_RIGHTSCROLLBAR|Места вертикальной панели прокрутки (если присутствует) справа от области клиента. Это значение по умолчанию.|
|WS_EX_RTLREADING|Отображает текст окна с помощью свойств ордера на чтение справа налево.|
|WS_EX_STATICEDGE|Создает окно с трехмерным пограничным стилем, предназначенным для элементов, не принимающих пользовательский ввод.|
|WS_EX_TOOLWINDOW|Создает окно инструмента, которое представляет собой окно, предназначенное для использования в качестве плавающей панели инструментов. Окно инструмента имеет заголовок бар, который короче, чем обычный заголовок бар, и название окна обращается с помощью меньшего шрифта. Окно инструмента не отображается в панели задач или в окне, которое появляется при нажатии пользователя на ALT-TAB.|
|WS_EX_TOPMOST|Уточняется, что окно, созданное с помощью этого стиля, должно быть размещено над всеми нетопными окнами и оставаться над ними даже при отключении окна. Приложение может использовать `SetWindowPos` функцию участника для добавления или удаления этого атрибута.|
|WS_EX_TRANSPARENT|Уточняется, что окно, созданное с помощью этого стиля, должно быть прозрачным. То есть любые окна, которые находятся под окном, не заслоняются окном. Окно, созданное с помощью этого стиля, получает WM_PAINT сообщений только после того, как все окна братьев и сестер под ним были обновлены.|
|WS_EX_WINDOWEDGE|Указывается, что окно имеет границу с поднятым краем.|
