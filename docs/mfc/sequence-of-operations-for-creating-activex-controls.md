# <a name="sequence-of-operations-for-creating-activex-controls"></a>Последовательность операций при создании элементов управления ActiveX

В следующей таблице показана роль и роль платформы при создании элементов управления ActiveX (ранее известные как элементы управления OLE).

### <a name="creating-activex-controls"></a>Создание элементов управления ActiveX

|Задача|Вы|Платформа выполняет|
|----------|------------|------------------------|
|Создание платформы элементов управления ActiveX.|Запустите мастер элементов ActiveX MFC, чтобы создать элемент управления. Укажите необходимые параметры на страницах параметров. Параметры включают тип и имя элемента управления в проекте, лицензирование, подклассы и метод "о программе".|Мастер элементов ActiveX MFC создает файлы для элемента управления ActiveX с базовой функциональностью, включая исходные файлы для приложения, страницы управления, страниц свойств или страниц. файл ресурсов; файл проекта; и другие, адаптированные к вашим спецификациям.|
|Узнайте, что предлагает элемент управления и мастер элементов управления ActiveX, не добавляя строку собственного кода.|Создайте элемент управления ActiveX и протестируйте его с помощью Internet Explorer или [образца тсткон](../overview/visual-cpp-samples.md).|У выполняемого элемента управления может быть возможность изменения размера и перемещения. В нем также имеется метод **"о программе"** (если он выбран), который может быть вызван.|
|Реализуйте методы и свойства элемента управления.|Реализуйте методы и свойства конкретного элемента управления, добавив функции-члены для предоставления доступного интерфейса к данным элемента управления. Добавьте переменные члена для хранения структур данных и используйте обработчики событий для запуска событий при определении.|Платформа уже определила карту для поддержки событий, свойств и методов элемента управления, позволяя сосредоточиться на реализации свойств и методов. Страница свойств по умолчанию доступна для просмотра, и предоставляется метод по умолчанию для поля About.|
|Создайте страницу свойств или страницы элемента управления.|Используйте Visual C++ редакторы ресурсов для визуального редактирования интерфейса страницы свойств элемента управления.<br /><br />— Создание дополнительных страниц свойств.<br />— Создание и изменение точечных рисунков, значков и курсоров.<br /><br /> Можно также протестировать страницы свойств в редакторе диалоговых окон.|Файл ресурсов по умолчанию, созданный мастером приложений MFC, предоставляет множество необходимых ресурсов. Visual C++ позволяет изменять существующие ресурсы и легко и наглядно добавлять новые ресурсы.|
|Протестируйте события, методы и свойства элемента управления.|Перестройте элемент управления и используйте тестовый контейнер, чтобы проверить правильность работы обработчиков.|Можно вызвать методы элемента управления и управлять его свойствами с помощью интерфейса страницы свойств или тестового контейнера. Кроме того, используйте тестовый контейнер для трассировки событий, возникающих в элементе управления и уведомлениях, полученных контейнером элемента управления.|
