# <a name="advantages-of-the-documentview-architecture"></a>Преимущества архитектуры "документ-представление"

Ключевое преимущество использования архитектуры документ/представление MFC — это архитектура особенно хорошо поддерживает несколько представлений одного документа. (Если не требуется несколько представлений и избыточно небольшой объем затрат, документа или представления в приложении, можно избежать архитектуры. [Альтернативы для архитектуры документ/представление](../mfc/alternatives-to-the-document-view-architecture.md).)

Предположим, что приложение позволяет пользователям просматривать числовые данные, либо в виде электронной таблицы, либо в виде диаграммы. Пользователь может потребоваться просмотреть одновременно оба необработанные данные, в виде электронной таблицы и диаграммы, полученный в результате данные. Эти отдельные представления отображаются в windows отдельный кадр или разделитель области в пределах одного окна. Теперь предположим, что пользователь может изменять данные в электронную таблицу и см. в разделе изменения сразу же отражаются на диаграмме.

В MFC представление электронной таблицы и представления диаграммы будет основываться на различные классы, производные от CView. Оба представления будут сопоставлены с объектом одного документа. Документ хранит данные (или может быть получен из базы данных). Оба представления доступа к документу и отобразить данные, получаемые из него.

Когда пользователь обновляет одно из представлений, которые просматривать вызовы объектной `CDocument::UpdateAllViews`. Эта функция уведомляет все представления документа, и каждое представление обновляется с использованием последних данных из документа. Один вызов `UpdateAllViews` синхронизирует различные представления.

Этот сценарий будет трудно кода без разделения данных из представления, особенно в том случае, если представления хранятся сами данные. С помощью документов и представлений удобно. Платформа выполняет большую часть работы координации.
