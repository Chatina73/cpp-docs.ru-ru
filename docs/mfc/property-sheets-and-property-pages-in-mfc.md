# <a name="property-sheets-and-property-pages-in-mfc"></a>Вкладки свойств и страницы свойств в MFC

Лист свойств, также известный как диалоговая коробка вкладок, представляет собой диалоговый ящик, содержащий страницы свойств. Каждая страница свойств основана на ресурсе шаблона диалога и содержит элементы управления. Он заключен на странице с вкладкой на вершине. Вкладка называет страницу и указывает ее назначение. Пользователи нажимают вкладку в листе свойств, чтобы выбрать набор элементов управления.

Используйте страницы для группы элементов управления в листе свойств в значимые наборы. Содержащийся лист свойств обычно имеет несколько собственных элементов управления. Они применяются ко всем страницам.

Листы свойств основаны на классе [CPropertySheet](../mfc/reference/cpropertysheet-class.md). Страницы свойств основаны на классе [CPropertyPage](../mfc/reference/cpropertypage-class.md).

Лист свойств — это особый вид диалогового окна, который обычно используется для изменения атрибутов некоторых внешних объектов, таких как текущий выбор в представлении. Лист свойств имеет три основные части: содержащее диалоговое окно, одну или несколько страниц свойств, показанных по одному, и вкладку в верхней части каждой страницы, которую пользователь нажимает, чтобы выбрать эту страницу. Листы свойств полезны для ситуаций, когда у вас есть несколько одинаковых групп настроек или вариантов для изменения. Лист собственности группирует информацию в легко понятной манере.

> [!NOTE]
> При попытке отображить `CPropertySheet::DoModal`лист свойств с помощью, система может создать исключение первого шанса. Это исключение происходит потому, что система пытается изменить [стили окна](../mfc/reference/styles-used-by-mfc.md#window-styles) объекта до создания объекта. Для получения дополнительной информации об этом исключении, а также о том, как избежать его или справиться с ним, [см. CPropertySheet::DoModal](../mfc/reference/cpropertysheet-class.md#domodal).
