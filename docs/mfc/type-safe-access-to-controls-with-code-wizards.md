# <a name="type-safe-access-to-controls-with-code-wizards"></a>Типобезопасный доступ к элементам управления с использованием мастеров кода

Если вы знакомы с функциями DDX, вы можете использовать свойство управления в [переменном мастере Add Member](../ide/add-member-variable-wizard.md) для создания безопасного доступа к типу. Этот подход проще, чем создание элементов управления без мастеров кода.

Если вы просто хотите получить доступ к значению элемента управления, DDX предоставляет его. Если вы хотите сделать больше, чем получить доступ к значению элемента управления, используйте переменный мастер добавить элемента, чтобы добавить переменную участника соответствующего класса в класс диалога. Прикрепите эту переменную элемента к свойству управления.

Переменные участников могут иметь свойство управления вместо свойства Значение. Свойство значения относится к типу данных, `CString` возвращенных из элемента управления, например или **int.** Свойство управления обеспечивает прямой доступ к управлению через участника данных, тип `CButton` которого является одним из классов управления в MFC, таких как или `CEdit`.

> [!NOTE]
> Для данного элемента управления вы можете, если хотите, иметь несколько переменных членов с свойством значения значения и не более одного элемента переменной с свойством управления. Можно сопоставить только один объект MFC с элементом управления, поскольку несколько объектов, прикрепленных к элементу управления, или любому другому окну, приведут к двусмысленности на карте сообщений.

Этот объект можно использовать для вызова любых функций элемента для объекта управления. Такие вызовы влияют на управление в диалоговом поле. Например, для управления флажком, представленного переменным `CButton` *m_Checkbox*типа, можно вызвать:
