# <a name="managing-data-with-document-data-variables"></a>Управление данными с помощью переменных данных документа

Реализация распределения данных документа как переменные-члены класса документа. Например, программа Scribble объявляет член данных типа `CObList` — связанного списка, который содержит указатели на `CObject` объектов. Этот список используется для хранения массивы точек, составляющих рисовать линии.

Как реализовать данные документа элемента зависит от характера приложения. Чтобы помочь вам out, MFC предоставляет группу «коллекции классов» — массивов, списков и карт (словари), включая коллекциях, основанных на шаблонах C++, а также классы, инкапсулирующие разнообразные общие типы данных, такие как `CString`, `CRect`, `CPoint`, `CSize`, и `CTime`. Дополнительные сведения об этих классах см. в разделе [Обзор библиотеки классов](../mfc/class-library-overview.md) в *Справочник по библиотеке MFC*.

При определении данных элементов в документе, вы добавите обычно функции-члены в класс документа, чтобы задать и получить элементы данных, а также другие полезные операции над ними.

Представлений, доступ к объекту документа, используя указатель на документ, установлены в режиме во время создания представления. Указатель this в функции-члены представления можно получить, вызвав `CView` функция-член `GetDocument`. Убедитесь, что приведение этого указателя в собственный тип документа. Теперь вы сможете подключаться члены общих документов через указатель.

Если передача данных часто требуется прямой доступ или вы хотите использовать закрытые члены класса документа, может потребоваться сделать представление класса friend (в терминах C++) класса документа.
