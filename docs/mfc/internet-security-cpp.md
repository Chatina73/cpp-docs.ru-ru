# <a name="internet-security-c"></a>Безопасность в Интернете (C++)

Безопасность кода является серьезной проблемой для разработчиков и пользователей Интернет-приложений. Существуют риски: вредоносный код, код, который был изменен, и код из неизвестных веб-сайтов или авторов.

Существует два основных подхода к безопасности при разработке для Интернета. Первый называется "песочницей". При таком подходе приложение ограничено определенным набором интерфейсов API и исключается из потенциально опасных, таких как файловый ввод-вывод, когда программа может уничтожить данные на компьютере пользователя. Второй реализуется с помощью цифровых подписей. Этот подход называется "shrinkwrap" для Интернета. Код проверяется и подписывается с помощью технологии закрытых ключей и открытого ключа. Перед выполнением кода проверяется его цифровая подпись, чтобы убедиться, что код находится в известном источнике, прошедшем проверку подлинности, и что код не был изменен с момента подписания.

В первом случае вы доверяете, что приложение не несет вреда и вы доверяете источнику приложения. Во второй цифровые подписи используются для проверки подлинности. Цифровая подпись — это промышленный стандарт, используемый для обнаружения и предоставления сведений об издателе кода. Его технология основана на стандартах, включая RSA и X. 509. Обычно браузеры позволяют пользователям выбирать, следует ли загружать и запускать код неизвестного происхождения.
