# <a name="command-routing-illustration"></a>Демонстрация маршрутизации команд

Чтобы продемонстрировать, рассмотрим сообщением команды из пункта меню очистить все в меню "Правка" приложение MDI. Предположим, что функция обработчика для этой команды может быть функцией-членом класса документа приложения. Вот, как команда достигает своего обработчика, после выбора пункта меню.

1. Главное окно получает сообщение, команда сначала.

1. Главное окно рамки MDI дает дочернее окно текущего активного MDI возможность обработать команду.

1. Стандартный маршрут кадра дочернего окна MDI позволяет его представление в командной строке перед проверкой собственную схему сообщений.

1. Представление сначала проверяет собственную схему сообщений и затем найти обработчик не направляет команду в его связанный документ.

1. Документ проверяет схему сообщений и находит обработчик. Вызывается эта функция-член документа и маршрутизацию останавливается.

Если документа нет обработчика, он затем маршрут команды к шаблону документа. Затем команда вернет представление, а затем фрейма окна. Наконец окна фрейма проверит схему сообщений. Если также не прошла, проверка, команда будет перенаправлен обратно в главное окно рамки MDI, а затем на объект приложения — конечному получателю необработанное команд.
