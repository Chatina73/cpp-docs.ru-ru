# <a name="c6278"></a>C6278

> предупреждение C6278: \<variable> выделено с массивом new [], но удаляется с скалярным удалением. Деструкторы не будут вызываться

Это предупреждение появляется только в коде C++ и указывает, что вызывающая функция имеет непоследовательно выделенную память с помощью оператора **new []** массива, но освобождает его с помощью скалярного **`delete`** оператора. Это поведение не определено в соответствии со стандартом C++ и реализацией Microsoft C++. По крайней мере три причины могут вызвать проблемы.

- Вызываются конструкторы для отдельных объектов в массиве, но деструкторы не вызываются.

- Если глобальные или специфичные для класса **операторы new** и **Delete** не совместимы с **оператором new []** и **operator delete []**, то, скорее всего, будут возникать непредвиденные результаты.

- Всегда рискованно полагаться на неопределенное поведение.

Последствия этого дефекта сложно предсказать. Это может привести к утечке классов с деструкторами, которые выполняют отмену выделения памяти. несоответствие поведения для классов с деструкторами, которые выполняют некоторую семантическую операцию. или повреждения памяти и сбои при переопределении операторов. В других случаях несоответствие может быть неважным в зависимости от реализации компилятора и его библиотек. Средство анализа не всегда различает эти ситуации.

Если память выделяется с помощью массива **new []**, то, как правило, она должна быть освобождена с помощью **Delete массива []**.

## <a name="example"></a>Пример

Это предупреждение создается в следующем примере кода:

```cpp
class A
{
  // members
};

void f( )
{
  A *pA = new A[5];
  // code ...
  delete pA;
}
```

Чтобы исправить это предупреждение, используйте следующий пример кода:

```cpp
void f( )
{
  A *pA = new A[5];
  // code ...
  delete[] pA;
}
```

Если базовый объект в массиве является примитивным типом, например, **`int`** , **`float`** **`enum`** или указателем, то нет необходимости вызывать деструкторы. В этих случаях вместо этого выводится предупреждение [C6283](../code-quality/c6283.md) .

Использование новых и удаляемых элементов имеет много ловушек с точки зрения утечек памяти и исключений. Чтобы избежать такого рода утечек и проблем с исключениями, используйте механизмы, предоставляемые библиотекой стандартных шаблонов C++ (STL). К ним относятся [shared_ptr](../standard-library/shared-ptr-class.md), [unique_ptr](../standard-library/unique-ptr-class.md)и [vector](../standard-library/vector.md). Дополнительные сведения см. в разделе [интеллектуальные указатели](../cpp/smart-pointers-modern-cpp.md) и [Стандартная библиотека C++](../standard-library/cpp-standard-library-reference.md).
