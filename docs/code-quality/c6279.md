# <a name="c6279"></a>C6279

> предупреждение C6279: \<variable> выделено скалярным New, удалено с массивом Delete []

Это предупреждение появляется только в коде C++ и указывает, что вызывающая функция имеет неоднородное выделение памяти с помощью скалярного **`new`** оператора, но освобождает его с помощью оператора array **Delete []** . Если память выделяется с помощью скаляра **`new`** , она обычно должна быть освобождена с помощью скаляра **`delete`** .

По крайней мере три причины могут вызвать проблемы.

- Конструкторы для отдельных объектов в массиве не вызываются, хотя деструкторы являются.

- Если глобальный (или специфичный для класса) **оператор New** и **operator delete** несовместимы с **оператором new []** и **operator delete []**, то, скорее всего, будут возникать непредвиденные результаты.

Последствия этого дефекта сложно предсказать. Это может вызвать случайное поведение или сбои из-за использования неинициализированной памяти, поскольку конструкторы не вызываются. Кроме того, это может привести к выделению памяти и сбоям в ситуациях, когда операторы были переопределены. В редких случаях несоответствие может быть неважным. В настоящее время средство анализа не различает эти ситуации.

## <a name="example"></a>Пример

Это предупреждение создается в следующем коде:

```cpp
class A
{
  // members
};

void f ( )
{
  A *pA = new A;
  //code ...
  delete[] pA;
}
```

Чтобы исправить это предупреждение, используйте следующий код:

```cpp
void f( )
{
  A *pA = new A;
  //code ...
  delete pA;
}
```

Чтобы избежать подобные проблемы выделения, используйте механизмы, предоставляемые библиотекой стандартных шаблонов (STL) C++. К ним относятся [shared_ptr](../standard-library/shared-ptr-class.md), [unique_ptr](../standard-library/unique-ptr-class.md)и [vector](../standard-library/vector.md). Дополнительные сведения см. в разделе [интеллектуальные указатели](../cpp/smart-pointers-modern-cpp.md) и [Стандартная библиотека C++](../standard-library/cpp-standard-library-reference.md).
