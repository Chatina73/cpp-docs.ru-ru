# <a name="c26401-dont_delete_non_owner"></a>C26401 DONT_DELETE_NON_OWNER

Эта проверка определяет, где переход на `owner<T>` может быть хорошим вариантом для первого этапа рефакторинга. Как и C26400, он применяет правила I. 11 и R. 3, но фокусируется на части «Release» времени существования указателя. Он предупреждает о любом вызове оператора `delete`, если его цель не является ни `owner<T>`, ни неявно предполагаемым владельцем. Дополнительные сведения см. в разделе [C26400](c26400.md) об автоматическом объявлении. Сюда входят выражения, которые ссылаются на глобальные переменные, формы и т. д.

Предупреждения C26400 и C26401 всегда возникают в [C26409](c26409.md), но они более подходят для сценариев, в которых немедленная миграция на интеллектуальные указатели нецелесообразна. В таких случаях можно сначала присвоить `owner<T>` концепцию, а C26409 временно подавить.
