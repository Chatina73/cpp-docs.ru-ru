# <a name="c6101"></a>C6101

> предупреждение C6101: Возвращение неинициализированной памяти

Успешный путь через функцию не устанавливает именованный параметр `_Out_`. Это сообщение создано на основе аннотации SAL, которая указывает, что анализируемая функция всегда завершается успешно. Функция, которая не возвращает индикации успешного или неуспешного выполнения, должна установить все свои `_Out_` Параметры, так как анализатор предполагает, что `_Out_` параметр не инициализирован до вызова функции, а функция задаст параметр, чтобы он больше не был инициализирован. Если функция указывает на успешное завершение или сбой, то в случае сбоя параметр `_Out_` не следует задавать, а также можно выявить неинициализированные расположения и избежать их. В любом случае следует избежать чтения из неинициализированных расположений. Если функция иногда не затрагивает `_Out_` параметр, который впоследствии используется, то параметр должен быть инициализирован перед вызовом функции и помечен `_Inout_` заметкой, а также более явным `_Pre_null_` или `_Pre_satisfies_()` при необходимости. "Частичный успех" можно обработать аннотацией `_When_`. Дополнительные сведения см. [в разделе Использование аннотаций SAL для сокращения числа дефектов кода C/C++](../code-quality/using-sal-annotations-to-reduce-c-cpp-code-defects.md).
