# <a name="c26433-override_explicitly"></a>C26433 OVERRIDE_EXPLICITLY

Функция должна быть помечена`override`

## <a name="c-core-guidelines"></a>Основные руководящие принципы СЗ

[C.128: Виртуальные функции должны указывать точно один из виртуальных, переопределить, или окончательный](https://github.com/isocpp/CppCoreGuidelines/blob/master/CppCoreGuidelines.md)

Компилятор не обязан четко заявить, что виртуальная функция перекрывает ее базу. Не указание `override` может привести к незначительным проблемам во время обслуживания, если виртуальная спецификация когда-либо изменяется в иерархии классов. Это также снижает читаемость и делает полиморфное поведение интерфейса менее очевидным. Если функция четко обозначена как, `override`она позволяет компилятору проверить согласованность интерфейса и помочь определить проблемы, прежде чем они проявляются во время выполнения.

## <a name="notes"></a>Примечания

1. Это правило не применимо к деструкторам. Деструкторов имеют свои особенности виртуальности.
1. Правило не помечает функции, `final`явно отмеченные как , что само по себе является особым разнообразием виртуального спекулянта.
1. Предупреждения отображаются на определениях функций, а не на декларациях. Это может ввести в заблуждение, так как определения не имеют виртуальных спецификаторов, но предупреждение по-прежнему верно.

## <a name="example--implicit-overriding"></a>Пример: Неявное переопределение

```cpp
class Shape {
public:
    virtual void Draw() = 0;
    // ...
};

class Ellipse : public Shape {
public:
    void Draw() { // C26433
        //...
    }
};
```
