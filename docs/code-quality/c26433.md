# <a name="c26433-override_explicitly"></a>C26433 OVERRIDE_EXPLICITLY

Функция должна быть помечена атрибутом `override`

## <a name="c-core-guidelines"></a>C++ Core Guidelines

[C. 128: виртуальные функции должны указывать только один из виртуальных, переопределений или завершать](https://github.com/isocpp/CppCoreGuidelines/blob/master/CppCoreGuidelines.md)

Компилятор не требует четкого состояния, чтобы виртуальная функция переопределила ее базу. Отсутствие указания `override` может привести к возникновению незначительных проблем во время обслуживания, если виртуальная спецификация когда-либо изменяется в иерархии классов. Он также снижает удобочитаемость и делает менее очевидным поведение интерфейса. Если функция явно помечена как `override` , она позволяет компилятору проверить согласованность интерфейса и помочь выявить проблемы до того, как они проявляются во время выполнения.

## <a name="notes"></a>Примечания

1. Это правило неприменимо к деструкторам. Деструкторы имеют собственные особенности виртуальных особенностей.
1. Правило не помечает функции, явно помеченные как `final` , которые сами по себе являются особым набором виртуальных описателей.
1. Предупреждения отображаются в определениях функций, а не в объявлениях. Это может вызвать путаницу, так как у определений нет виртуальных описателей, но предупреждение по-прежнему является правильным.

## <a name="example--implicit-overriding"></a>Пример: неявное переопределение

```cpp
class Shape {
public:
    virtual void Draw() = 0;
    // ...
};

class Ellipse : public Shape {
public:
    void Draw() { // C26433
        //...
    }
};
```
