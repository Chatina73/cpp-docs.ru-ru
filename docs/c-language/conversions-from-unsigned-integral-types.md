# <a name="conversions-from-unsigned-integral-types"></a>Преобразования из целочисленных типов без знака

Если целое число без знака преобразуется в тип Integer или с плавающей запятой, если исходное значение может быть представлено в результате типа результата, значение не изменяется.

При преобразовании целого числа без знака в целое число с большим размером значение не увеличивается. При преобразовании в целое число меньшего размера биты с высоким порядком усекаются. Результат интерпретируется с использованием типа результата, как показано в этом примере.

```C
unsigned k = 65533;
short j;

j = k;
printf_s( "%hd\n", j );   // Prints -3
```

При преобразовании целого числа без знака в тип с плавающей запятой, если исходное значение не может быть точно представлено в результирующем типе, результат будет следующим более высоким или меньшим значением.

Сведения о размерах целочисленных и типов с плавающей запятой см. в разделе [хранилище базовых типов](../c-language/storage-of-basic-types.md) .

**Блок, относящийся только к системам Microsoft**

В компиляторе Майкрософт **неподписанные** (или **неподписанные int**) и **Long без знака** имеют уникальные, но эквивалентные типы. Преобразование значения **unsigned int** выполняется так же, как и преобразование **unsigned long**.
