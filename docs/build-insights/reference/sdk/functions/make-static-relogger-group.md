# <a name="makestaticreloggergroup"></a>макестатикрелогжерграуп

::: moniker range="<=vs-2015"

Пакет C++ SDK для Build Insights совместим с Visual Studio 2017 и более поздних версий. Чтобы просмотреть документацию по этим версиям, присвойте элементу управления "Выбор версий Visual Studio" для этой статьи значение Visual Studio 2017 или Visual Studio 2019.

::: moniker-end
::: moniker range=">=vs-2017"

Функция `MakeStaticReloggerGroup` используется для создания статической группы повторного ведения журнала, которую можно передать в такие функции, как [Relog](relog.md). Члены группы повторного ведения журнала получают события по одному слева направо, пока все события в трассировке не будут обработаны.

## <a name="syntax"></a>Синтаксис

```cpp
template <typename... TReloggerPtrs>
auto MakeStaticReloggerGroup(TReloggerPtrs... reloggers);
```

### <a name="parameters"></a>Параметры

*Трелогжерптрс*\
Этот параметр всегда выводится.

*средства регистрации*\
Пакет параметров [ирелогжер](../other-types/irelogger-class.md) указателей, включаемых в группу "статическая регистрация". Эти указатели могут быть необработанными, `std::unique_ptr`или `std::shared_ptr`. Указатели [ианализер](../other-types/ianalyzer-class.md) также считаются `IRelogger` указателями из-за отношения наследования.

### <a name="return-value"></a>Возвращаемое значение

Статическая группа переведения журнала. Используйте ключевое слово **Auto** для записи возвращаемого значения.

## <a name="remarks"></a>Remarks

В отличие от динамических групп повторного ведения журнала, члены статической группы повторного ведения журнала должны быть известны во время компиляции. Кроме того, статическая группа переведения журнала содержит [ирелогжер](../other-types/irelogger-class.md) указатели, которые не имеют полиморфизма. При использовании статической группы повторного ведения журнала для анализа трассировки трассировки событий Windows (ETW) вызовы интерфейса `IRelogger` всегда разрешаются в объект, непосредственно указываемый элементом группы повторного ведения журнала. Такая утрата гибкости связана с возможностью ускорения обработки событий. Если элементы группы повторного ведения журнала не могут быть известны во время компиляции или если требуется полиморфизм на `IRelogger`ных указателях, рассмотрите возможность использования динамической группы повторного ведения журнала. Вы можете использовать динамическую группу переведения журнала, вызвав [макединамикрелогжерграуп](make-dynamic-relogger-group.md) .

::: moniker-end
