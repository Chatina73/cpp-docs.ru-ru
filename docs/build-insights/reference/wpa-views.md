# <a name="reference-windows-performance-analyzer-views"></a>Справка: Представления анализа производительности Windows

::: moniker range="<=vs-2017"

Инструменты для построения СЗ доступны в Visual Studio 2019. Чтобы увидеть документацию для этой версии, установите элемент управления **селектора** визуальной версии для этой статьи на Visual Studio 2019. Он находится в верхней части таблицы содержимого на этой странице.

::: moniker-end
::: moniker range="vs-2019"

В этой статье приводится подробная информация о каждом из представлений « Построить Исследования», доступных в анализаторе производительности Windows (WPA). Используйте эту страницу, чтобы найти:

- описания столбцов данных; И
- доступные пресеты для каждого представления, включая их предполагаемое использование и предпочтительный режим просмотра.

Если вы новичок в WPA, мы рекомендуем вам сначала ознакомиться с [основами WPA для исследования сборки.](/cpp/build-insights/tutorials/wpa-basics)

## <a name="build-explorer"></a>Обозреватель построений

Представление Build Explorer используется для:

- диагностировать проблемы параллелизма,
- определить, доминирует ли время сборки путем разбора, генерации кода или ссылок, а также
- выявлять узкие места и необычно длительную построить.

### <a name="build-explorer-view-data-columns"></a>Создание столбцов данных представления Explorer

| Имя столбца | Описание |
|-|-|
| СборкаВременнаялинияОписание | Текстовое описание временной шкалы текущего действия или свойства происходит на. |
| СборкаВременная линия          | Идентификатор с нулевым уровнем времени для временной шкалы текущего действия или свойства происходит на. |
| Компонент                | Компонент, комбинесирирующийся или связанный при испускании текущего события. Значение этой колонки — * \<Информация\> «Вызов X»,* когда с этим событием нет компонента. X — уникальный численный идентификатор для выполнения вызова на момент испускаемого события. Этот идентификатор такой же, как и в столбце InvocationId для этого события. |
| Count                    | Количество действий или свойств, представленных в этой строке данных. Это значение всегда 1 и полезно только в сценариях агрегации, когда несколько строк сгруппированы. |
| ExclusiveCPUTime         | Количество времени процессора в миллисекундах, используемых этим действием. Время, затрачиваемые на деятельность ребенка, не включено в эту сумму. |
| ExclusiveDuration        | Миллисекундная продолжительность действия. Продолжительность действий ребенка не включена в эту сумму. |
| ИнклюзивныйCPUTime         | Количество времени процессора в миллисекундах, используемых этим действием и всеми действиями ребенка. |
| ИнклюзивныйПродолжительност        | Миллисекундная продолжительность этой деятельности, включая все действия ребенка. |
| ВызовОписание    | Произошло текстовое описание вызова, в которое произошло это событие. Описание включает в себя, было ли это *cl.exe* или *link.exe*, и уникальный числовый идентификатор вызова. Если это применимо, он включает в себя полный путь к компоненту, составленному или связанному во время призыва. Для вызовов, которые не создают компонента, или для тех, которые создают несколько компонентов, путь является пустым. Идентификатор вызова такой же, как и в столбце «ПризывИк». |
| ВызовId             | Уникальный численный идентификатор для вызова, в котором произошло это событие. |
| Имя                     | Название действия или свойства, представленного этим событием. |
| Time                     | Метка времени, идентифицирует, когда произошло событие. |
| Средство                     | Выполнение инструмента при возникновении этого события. Значение этого столбца — CL или Link. |
| Тип                     | Тип текущего события. Это значение является либо активностью, либо свойством. |
| Значение                    | Если текущее событие является свойством, то в этом столбце содержится его значение. Этот столбец остается пустым, когда текущее событие является действием. |

### <a name="build-explorer-view-presets"></a>Сборка предпосыла представления Explorer

| Предустановленное имя           | Предпочтительный режим просмотра | Использование |
|-----------------------|---------------------|------------|
| Статистика деятельности   | График / Таблица       | Используйте эту предустановленную для просмотра агрегированных статистических данных для всех действий Build Explorer. В режиме таблицы сразу расскажите, доминирует ли сборка разбора, генерации кода или связующего. Агрегированные длительности для каждого вида деятельности сортируются в порядке убывания. Просверлить, расширяя верхний узлы, чтобы легко найти, какие вызовы занять больше всего времени для этих верхних видов деятельности. Если хотите, вы можете настроить параметры WPA, чтобы показать средние значения или другие типы агрегаций. В режиме графика смотрите, когда каждое действие активируется во время сборки. |
| Вызовов           | График               | Прокрутите вниз список вызовов в представлении графика, отсортированном по времени начала. Вы можете использовать его вместе с представлением процессора (Sampled) для поиска вызовов, которые согласуются с низкими зонами использования процессора. Обнаружение проблем с параллелизмом. |
| Свойства вызова | Таблица               | Быстро найдите ключевую информацию о данном компиляторе или ссылке. Определите его версию, рабочий каталог или полную командную строку, используемую для ее использования. |
| График выполнения процессов             | График               | Смотрите диаграмму панели того, как ваша сборка была параллелизирована. Определите проблемы параллелизма и узкие места с первого взгляда. Назначьте WPA, чтобы присвоить различные значения барам в соответствии с вашими потребностями. Выберите описания вызовов в качестве последнего сгруппированного столбца для просмотра цветового штрих-графа всех ваших призывов. Это поможет вам быстро определить трудоемкие виновники. Затем масштабируйте и выберите имя действия в качестве последнего сгруппированного столбца, чтобы увидеть самые длинные части. |

## <a name="files"></a>Файлы

Представление файлов используется для:

- определить, какие заголовки включаются чаще всего, и
- помочь вам решить, что включить в предварительно составленный заголовок (PCH).

### <a name="files-view-data-columns"></a>Столбцы данных файлов

| Имя столбца              | Описание |
|--------------------------|-------------|
| ActivityName             | Действие, находясь в процессе испускаемого этого события файла. В настоящее время это значение всегда *разбор*. |
| СборкаВременнаялинияОписание | * |
| СборкаВременная линия          | * |
| Компонент                | * |
| Count                    | * |
| Глубина                    | Положение на нулевой основе в включении дерева, в котором находится этот файл. Подсчет начинается в корне дерева включают. Значение 0 обычно соответствует файлу .c/.cpp. |
| ExclusiveDuration        | * |
| ВключеноПо               | Полный путь файла, включаввв в него текущий файл. |
| ВключеноПат             | Полный путь текущего файла. |
| ИнклюзивныйПродолжительност        | * |
| ВызовId             | * |
| StartTime                | Метка времени, представляющая время, в которое было излучено текущее событие файла. |
| Средство                     | * |

\*Значение этой колонки такое же, как и в представлении [Build Explorer.](#build-explorer-view-data-columns)

### <a name="files-view-presets"></a>Предмножества файлов представления

| Предустановленное имя | Предпочтительный режим просмотра | Использование |
|-------------|---------------------|------------|
| Статистика  | Таблица               | Узнайте, какие файлы имели наивысшее агрегированное время разбора, посмотрев на список в порядке убывания. Используйте эту информацию, чтобы помочь вам реструктурировать заголовки или решить, что включить в PCH. |

## <a name="functions"></a>Функции

Представление функций используется для идентификации функций с чрезмерно длительным временем генерации кода.

### <a name="functions-view-data-columns"></a>Столбы данных функций

| Имя столбца              | Описание |
|--------------------------|-------------|
| ActivityName             | Действие, находясь в процессе испускаемого этого события функции. В настоящее время это значение всегда *CodeGeneration*. |
| СборкаВременнаялинияОписание | * |
| СборкаВременная линия          | * |
| Компонент                | * |
| Count                    | * |
| Duration                 | Продолжительность действия генерации кода для этой функции. |
| FunctionName             | Название функции, подвергающееся генерации кода. |
| ВызовId             | * |
| StartTime                | Отметка времени, отопоедающая при испускании текущего события функции. |
| Средство                     | * |

\*Значение этой колонки такое же, как и в представлении [Build Explorer.](#build-explorer-view-data-columns)

### <a name="functions-view-presets"></a>Предмножества функций представления

| Предустановленное имя | Предпочтительный режим просмотра | Использование |
|-------------|---------------------|------------|
| Статистика  | Таблица               | Узнайте, какие функции имели наивысшее агрегированное время генерации кода, взглянув на список в порядке убывания. Они могут намекать, где ваш код устраняет **__forceinline** ключевое слово, или что некоторые функции могут быть слишком большими. |
| График выполнения процессов   | График               | Посмотрите на этот график бар, чтобы узнать расположение и продолжительность функций, которые занимают больше всего времени для создания. Просмотрите, совпадают ли они с узкими местами в представлении Build Explorer. Если это так, примите соответствующие меры, чтобы сократить время генерации кода и принести пользу времени сборки. |
