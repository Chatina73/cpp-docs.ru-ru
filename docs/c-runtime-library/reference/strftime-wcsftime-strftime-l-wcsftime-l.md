# <a name="strftime-wcsftime-strftimel-wcsftimel"></a>strftime, wcsftime, _strftime_l, _wcsftime_l

Форматируют строку времени.

## <a name="syntax"></a>Синтаксис

```C
size_t strftime(
   char *strDest,
   size_t maxsize,
   const char *format,
   const struct tm *timeptr
);
size_t _strftime_l(
   char *strDest,
   size_t maxsize,
   const char *format,
   const struct tm *timeptr,
   _locale_t locale
);
size_t wcsftime(
   wchar_t *strDest,
   size_t maxsize,
   const wchar_t *format,
   const struct tm *timeptr
);
size_t _wcsftime_l(
   wchar_t *strDest,
   size_t maxsize,
   const wchar_t *format,
   const struct tm *timeptr,
   _locale_t locale
);
```

### <a name="parameters"></a>Параметры

*strDest*<br/>
Выходная строка.

*параметр MaxSize*<br/>
Размер *strDest* буфера, выраженную в символах (**char** или **wchar_t**).

*format*<br/>
Строка управления форматом.

*timeptr*<br/>
**TM** структуру данных.

*locale*<br/>
Используемый языковой стандарт.

## <a name="return-value"></a>Возвращаемое значение

**strftime** возвращает количество символов в *strDest* и **wcsftime** возвращает соответствующее число расширенных символов.

Если общее число символов, включая завершающий символ null, является более *maxsize*, оба **strftime** и **wcsftime** возвращают 0 и содержимое  *strDest* не определены.

Число символов в *strDest* равен количеству литеральными символами в *формат* плюс все символы, которые могут быть добавлены к *формат* через коды форматирования. нуль-символ, завершающий строку, не учитывается в возвращаемом значении.

## <a name="remarks"></a>Примечания

**Strftime** и **wcsftime** формат функции **tm** время значение в *timeptr* в соответствии с предоставленным  *формат* аргумент и сохраняют результат в буфере *strDest*. В основном *maxsize* символов, помещаются в строку. Описание полей в *timeptr* структуры, см. в разделе [asctime](asctime-wasctime.md). **wcsftime** является эквивалентом расширенных символов **strftime**; ее указатель на строку аргумент указывает на строку расширенных символов. В остальном эти функции ведут себя одинаково.

Эта функция проверяет свои параметры. Если *strDest*, *формат*, или *timeptr* является пустым указателем, или если **tm** структуру данных, адресуемый другим *timeptr* является недопустимым (например, если он содержит значения вне допустимого диапазона времени или даты), или если *формат* строка содержит недопустимый код форматирования, вызывается обработчик недопустимого параметра, как описано в [Проверка параметров](../../c-runtime-library/parameter-validation.md). Если выполнение может быть продолжено, функция возвращает 0 и устанавливает **errno** для **EINVAL**.

### <a name="generic-text-routine-mappings"></a>Сопоставления подпрограмм обработки обычного текста

|Подпрограмма TCHAR.H|_UNICODE и _MBCS не определены|_MBCS определено|_UNICODE определено|
|---------------------|------------------------------------|--------------------|-----------------------|
|**_tcsftime**|**strftime**|**strftime**|**wcsftime**|

*Формат* аргумент состоит из одного или нескольких кодов; как **printf**, коды форматирования начинаются знаком процента (**%**). Символы, которые не начинаются с **%** копируются без изменений, чтобы *strDest*. **LC_TIME** категории текущего языкового стандарта влияет на форматирование выходных данных функции **strftime**. (Дополнительные сведения о **LC_TIME**, см. в разделе [setlocale](setlocale-wsetlocale.md).) **Strftime** и **wcsftime** функции используют текущий языковой языкового стандарта. **_Strftime_l** и **_wcsftime_l** версии этих функций идентичны, за исключением того, что они принимают в качестве параметра языковой стандарт и использовать его вместо текущего языкового языкового стандарта. Для получения дополнительной информации см. [Locale](../../c-runtime-library/locale.md).

**Strftime** функции поддерживают эти коды форматирования:

|||
|-|-|

|**%c**|Представление даты и времени, соответствующее языковому стандарту|
|**%C**|Год разделено на 100 и округленное до целого числа, как десятичное число (00−99)|
|**%d**|День месяца в виде десятичного числа (01-31)|
|**%D**|Эквивалентно **%m/%d/%y**|
|**%e**|День месяца, как десятичное число (1-31), где один цифрам предшествуют пробел|
|**%F**|Эквивалентно **%Y-%m-%d**|
|**%g**|Две последние цифры года ISO 8601 Неделя, определяемая как десятичное число (00 - 99)|
|**%G**|ISO 8601 основе неделя года как десятичное число|
|**%h**|Сокращенное название месяца (эквивалентно **%b**)|

|**%I**|Час в 12-часовом формате (01 – 12)|

|**%r**|12-часовом формате времени языкового стандарта|

|**%S**|Секунды как десятичное число (00 - 59)|
|**%t**|Символ горизонтальной табуляции (**\t**)|
|**%T**|Эквивалентно **% H: % M: %S**, формат времени ISO 8601|
|**%u**|ISO 8601 День недели как десятичное число (1 – 7; Понедельник — 1)|
|**%U**|Номер недели года как десятичное число (00 – 53), где первое воскресенье-первый день недели 1|
|**%V**|ISO 8601 Неделя число как десятичное число (00 – 53)|
|**%w**|День недели как десятичное число (0 – 6; 0 — воскресенье)|
|**%W**|Номер недели года как десятичное число (00 – 53), где первый понедельник — первый день недели 1|

|**%X**|Представление времени для языкового стандарта|
|**%y**|Год без века как десятичное число (00 - 99)|
|**%Y**|Год с веком как десятичное число|
|**%z**|Смещение от времени UTC в формате ISO 8601; никакие символы, если часовой пояс неизвестен|
|**%Z**|Языкового стандарта часового пояса имя или сокращенное название часового пояса, в зависимости от настроек реестра; никакие символы, если часовой пояс неизвестен|
|**%%**|Знак процента|

Как и в **printf** функции **#** любому коду форматирования может предшествовать флаг. В этом случае значение кода формата изменяется следующим образом.

|Код формата|Значение|
|-----------------|-------------|
|**% #a**, **%#A**, **%#b**, **%#B**, **%#g**, **%#G**, **%#h**, **%#n**, **%#p**, **%#t**, **%#u**, **%#w**, **%#X** , **%#z**, **%#Z**, **%#%**|**#** флаг игнорируется.|
|**%#c**|Долго представление даты и времени, соответствующее языковому стандарту. Например: "Tuesday, March 14, 1995, 12:41:29".|
|**%#x**|Длинный формат даты, соответствующей языковому стандарту. Например: "Tuesday, March 14, 1995".|
|**%#d**, **%#D**, **%#e**, **%#F**, **%#H**, **% #I**, **%#j**, **%#m**, **%#M**, **%#r**, **%#R**, **%#S**, **%#T** , **%#U**, **%#V**, **%#W**, **%#y**, **%#Y**|Удаление начальных нулей или пробелов (если таковые имеются).|

ISO 8601 Неделя и основе неделю года, полученных при **%V**, **%g**, и **%G**, использует недели, который начинается в понедельник, где 1-я неделя — неделя, на 4 января, который является первым Неделя, которая включает по крайней мере четыре дня года. Если первый понедельник года вторая, 3-й или 4-й, выше дней являются частью последней недели предыдущего года. На эти дни **%V** заменяется 53 и оба **%g** и **%G** заменяются цифры предыдущего года.
