# <a name="_sopen_s-_wsopen_s"></a>_sopen_s, _wsopen_s

Открывает файл для общего доступа. Это версии функций [_sopen and _wsopen](sopen-wsopen.md) с усовершенствованной безопасностью, как описано в разделе [Функции безопасности в CRT](../../c-runtime-library/security-features-in-the-crt.md).

## <a name="syntax"></a>Синтаксис

```C
errno_t _sopen_s(
   int* pfh,
   const char *filename,
   int oflag,
   int shflag,
   int pmode
);
errno_t _wsopen_s(
   int* pfh,
   const wchar_t *filename,
   int oflag,
   int shflag,
   int pmode,
);
```

### <a name="parameters"></a>Параметры

*pfh*<br/>
Дескриптор файла или значение -1 в случае ошибки.

*Имени файла*<br/>
Имя файла.

*офлаг*<br/>
Разрешенные типы операций.

*шфлаг*<br/>
Разрешенные типы общего доступа.

*pmode*<br/>
Настройка разрешений.

## <a name="return-value"></a>Возвращаемое значение

Значение ненулевой доходности указывает на ошибку; в этом случае **errno** устанавливается на одно из следующих значений.

|Значение errno|Условие|
|-|-|
| **EACCES** |  Заданный путь является каталогом, или файл доступен только для чтения, однако была выполнена попытка операции открытия для записи. |
| **EEXIST** |  **_O_CREAT** и **_O_EXCL** флаги были указаны, но *имя файла* уже существует. |
| **EINVAL** |  Недействительным *oflag*, *шфлаг*, или *pmode* аргумент, или *pfh* или *filename* был нулевой указатель. |
| **EMFILE** | Больше нет доступных дескрипторов файлов. |
| **ENOENT** | Файл или путь не найден. |

Если в функцию передается недопустимый аргумент, вызывается обработчик недопустимых параметров, как описано в разделе [Проверка параметров](../../c-runtime-library/parameter-validation.md). Если выполнение разрешено продолжать, **errno** устанавливается **eINVAL** и **EINVAL** возвращается.

Для получения дополнительной информации об этих и других кодах возврата [_sys_nerr _sys_errlist _doserrno](../../c-runtime-library/errno-doserrno-sys-errlist-and-sys-nerr.md)см.

В случае ошибки,-1 возвращается через *pfh* (если *pfh* не является нулевым указателем).

## <a name="remarks"></a>Remarks

Функция **_sopen_s** открывает файл, указанный *по имени файла,* и готовит файл для совместного чтения или письма, как это определено *oflag* и *shflag.* **_wsopen_s** является широкохарактерным вариантом **_sopen_s;** аргумент *имя файла* для **_wsopen_s** является широкохарактерной строкой. **_wsopen_s** и **_sopen_s** ведут себя одинаково иначе.

По умолчанию глобальное состояние этой функции приспозировано к приложению. Чтобы изменить это, [см. Глобальное состояние в CRT](../global-state.md).

### <a name="generic-text-routine-mappings"></a>Универсальное текстовое сопоставление функций

|Процедура Tchar.h|_UNICODE и _MBCS не определены|_MBCS определено|_UNICODE определено|
|---------------------|--------------------------------------|--------------------|-----------------------|
|**_tsopen_s**|**_sopen_s**|**_sopen_s**|**_wsopen_s**|

Несообразное выражение *лага* формируется путем объединения одной или \<нескольких явных констант, которые определяются в> fcntl.h. Когда две или более константы образуют аргумент *лага,* они сочетаются с оператором bitwise-OR **(&#124;).**

|*oflag* константа|Поведение|
|-|-|
| **_O_APPEND** | Перемещает указатель файла в конец файла перед каждой операцией записи. |
| **_O_BINARY** | Открывает файл в двоичном режиме (без преобразования). (Описание двоичного режима см. в разделе [fopen](fopen-wfopen.md).) |
| **_O_CREAT** | Создает файл и открывает его для записи. Не имеет эффекта, если файл, указанный *именем файла,* существует. Аргумент *pmode* необходим, когда **указано _O_CREAT.** |
| **_O_SHORT_LIVED** **_O_SHORT_LIVED** _O_CREAT &#124; | Создает файл в качестве временного файла и, по возможности, не сбрасывает его на диск. Аргумент *pmode* необходим, когда **указано _O_CREAT.** |
| **_O_CREAT** **&#124; _O_TEMPORARY** | Создает файл в качестве временного файла; файл удаляется при закрытии последнего дескриптора файла. Аргумент *pmode* необходим, когда **указано _O_CREAT.** |
| **_O_CREAT** &#124;`_O_EXCL` | Возвращает значение ошибки, если файл, указанный *именем файла,* существует. Применяется только при использовании с **_O_CREAT**. |
| **_O_NOINHERIT** | Предотвращает создание общего дескриптора файла. |
| **_O_RANDOM** | Указывает, что кэширование оптимизировано для случайного доступа с диска, но не ограничивается им. |
| **_O_RDONLY** | Открывает файл только для чтения. Не удается указать **с _O_RDWR** или **_O_WRONLY**. |
| **_O_RDWR** | Открывает файл для чтения и записи. Не удается указать **с _O_RDONLY** или **_O_WRONLY**. |
| **_O_SEQUENTIAL** | Указывает, что кэширование оптимизировано для последовательного доступа с диска, но не ограничивается им. |
| **_O_TEXT** | Открывает файл в текстовом режиме (с преобразованием). (Дополнительные сведения см. в разделах [Файловый ввод-вывод в текстовом и двоичном режиме](../../c-runtime-library/text-and-binary-mode-file-i-o.md) и [fopen](fopen-wfopen.md).) |
| **_O_TRUNC** | Открывает файл и обрезает его до нулевой длины; необходимо разрешение на запись в файл. Не удается указать **с _O_RDONLY**. **_O_TRUNC,** используемый с **_O_CREAT** открывает существующий файл или создает файл. **Примечание:** **Флаг _O_TRUNC** уничтожает содержимое указанного файла. |
| **_O_WRONLY** | Открывает файл только для записи. Не удается указать **_O_RDONLY** или **_O_RDWR.** |
| **_O_U16TEXT** | Открывает файл в режиме Юникода UTF-16. |
| **_O_U8TEXT** | Открывает файл в режиме Юникода UTF-8. |
| **_O_WTEXT** | Открывает файл в режиме Юникода. |

Чтобы указать режим доступа к файлу, необходимо указать либо **_O_RDONLY,** **_O_RDWR,** либо **_O_WRONLY.** Для режима доступа нет значения по умолчанию.

При открытии файла в режиме Unicode с помощью **_O_WTEXT,** **_O_U8TEXT**или **_O_U16TEXT**функции ввода переводят данные, считываемые из файла, в данные UTF-16, хранящиеся как **wchar_t**типа. Функции, записываемые в файл, открытый в режиме Unicode, ожидают буферов, содержащих данные UTF-16, хранящиеся как **wchar_t**типа. Если кодировка файла — UTF-8, при его записи данные UTF-16 преобразуются в UTF-8, а содержимое файла с кодировкой UTF-8 преобразуется в данные UTF-16 при его считывании. Попытка чтения или записи нечетного числа байт в режиме Юникода приводит к возникновению ошибки проверки параметра. Для чтения или записи данных, хранимых в программе в кодировке UTF-8, используйте режим текстового или двоичного файла вместо режима Юникода. Вам необходимо реализовать все обязательные преобразования кодировки.

Если **_sopen_s** вызывается с **_O_WRONLY** | **_O_APPEND** (режим приложения) и **_O_WTEXT,** **_O_U16TEXT,** или **_O_U8TEXT,** он сначала пытается открыть файл для чтения и письма, читать BOM, а затем вновь открыть его для записи только. Если происходит сбой открытия файла для чтения и записи, файл открывается только для записи и для параметра режима Юникода используется значение по умолчанию.

Аргумент *шфлаг* является постоянным выражением, которое состоит из одного из \<следующих констант манифеста, которые определяются в share.h>.

|*шфлаг постоянной*|Поведение|
|-|-|
| **_SH_DENYRW** | Запрещает доступ на чтение и запись. |
| **_SH_DENYWR** | Запрещает доступ на запись. |
| **_SH_DENYRD** | Запрещает доступ на чтение. |
| **_SH_DENYNO** | Разрешает доступ на чтение и запись. |

*Аргумент pmode* всегда требуется, в отличие от **_sopen**. При **указании _O_CREAT,** если файл не существует, *pmode* указывает параметры разрешения файла, которые устанавливаются при первом закрытии нового файла. В противном *случае, pmode* игнорируется. *pmode* является инкеме выражение, которое содержит один или оба манифеста \<константы **_S_IWRITE** и **_S_IREAD,** которые определяются в sys'stat.h>. Если заданы обе константы, они объединяются с помощью битового оператора ИЛИ. Смысл *pmode* заключается в следующем.

|*pmode*|Значение|
|-|-|
| **_S_IREAD** | Разрешено только чтение. |
| **_S_IWRITE** | Разрешена запись. (Если действует, разрешает чтение и запись.) |
| **_S_IREAD** **&#124; _S_IWRITE** | Разрешены чтение и запись. |

Если разрешение на запись не предоставлено, файл остается доступным только для чтения. В операционной системе Windows все файлы доступны для чтения; невозможно предоставить разрешение только на запись. Таким образом, режимы **_S_IWRITE** и **_S_IREAD** | **_S_IWRITE** эквивалентны.

**_sopen_s** применяет текущую маску разрешения файла к *pmode* до установки разрешений. (См. [_umask](umask.md).)
