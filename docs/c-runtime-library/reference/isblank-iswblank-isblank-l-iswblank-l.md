# <a name="isblank-iswblank-_isblank_l-_iswblank_l"></a>isblank, iswblank, _isblank_l, _iswblank_l

Определяет, представляет ли целое число пустой символ.

## <a name="syntax"></a>Синтаксис

```C
int isblank(
   int c
);
int iswblank(
   wint_t c
);
int _isblank_l(
   int c,
   _locale_t locale
);
int _iswblank_l(
   wint_t c,
   _locale_t locale
);
```

### <a name="parameters"></a>Параметры

*C*<br/>
Проверяемое целое число.

*Языкового стандарта*<br/>
Используемый языковой стандарт.

## <a name="return-value"></a>Возвращаемое значение

Каждая из этих процедур возвращается ненулевой, если *c* является определенным представлением символа пространства или горизонтальной вкладки, или является одним из набора символов, специфичных для локального использования, которые используются для разделения слов в строке текста. **isblank** возвращает ненулевое значение, если *c* является космическим персонажем (0x20) или горизонтальным символом вкладок (0x09). Результат тестового условия для **излиневого** состояния зависит от LC_CTYPE **настройки** категории локализуем; для получения дополнительной информации, [см setlocale, _wsetlocale](setlocale-wsetlocale.md). Версии этих функций, которые не имеют **_l** суффикса, используют текущий локали для любого поведения, зависящем от локали; версии, которые имеют **_l** суффикс идентичны, за исключением того, что они используют локали, который прошел в вместо. Для получения дополнительной информации см. [Locale](../../c-runtime-library/locale.md).

**iswblank** возвращает ненулевое значение, если *c* является широким символом, который соответствует стандартному пространству или горизонтальному символу вкладки.

Поведение **isblank** и **_isblank_l** не определено, если *c* не EOF или в диапазоне от 0 до 0xFF, включительно. При использовании библиотеки CRT отладки и *c* не является одним из этих значений, функции повышают утверждение.
