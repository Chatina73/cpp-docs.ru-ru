# <a name="window-panes-image-editor-for-icons"></a>Области окна (редактор изображений для значков)

**Редактор изображений** окно обычно отображается в две области, отделенные линией разделения. Одно представление — это фактический размер, и другой увеличивается (коэффициент увеличения по умолчанию — 6). Представления, на этих двух панелях обновляются автоматически: изменения, внесенные в одной области сразу же отображаются в другой. Области упрощают работу на увеличенное изображение, в котором можно различать отдельные точки и, в то же время, результат работы для представления фактического размера образа.

Левой использует столько пространства, необходимого (до половины **изображение** окно) для отображения представления масштабом 1:1 (по умолчанию) образа. На правой панели отображаются увеличенное изображение (по умолчанию — увеличение 6:1). Вы можете [изменение масштаба](../windows/changing-the-magnification-factor-image-editor-for-icons.md) в каждой области с помощью **увеличить** средство [панель инструментов редактора изображений](../windows/toolbar-image-editor-for-icons.md) или с помощью сочетания клавиш.

Можно увеличить в области меньшего **редактор изображений** окно и использовать две области для отображения большого изображения. Щелкните на панели, чтобы выбрать его.

Можно изменить соотношение областей, поместив указатель на разделителе и переместив разделителе вправо или влево. Разделитель можно перемещать вплоть до обеих сторон, если вы собираетесь работать только одна область.

Если **редактор изображений** панели увеличенный вдвое 4 или более поздней версии, вы можете [отображения пиксельную сетку](../windows/displaying-or-hiding-the-pixel-grid-image-editor-for-icons.md) , разделяет отдельные точки в образе.

Сведения о добавлении ресурсов в управляемые проекты см. в разделе [ресурсы в приложениях для настольных систем](/dotnet/framework/resources/index) в *руководства разработчика .NET Framework*. Сведения о вручную добавлять файлы ресурсов в управляемые проекты, осуществлять доступ к ресурсам, отображать статические ресурсы и присваивать строки ресурсов свойствам, см. в разделе [Создание файлов ресурсов для приложений рабочего стола](/dotnet/framework/resources/creating-resource-files-for-desktop-apps). Сведения о глобализации и локализации ресурсов в управляемых приложениях, см. в разделе [Globalizing and Localizing .NET Framework Applications](/dotnet/standard/globalization-localization/index).
