# <a name="how-to-define-control-access-and-values-c"></a>Практическое руководство. Определить управление доступом и значения (C++)

## <a name="tab-order"></a>Последовательность табуляции

Последовательность табуляции — это порядок, в котором **вкладке** фокус перемещается клавишей ввода от одного элемента управления к другому в диалоговом окне. Обычно в последовательности табуляции выполняется слева направо и сверху вниз в диалоговом окне. Каждый элемент управления имеет **Tabstop** свойство, которое определяет, является ли элемент управления получает фокус ввода.

- Чтобы задать фокус ввода для элемента управления в [окно "Свойства"](/visualstudio/ide/reference/properties-window)выберите **True** или **False** в **Tabstop** свойство.

Даже элементы управления, у которых нет **Tabstop** свойство значение **True** должны быть частью последовательности табуляции, особенно для элементов управления, которые не имеют подписи. Статический текст, который содержит ключ доступа для связанный элемент управления должен находиться непосредственно перед связанный элемент управления в последовательности табуляции.

> [!NOTE]
> Если диалоговое окно содержит перекрывающиеся элементы управления, изменение последовательности табуляции может изменить способ отображения элементов управления. Элементы управления, которые далее в последовательности табуляции всегда отображаются поверх элементов управления, которые предшествуют им в последовательности табуляции.

- Для просмотра последовательности табуляции для всех элементов управления, перейдите к меню **формат** > **последовательности табуляции**, или нажмите клавишу **Ctrl** + **D**.

   Число в левом верхнем углу каждого элемента управления показаны его место в последовательности табуляции.

- Изменение последовательности табуляции для всех элементов управления, перейдите к меню **формат** > **последовательности табуляции** и установка последовательности переходов, выбрав каждый элемент управления в том порядке, в **вкладке** ключ для выполнения.

- Чтобы изменить последовательность табуляции для нескольких элементов управления, перейдите к меню **формат** > **последовательности табуляции**. Удерживая нажатой **Ctrl** ключа и выберите элемент управления, где изменения в порядке будет начать, затем отпустите **Ctrl** ключ и выбрать элементы управления в том порядке, в **вкладке** ключа следуйте этой точки.

   Например, если вы хотите изменить порядок элементов управления `7` через `9`, удерживая нажатой **Ctrl**, выберите элемент управления `6` первого.

- Для задания конкретного элемента управления в число `1`, или сначала в последовательности табуляции, дважды щелкните элемент управления.

> [!TIP]
> После ввода **последовательности табуляции** режим, нажмите клавишу **Esc** или **ввод** для выхода из **последовательности табуляции** режим и отключить возможность изменения порядка вкладок.

## <a name="mnemonics-access-keys"></a>Мнемоник (клавиш доступа)

Как правило, пользователи перемещают фокус ввода от одного элемента управления на другой в диалоговое окно с **вкладке** и **стрелку** ключи. Тем не менее можно определить ключ доступа (сокращение или легко запоминаемых имя), который позволяет пользователю выбрать элемент управления, нажав один ключ.

### <a name="to-define-an-access-key-for-a-control-with-a-visible-caption-push-buttons-check-boxes-and-radio-buttons"></a>Чтобы определить ключ доступа для элемента управления в видимом заголовке (кнопки, флажки и переключатели)

1. Выберите элемент управления в диалоговом окне.

1. В [окно "Свойства"](/visualstudio/ide/reference/properties-window)в **заголовок** свойство, введите новое имя для элемента управления, введя амперсанд (`&`) перед буквой, как клавиша доступа для этого элемента управления. Например, `&Radio1`.

1. Нажмите клавишу **ВВОД**.

   Подчеркивается в отображаемом заголовке, чтобы указать ключ доступа, например, **R**adio1.

### <a name="to-define-an-access-key-for-a-control-without-a-visible-caption"></a>Чтобы определить ключ доступа для элемента управления без видимого заголовка

1. Создайте заголовок для элемента управления с помощью **статический текст** контролировать [элементов](/visualstudio/ide/reference/toolbox).

1. В заголовке статического текста, введите знак амперсанда (`&`) перед буквой, как клавиша доступа.

1. Убедитесь, что управление статическим текстом непосредственно предшествует элементу статического в последовательности табуляции.

> [!NOTE]
> Все ключи доступа в диалоговое окно должно быть уникальным. Чтобы проверить наличие дублирующиеся клавиши доступа, перейдите к меню **формат** > **проверить назначенные клавиши**.

## <a name="combo-box-values"></a>Значения поля со списком

Можно добавить значения в поле со списком, до тех пор, пока у вас есть **редактор диалоговых окон** открыть.

> [!TIP]
> Рекомендуется добавить все значения в поле со списком *перед* размера поля в **редактор диалоговых окон**, или можно усекать текст, который должен отображаться в элементе управления поля со списком.

### <a name="to-enter-values-into-a-combo-box-control"></a>Для ввода значений в поле со списком

1. Выберите поле со списком, выбрав его.

1. В [окно "Свойства"](/visualstudio/ide/reference/properties-window), прокрутите вниз до раздела **данных** свойство.

   > [!NOTE]
   > Если вы отображаете свойства, сгруппированных по типам, **данных** отображается в **Прочее** свойства.

1. Выберите область значения для **данных** свойство и введите нужные значения данных, разделенных точкой с запятой.

   > [!NOTE]
   > Не добавляйте пробелы между значениями, так как пробелы влияют в алфавитном порядке в раскрывающемся списке.

1. Нажмите клавишу **ввод** при завершении добавления значений.

Сведения об увеличении раскрывающаяся часть поля со списком, см. в разделе [задание размера для поля со списком и его с раскрывающимся списком](setting-the-size-of-the-combo-box-and-its-drop-down-list.md).

> [!NOTE]
> Проекты Win32, с помощью этой процедуры нельзя добавлять значения ( **данных** свойство является недоступным для проектов Win32). Поскольку проекты Win32 не имеют библиотеки, поддерживающие эту возможность, необходимо добавить значения в поле со списком в проекте Win32 программным способом.

### <a name="to-test-the-appearance-of-values-in-a-combo-box"></a>Проверка значений в поле со списком

1. После ввода значений в **данных** выберите **теста** кнопку [редактор диалоговых окон инструментов](../windows/showing-or-hiding-the-dialog-editor-toolbar.md).

1. Попробуйте прокрутить список всего значения. Значения отображаются точно так, как они были введены в **данных** свойство в **свойства** окна. Нет, написание или проверка регистр букв.

1. Нажмите клавишу **Esc** для возврата **диалоговое окно** редактора.

## <a name="radio-button-values"></a>Значения "переключатель"

При добавлении переключателей в диалоговое окно, обрабатывать их как группу, задав **группы** свойство в **свойства** окно для первого переключателя в группе. После этого идентификатор элемента управления для этого переключателя появляется в [мастере добавления переменной-члена](../ide/add-member-variable-wizard.md), что позволяет добавить переменную-член для группы переключателей.

Может иметь несколько групп переключателей в диалоговом окне. Добавьте каждую группу, выполнив указанные ниже действия.

### <a name="to-add-a-group-of-radio-buttons-to-a-dialog-box"></a>Добавление группы переключателей в диалоговое окно

1. Выберите элемент управления переключателя в [окно панели элементов](/visualstudio/ide/reference/toolbox) и выбрать в диалоговом окне расположение, куда поместить элемент управления.

1. Повторите шаг выше, чтобы добавить необходимое количество элементов управления. Убедитесь, что переключатели в группе являются последовательными в последовательности табуляции.

1. В [окне свойств](/visualstudio/ide/reference/properties-window)задайте свойство **Группа** *первого* переключателя в последовательности табуляции равным **True**.

   Изменение **группы** свойства **True** добавляет стиля WS_GROUP для записи кнопки в объекте диалогового окна скрипта ресурсов и предотвращает пользователь может выбирать одновременно в более чем один переключатель в группе (если пользователь выбирает один переключатель, другим пользователям в группе очищаются).

   > [!NOTE]
   > Свойство **Группа** необходимо задать равным **True**только для первого переключателя в группе. Если у вас есть дополнительные элементы управления, которые не являются частью группы переключателей, задайте **группы** свойства первого элемента управления *, не входящего в группу* для **True** также. Можно быстро определить первый элемент управления, не входящий в группу с помощью **Ctrl**+**D** для просмотра последовательности табуляции.

### <a name="to-add-a-member-variable-for-the-radio-button-group"></a>Добавление переменной-члена для группы переключателей

1. Щелкните правой кнопкой мыши первый переключатель в последовательности табуляции (главный элемент управления и с **группы** свойству присвоено **True**) и выберите **добавить переменную**.

1. В [мастере добавления переменной-члена](../ide/add-member-variable-wizard.md)установите флажок **Переменная элемента управления** , а затем установите переключатель в положение **Значение** .

   - В поле **Имя переменной** введите имя новой переменной-члена.

   - В поле со списком **Тип переменной** выберите пункт **int** или введите *int*.

   Теперь вы можете изменить свой код, чтобы указать, какой переключатель должен быть выбранным. Например `m_radioBox1 = 0;` выбирает первый переключатель в группе.
