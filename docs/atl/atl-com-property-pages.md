# <a name="atl-com-property-pages"></a>Страницы свойств COM в ATL

Страницы свойств COM предоставляют пользовательский интерфейс для установки свойств (или вызова методов) одного или нескольких COM-объектов. Страницы свойств широко используются элементами управления ActiveX для предоставления многофункциональных пользовательских интерфейсов, позволяющих устанавливать свойства элементов управления во время разработки.

Страницы свойств — это COM-объекты, реализующие интерфейс [ипропертипаже](/windows/win32/api/ocidl/nn-ocidl-ipropertypage) или [IPropertyPage2](/windows/win32/api/ocidl/nn-ocidl-ipropertypage2) . Эти интерфейсы предоставляют методы, позволяющие связать страницу с `site` (COM-объектом, представляющим контейнер страницы) и количеством *объектов* (COM-объекты, методы которых будут вызываться в ответ на изменения, внесенные пользователем Страница свойств). Контейнер страницы свойств отвечает за вызов методов в интерфейсе страницы свойств, чтобы сообщить странице, когда следует отображать или скрывать пользовательский интерфейс, а также когда применять изменения, внесенные пользователем, к базовым объектам.

Каждая страница свойств может быть полностью построена независимо от объектов, свойства которых могут быть заданы. Все, что требуется странице свойств, — это понимание определенного интерфейса (или набора интерфейсов) и предоставление пользовательского интерфейса для вызова методов в этом интерфейсе.

Дополнительные сведения см. в разделе [вкладки свойств и страницы свойств](/windows/win32/com/property-sheets-and-property-pages) в Windows SDK.
