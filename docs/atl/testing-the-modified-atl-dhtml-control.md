# <a name="testing-the-modified-atl-dhtml-control"></a>Тестирование измененного элемента управления DHTML ATL

Попробуйте новый элемент управления, чтобы увидеть, как он работает.

## <a name="to-build-and-test-the-modified-control"></a>Построение и тестирование измененного элемента управления

1. Перестройте проект и откройте его в **тестовом контейнере**. Сведения о том, как получить доступ к **тестовому контейнеру**, см. в разделе [Тестирование свойств и событий с помощью тестового контейнера](../mfc/testing-properties-and-events-with-test-container.md) .

   Измените размер элемента управления, чтобы отобразить все добавленные кнопки.

1. Изучите две вставленные кнопки, изменив HTML. Каждая кнопка несет метку, определенную при [изменении элемента управления ATL DHTML](../atl/modifying-the-atl-dhtml-control.md): **Refresh** и **хеллохтмл**.

1. Проверьте две новые кнопки, чтобы увидеть, как они работают.

Теперь протестируйте методы, которые не являются частью пользовательского интерфейса.

1. Выделите элемент управления, и граница активируется.

1. В меню **Управление** выберите команду **вызвать методы**.

   Методы в списке с **именем Method Name** являются методами, которые контейнер может вызывать: `MethodInvoked` и `GoToURL` . Все остальные методы контролируются с помощью пользовательского интерфейса.

1. Выберите вызываемый метод и нажмите кнопку **вызвать** , чтобы отобразить окно сообщения метода или перейти к `www.microsoft.com` .

1. В диалоговом окне **Invoke Methods (вызвать методы** ) нажмите кнопку **Закрыть**.

Дополнительные сведения о различных элементах и файлах, составляющих DHTML ATL, см. в разделе [Определение элементов проекта элемента управления DHTML](../atl/identifying-the-elements-of-the-dhtml-control-project.md).
