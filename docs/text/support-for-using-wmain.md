# <a name="support-for-using-wmain"></a>Поддержка использования wmain

Визуальный C++ элемент поддерживает определение функции **wmain** и передачу в приложение Юникода аргументов с расширенными символами. Вы объявляете формальные параметры для **wmain**, используя формат, `main`аналогичный. Затем можно передать в качестве аргументов "широкие" символы и указатель среды кодировки Юникод (необязательно) в программу. Параметры `argv` и `envp` для функции **wmain** относятся к типу `wchar_t*`. Например:

```cpp
wmain( int argc, wchar_t *argv[ ], wchar_t *envp[ ] )
```

> [!NOTE]
> Приложения MFC Юникода используют `wWinMain` в качестве точки входа. В этом случае `CWinApp::m_lpCmdLine` — строка в Юникоде. Не забудьте задать `wWinMainCRTStartup` с помощью параметра компоновщика [/entry](../build/reference/entry-entry-point-symbol.md) .

Если программа использует функцию **main**, окружение многобайтовой кодировки создается библиотекой времени выполнения при запуске программы. Копия среды для Юникода создается только при необходимости (например, для вызова функции `_wgetenv` или `_wputenv`). При первом вызове метода `_wputenv`или при первом `_wgetenv` вызове метода, если среда многобайтовой кодировки уже существует, создается соответствующая среда строк расширенных символов. Затем на среду указывает `_wenviron` глобальная переменная, которая является версией `_environ` глобальной переменной для расширенных символов. На этом этапе две копии среды (MBCS и Unicode) существуют одновременно и обслуживаются системой времени выполнения в течение всего жизненного цикла программы.

Аналогичным образом, если программа использует функцию **wmain**, при запуске программы создается окружение расширенных символов и ссылка на него сохраняется в глобальной переменной `_wenviron`. Среда MBCS (ASCII) создается при первом вызове функции `_putenv` или `getenv` , на которую указывает `_environ` глобальная переменная.
