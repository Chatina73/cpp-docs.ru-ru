# <a name="internationalization-strategies"></a>Стратегии международного использования

В зависимости от целевых операционных систем и рынков у вас есть несколько стратегий интернационализации:

- Приложение использует Юникод.

   Вы используете функции для Юникода, и все символы имеют ширину 16 бит (хотя в некоторых частях программы можно использовать символы ANSI для особых целей). Библиотека времени выполнения языка C предоставляет функции, макросы и типы данных для программирования только в Юникоде. MFC полностью поддерживает Юникод.

- Приложение использует MBCS и может выполняться на любой платформе Win32.

   Используются функции многобайтовой кодировки. Строки могут содержать однобайтовые символы, двухбайтовые символы или и то, и другое. Библиотека времени выполнения языка C предоставляет функции, макросы и типы данных для программирования только MBCS. MFC полностью поддерживает многобайтовую кодировку.

- Исходный код приложения написан для обеспечения полной переносимости — путем повторной компиляции с использованием символа `_UNICODE` или `_MBCS` определенного символа можно создавать версии, использующие любой из них. Дополнительные сведения см. [в разделе Универсальные текстовые сопоставления в файле Tchar. h](../text/generic-text-mappings-in-tchar-h.md).

   Вы используете полностью переносимые функции времени выполнения C, макросы и типы данных. Гибкость MFC поддерживает любую из этих стратегий.

Оставшаяся часть этих статей сосредоточена на написании полностью переносимого кода, который можно создать как Юникод или как MBCS.
