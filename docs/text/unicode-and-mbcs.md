# <a name="unicode-and-mbcs"></a>Юникод и многобайтовая кодировка

Библиотека классов Фонда Майкрософт (MFC), библиотека c-времени для визуального СЗ и среда разработки Visual C'' могут помочь вашему международному программированию. Они предоставляют:

- Поддержка стандарта Unicode в Windows. Unicode является текущим стандартом и должен использоваться по возможности.

   Unicode — это кодирование 16-битного персонажа, обеспечивающее достаточное количество кодов для всех языков. Все символы ASCII включены в Unicode в качестве расширенных символов.

- Поддержка формы набора мультибайт ных символов (MBCS), называемого набором символов с двойным байтом (DBCS) на всех платформах.

   Символы DBCS состоят из 1 или 2 байтов. Некоторые диапазоны байтов отведены для использования в качестве свинцовых байтов. Ведущий байт указывает, что он и следующий байт след состоят из одного персонажа шириной 2 байта. Вы должны следить за тем, какие байты являются свинцово-битами. В отдельных многобайтовых кодировках старшие и младшие байты попадают в определенный диапазон. Когда эти диапазоны перекрываются, может потребоваться оценить контекст, чтобы определить, функционирует ли данный байт в качестве свинцового байта или тропы байта.

- Поддержка инструментов, упрощающие программирование MBCS приложений, написанных для международных рынков.

   При запуске на MBCS-версии операционной системы Windows, система разработки Visual C, включая интегрированный редактор исходного кода, отладчик, и инструменты командной строки - полностью включена MBCS. Для получения дополнительной информации, [см.](../text/mbcs-support-in-visual-cpp.md)

> [!NOTE]
> В этой документации MBCS используется для описания всех не-Unicode поддержки мультибайтных символов. В визуальном СЗ MBCS всегда означает DBCS. Наборы символов шире 2 байтов не поддерживаются.

По определению, набор символов ASCII представляет собой подмножество всех наборов мультибайт-символов. Во многих многобайтовых кодировках каждый символ в диапазоне от 0x00 до 0x7F совпадает с символом в кодировке ASCII, который имеет то же значение. Например, в строках символов ASCII и MBCS символ 1-байт NULL ('0') имеет значение 0x00 и указывает на термин null.
