# <a name="compiler-warning-level-4-c4458"></a>Предупреждение компилятора (уровень 4) C4458

> объявление "*идентификатор*" скрывает член класса

Объявление *идентификатора* в локальной области скрывает объявление *идентификатора* с идентичным именем в области видимости класса. Это предупреждение дает вам знать, что ссылки на *идентификатор* в этой области разрешаются в локально объявленную версию, а не в версию члена класса, которая может быть или не предполагалась. Чтобы устранить эту проблему, рекомендуется указать имена локальных переменных, которые не конфликтуют с именами членов класса.

## <a name="example"></a>Пример

Следующий пример приводит к возникновению ошибки C4458, поскольку параметр `x` и локальная переменная `y` в `member_fn` имеют те же имена, что и элементы данных в классе. Чтобы устранить эту проблему, используйте другие имена для параметров и локальных переменных.

```cpp
// C4458_hide.cpp
// compile with: cl /W4 /c C4458_hide.cpp

struct S {
    int x;
    float y;
    void member_fn(long x) {   // C4458
        double y;  // C4458
        y = x;
        // To fix this issue, change the parameter name x
        // and local name y to something that does not
        // conflict with the data member names.
    }
} s;
```
