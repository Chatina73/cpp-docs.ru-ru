# <a name="compiler-warning-level-1-c4407"></a>Предупреждение компилятора (уровень 1) C4407

приведение между разными представлениями указателя на член; компилятор может создать неверный код

Обнаружено неправильное приведение.

C4407 можно создать из-за действий по согласованности компилятора, выполненных в Visual Studio 2005. Для указателя на член теперь требуется полное имя и оператор взятия адреса (&).

C4407 может возникнуть при приведении между множественным наследованием указателя на член к одному указателю на элемент наследования. Иногда это может работать, но иногда это не так, потому что одно представление указателя на член не содержит достаточной информации. Компиляция с помощью **/VMM** может помочь (Дополнительные сведения см. в разделе [/VMM,/VMS,/vmv (общего назначения представление)](../../build/reference/vmm-vms-vmv-general-purpose-representation.md)). Также можно попробовать изменить расположение базовых классов. компилятор обнаруживает потери данных в преобразовании, так как базовый класс находится на ненулевом смещении от производного.

Следующий пример приводит к возникновению ошибки C4407:

```cpp
// C4407.cpp
// compile with: /W1 /c
struct C1 {};
struct C2 {};
struct C3 : C1, C2 {};

typedef void(C3::*PMF_C3)();
typedef void(C2::*PMF_C2)();

PMF_C2 f1(PMF_C3 pmf) {
   return (PMF_C2)pmf;   // C4407, change type of cast,
   // or reverse base class inheritance of C3 (i.e. : C2, C1)
}
```
