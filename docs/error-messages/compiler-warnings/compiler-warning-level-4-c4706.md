# <a name="compiler-warning-level-4-c4706"></a>Предупреждение компилятора (уровень 4) C4706

Присваивание в условном выражении

Тестовое значение в условном выражении было результатом присваивания.

Присваивание имеет значение (значение в левой части назначения), которое можно использовать в другом выражении, включая тестовое выражение.

Следующий пример приводит к возникновению ошибки C4706:

```cpp
// C4706a.cpp
// compile with: /W4
int main()
{
   int a = 0, b = 0;
   if ( a  = b ) // C4706
   {
   }
}
```

Предупреждение будет возникать, даже если в круглых скобках находится условие теста:

```cpp
// C4706b.cpp
// compile with: /W4
int main()
{
   int a = 0, b = 0;
   if ( ( a  =  b ) ) // C4706
   {
   }
}
```

Если ваша цель заключается в проверке связи, а не для создания назначения, используйте `==` оператор. Например, следующая строка проверяет, равны ли значения a и b:

```cpp
// C4706c.cpp
// compile with: /W4
int main()
{
   int a = 0, b = 0;
   if ( a == b )
   {
   }
}
```

Если вы хотите, чтобы тестовое значение имело результат присваивания, выполните проверку, чтобы убедиться, что присваивание не равно нулю или не имеет значение null. Например, следующий код не создаст это предупреждение:

```cpp
// C4706d.cpp
// compile with: /W4
int main()
{
   int a = 0, b = 0;
   if ( ( a = b ) != 0 )
   {
   }
}
```
