# <a name="compiler-warning-level-4-c4464"></a>Компилятор предупреждение (уровень 4) C4464

> **относительный путь включения содержит ".."**

Объект `#include` директива имеет путь содержит ".." Описатель родительского каталога.

## <a name="remarks"></a>Примечания

Начиная с Visual Studio 2015 с обновлением 1, компилятор может определить директиву include, которая содержит ".." сегмент пути и проблема создает предупреждение, если включен. Код, написанный таким образом, обычно предназначен для включения заголовков, находящихся за пределами проекта, путем неправильного использования относительных путей к проектам. Это создает риск, что компиляции программы мог включаться, включив другом файле исходного кода, не собирается программист, или что эти относительные пути невозможно было бы перенести в другие среды сборки. Несмотря на то, что имеется конкретного не предупреждение, мы также рекомендуем не использовать каталоги включаемых файлов сегмент родительский каталог пути для указания проекта.

Это предупреждение появился в Visual Studio 2015 с обновлением 1 и отключен по умолчанию. Используйте [/Wall](../../build/reference/compiler-option-warning-level.md) включить все предупреждения, отключенные по умолчанию или __/w__*n*__4464__ для включения C4464 как уровень *n* предупреждение. Дополнительные сведения см. в разделе [компилятора предупреждения, являются отключено по умолчанию](../../preprocessor/compiler-warnings-that-are-off-by-default.md). Сведения о том, как отключить предупреждения компилятора версией, см. в разделе [предупреждения компилятора по версиям компилятора](compiler-warnings-by-compiler-version.md).

## <a name="example"></a>Пример

Исходный код, содержащие ".." сегменты пути можно активировать данное предупреждение, если **/Wall** параметра:

```cpp
#include "..\headers\C4426.h"  // emits warning C4464

// To fix this issue, specify only the header file name, and add
// the absolute path to 'headers\' to your project's include directories
#include "C4426.h"
```
