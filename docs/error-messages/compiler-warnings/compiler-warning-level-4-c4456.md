# <a name="compiler-warning-level-4-c4456"></a>Предупреждение компилятора (уровень 4) C4456

> объявление "*идентификатор*" скрывает предыдущее локальное объявление

Объявление *идентификатора* в локальной области скрывает объявление предыдущего локального объявления с тем же именем. Это предупреждение позволяет знать, что ссылки на *идентификатор* в локальной области разрешаются в локально объявленную версию, а не в предыдущую локальную систему, которая может быть непреднамеренной. Чтобы устранить эту проблему, рекомендуется указать имена локальных переменных, которые не конфликтуют с другими локальными именами.

## <a name="example"></a>Пример

Следующий пример приводит к возникновению ошибки C4456, так как управляющая переменная цикла `int x` и локальная переменная `double x` в `member_fn` имеют одинаковые имена. Чтобы устранить эту проблему, используйте другие имена для локальных переменных.

```cpp
// C4456_hide.cpp
// compile with: cl /W4 /c C4456_hide.cpp

struct S {
    void member_fn(unsigned u) {
        double x = 0;
        for (int x = 0; x < 10; ++x) {  // C4456
            u += x; // uses local int x
        }
        x += u; // uses local double x
    }
} s;
```
