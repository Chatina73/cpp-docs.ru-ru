# <a name="compiler-warning-level-4-c4800"></a>Предупреждение компилятора (уровень 4) C4800

::: moniker range=">= vs-2019"
Visual Studio 2019 и более поздних версий:
> Неявное преобразование из "*тип*" в bool. Потери возможной информации
::: moniker-end

C4800 предупреждение имеет уровень 3 в Visual Studio 2015 и более ранних версий:
> "*тип*": принудительно задано логическое значение «true» или «false» (предупреждение о производительности)

Это предупреждение создается в том случае, когда значение неявно преобразуется в тип `bool`. Как правило, это сообщение в результате назначения `int` переменных `bool` переменные где `int` переменная содержит только значения **true** и **false**и может быть повторно объявлен как тип `bool`. Если невозможно перезаписать выражение для использования типа `bool`, затем можно добавить "`!=0`" к выражению, предоставляющего тип выражения `bool`. Приведение выражения к типу `bool` не отключить предупреждения, что было сделано намеренно.

::: moniker range=">= vs-2017"
Это предупреждение не создается в Visual Studio 2017.
::: moniker-end

::: moniker range=">= vs-2019"
Это предупреждение отключено по умолчанию, начиная с Visual Studio 2019. Используйте __/w__*n*__4800__ для включения C4800 как уровень *n* предупреждение "," или [/Wall](../../build/reference/compiler-option-warning-level.md) включить все предупреждения, по умолчанию отключены. Дополнительные сведения см. в разделе [компилятора предупреждения, являются отключено по умолчанию](../../preprocessor/compiler-warnings-that-are-off-by-default.md).
::: moniker-end

## <a name="example"></a>Пример

Следующий пример приводит к возникновению ошибки C4800 и показаны способы ее устранения:

```cpp
// C4800.cpp
// compile with: /W4 /w44800
int main() {
   int i = 0;

   // To fix, instead try:
   // bool i = 0;

   bool j = i;   // C4800
   j++;
}
```
