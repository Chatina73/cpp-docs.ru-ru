# <a name="compiler-warning-level-1-c4350"></a>Предупреждение компилятора (уровень 1) C4350

изменение поведения: "член 1" был вызван вместо "член 2"

Rvalue не может быть привязан к ссылке, не являющейся константной. В версиях Visual C++ до Visual Studio 2003 можно было привязать rvalue к неконстантной ссылке при прямой инициализации. Теперь этот код выдает предупреждение.

Для обеспечения обратной совместимости по-прежнему можно привязывать rvalue к неконстантным ссылкам, однако стандартные преобразования предпочтительнее везде, где это возможно.

Это предупреждение отражает изменение поведения компилятора Visual C++ .NET 2002. Если этот параметр включен, это предупреждение может быть предоставлено для правильного кода. Например, он может быть задан при использовании шаблона класса **std:: auto_ptr** .

Если вы получаете это предупреждение, проверьте код, чтобы узнать, зависит ли он от привязки rvalue к ссылкам, не являющимся константными. Добавление константы в ссылку или предоставление дополнительной перегруженной ссылки может решить проблему.

Это предупреждение отключено по умолчанию. Дополнительные сведения см. в разделе [Compiler Warnings That Are Off by Default](../../preprocessor/compiler-warnings-that-are-off-by-default.md).

Следующий пример приводит к возникновению ошибки C4350:

```cpp
// C4350.cpp
// compile with: /W1
#pragma warning (default : 4350)
class A {};

class B
{
public:
   B(B&){}
   // try the following instead:
   // B(const B&){}

   B(A){}
   operator A(){ return A();}
};

B source() { return A(); }

int main()
{
   B ap(source());   // C4350
}
```
