# <a name="compiler-warning-level-4-c4457"></a>Предупреждение компилятора (уровень 4) C4457

> объявление "*идентификатор*" скрывает параметр функции

Объявление *идентификатора* в локальной области скрывает объявление параметра функции с идентичным именем. Это предупреждение позволяет знать, что ссылки на *идентификатор* в локальной области разрешаются в локально объявленную версию, а не в параметре, который может быть или не быть вашим намерением. Чтобы устранить эту проблему, рекомендуется указать имена локальных переменных, которые не конфликтуют с именами параметров.

## <a name="example"></a>Пример

Следующий пример приводит к возникновению ошибки C4457, так как параметр `x` и локальная переменная `x` в `member_fn` имеют одинаковые имена. Чтобы устранить эту проблему, используйте другие имена для параметров и локальных переменных.

```cpp
// C4457_hide.cpp
// compile with: cl /W4 /c C4457_hide.cpp

struct S {
    void member_fn(unsigned x) {
        double a = 0;
        for (int x = 0; x < 10; ++x) {  // C4457
            a += x; // uses local x
        }
        a += x; // uses parameter x
    }
} s;
```
