# <a name="compiler-warning-level-1-c4743"></a>Предупреждение компилятора (уровень 1) C4743

"*тип*" имеет другой размер в "*file1*" и "*file2*": *число* и *число* байтов

Внешняя переменная, упоминаемая или определенная в двух файлах, имеет разные типы в этих файлах, и компилятор определил, что размер переменной в файле *file1* отличается от размера переменной в *file2*.

## <a name="remarks"></a>Комментарии

Существует важный случай, когда это предупреждение может быть выпущено для C++. Если вы объявили те же типы с одинаковыми именами в двух разных файлах, если эти объявления содержат виртуальные функции, и если объявления не совпадают, компилятор может выдать Предупреждение C4744 для таблиц виртуальных функций. Это предупреждение возникает из-за того, что для одного типа существует две таблицы виртуальных функций разных размеров, и компоновщик должен выбрать один из них для включения в исполняемый файл.  Возможно, это может привести к тому, что программа вызовет неправильную виртуальную функцию.

Чтобы устранить это предупреждение, используйте одно и то же определение типа или используйте разные имена для типов или переменных.

## <a name="example"></a>Пример

Следующий пример приводит к возникновению ошибки C4743. Чтобы скомпилировать файл, поместите оба файла в одну папку, а затем запустите  

```cmd
cl /EHsc /W1 /GL /O2 C4743a.cpp C4743b.cpp
```

```cpp
// C4743a.cpp
class C {
public:
    virtual void f1(void);
    virtual void f2(void);
    virtual void f3(void);
};

void C::f1(void) {}
void C::f2(void) {}
void C::f3(void) {}
C q;
```

```cpp
// C4743b.cpp
class C {
public:
    virtual void f1(void);
    virtual void f2(void);
    virtual void f3(void);
    virtual void f4(void);
    virtual void f5(void);
};

void C::f4(void) {}
void C::f5(void) {}
C x;

int main() {}
```
