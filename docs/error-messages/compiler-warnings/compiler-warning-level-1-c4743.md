# <a name="compiler-warning-level-1-c4743"></a>Предупреждение компилятора (уровень 1) C4743

"*тип*«имеет различный размер»*file1*«и»*" файл2 "*": *номер* и *номер* байт

Внешняя переменная или ссылаться на определенные в двух файлах существуют разные типы в этих файлах, и компилятор определила, что размер переменной в *file1* отличается от размера переменной в *"файл2"*.

## <a name="remarks"></a>Примечания

Имеется важные ситуация, когда это предупреждение может создаваться для C++. Если вы объявляете те же типы, с тем же именем в разных файлах, эти объявления содержат виртуальные функции, если объявления не совпадают, компилятор может вывести предупреждение C4744 для таблиц виртуальных функций. Предупреждение означает, что существует две таблицы нестандартного размера виртуальную функцию для одного типа, а компоновщик необходимо выбрать один из них для включения в исполняемый файл.  Вполне возможно, что может привести к программе, вызов неверную виртуальную функцию.

Чтобы устранить это предупреждение, используйте одно и то же определение типа или используйте разные имена для типов или переменных.

## <a name="example"></a>Пример

Следующий пример приводит к возникновению ошибки C4743. Чтобы скомпилировать его, поместите оба файла в той же папке, а затем выполните  

```cmd
cl /EHsc /W1 /GL /O2 C4743a.cpp C4743b.cpp
```

```cpp
// C4743a.cpp
class C {
public:
    virtual void f1(void);
    virtual void f2(void);
    virtual void f3(void);
};

void C::f1(void) {}
void C::f2(void) {}
void C::f3(void) {}
C q;
```

```cpp
// C4743b.cpp
class C {
public:
    virtual void f1(void);
    virtual void f2(void);
    virtual void f3(void);
    virtual void f4(void);
    virtual void f5(void);
};

void C::f4(void) {}
void C::f5(void) {}
C x;

int main() {}
```
