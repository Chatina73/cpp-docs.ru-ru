# <a name="compiler-warning-level-1-c4743"></a>Предупреждение компилятора (уровень 1) C4743

"*тип*«имеет различный размер»*file1*«и»*" файл2 "*": *номер* и *номер* байт

Внешняя переменная или ссылаться на определенные в двух файлах существуют разные типы в этих файлах, и компилятор определила, что размер переменной в *file1* отличается от размера переменной в *"файл2"*.

Это предупреждение может создаваться для C++ нет важный случай. Если вы объявляете те же типы, с тем же именем в разных файлах, эти объявления содержат виртуальные функции, если объявления не совпадают, компилятор может вывести предупреждение C4744 для таблиц виртуальных функций. Предупреждение означает, что существует две таблицы виртуальных функций разного размера для одного типа, а компоновщик необходимо выбрать один из них для включения в исполняемый файл.  Вполне возможно, что это может привести в приложении, вызвав неверную виртуальную функцию.

Чтобы устранить это предупреждение, используйте одно и то же определение типа или используйте разные имена для типов или переменных.

## <a name="example"></a>Пример

Этот пример содержит одно определение типа.

```
// C4743a.cpp
// compile with: /c
class C {
public:
    virtual void f1(void);
    virtual void f2(void);
    virtual void f3(void);
};

void C::f1(void) {}
void C::f2(void) {}
void C::f3(void) {}
C q;
```

## <a name="example"></a>Пример

Следующий пример приводит к возникновению ошибки C4743.

```
// C4743b.cpp
// compile with: C4743a.cpp /W1 /GL /O2
// C4743 expected
class C {
public:
    virtual void f1(void);
    virtual void f2(void);
    virtual void f3(void);
    virtual void f4(void);
    virtual void f5(void);
};

void C::f4(void) {}
void C::f5(void) {}
C x;

int main() {}
```
