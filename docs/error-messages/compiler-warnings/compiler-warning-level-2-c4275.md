# <a name="compiler-warning-level-2-c4275"></a>Предупреждение компилятора (уровень 2) C4275

> класс "*class_1*", не являющийся интерфейсом DLL, используется в качестве базового для класса интерфейса DLL "*class_2*"

Экспортированный класс был производным от класса, который не был экспортирован.

Чтобы максимально повысить вероятность повреждения данных при экспорте класса с [__declspec (dllexport)](../../cpp/dllexport-dllimport.md), убедитесь в том, что:

- Доступ ко всем статическим данным осуществляется через функции, экспортированные из библиотеки DLL.

- Встроенные методы класса не могут изменять статические данные.

- Ни один из встроенных методов класса не использует функции CRT или другие библиотечные функции, использующие статические данные.

- Ни одна встроенная функция класса не использует функции CRT или другие библиотечные функции, в которых можно получить доступ к статическим данным.

- Ни один из методов класса (вне зависимости от встраивания) не может использовать типы, в которых экземпляры в EXE и DLL имеют статические различия данных.

Вы можете избежать экспорта классов, определив библиотеку DLL, определяющую класс с виртуальными функциями, и функции, которые можно вызывать для создания и удаления объектов типа.  Затем можно просто вызвать виртуальные функции для типа.

C4275 можно игнорировать в Visual C++ при наследовании от типа в стандартной библиотеке C++, компиляции отладочного выпуска (**/MTD**) и места, на которое ссылается сообщение об ошибке компилятора `_Container_base` .

```cpp
// C4275.cpp
// compile with: /EHsc /MTd /W2 /c
#include <vector>
using namespace std;
class Node;
class __declspec(dllimport) VecWrapper : vector<Node *> {};   // C4275
```
