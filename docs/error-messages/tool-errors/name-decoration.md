# <a name="name-decoration"></a>Дополнение имен

Дополнение имен выполняется в соответствии с соглашениями об именовании C++ и также применяется в ряде случаев в языке C. По умолчанию в C++ для создания имени компоновщика для функции используются имя функции, параметры и тип возвращаемого значения. Рассмотрим следующее объявление функции:

`void CALLTYPE test(void);`

В таблице ниже представлены имена компоновщика для различных соглашений о вызовах.

|Соглашение о вызовах|`extern "C"` или `.c` файл|`.cpp`, `.cxx` или `/TP`|
|------------------------|---------------------------|------------------------|
|Соглашение об именовании C ( **`__cdecl`** )|`_test`|`?test@@ZAXXZ`|
|Соглашение об именовании быстрого вызова ( **`__fastcall`** )|`@test@0`|`?test@@YIXXZ`|
|Соглашение об именовании стандартных вызовов ( **`__stdcall`** )|`_test@0`|`?test@@YGXXZ`|
|Соглашение об именовании векторного вызова ( **`__vectorcall`** )|`test@@0`|`?test@@YQXXZ`|

Используйте `extern "C"` для вызова функции C из C++. `extern "C"` принудительно использует соглашение об именовании языка C для функций C++, не являющихся классами. Обратите внимание на параметры компилятора **/TC** или **/TP**, которые указывают компилятору игнорировать расширение имени файла и скомпилируйте файл как C или C++ соответственно. Эти параметры могут привести к невозможности ожидания имен компоновщика.

Эта ошибка также может возникать при использовании прототипов функции с несогласованными параметрами. Дополнение к имени включает параметры функции в окончательное дополненное имя функции. Вызов функции с типами параметров, которые не соответствуют параметрам в объявлении функции, может также вызвать ошибку LNK2001.

В настоящее время нет стандартов для именования C++ между поставщиками компиляторов или даже между разными версиями компилятора. Связывание объектных файлов, скомпилированных другими компиляторами, может не дать одинаковой схемы именования и привести к неразрешенным внешним объектам.
