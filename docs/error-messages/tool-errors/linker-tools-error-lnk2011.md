# <a name="linker-tools-error-lnk2011"></a>Ошибка средств компоновщика LNK2011

предкомпилированный объект не связан в; образ не может быть запущен

При использовании предварительно скомпилированных заголовков LINK требует, чтобы все объектные файлы, созданные с помощью предкомпилированных заголовков, были связаны в. При наличии исходного файла, который используется для создания предкомпилированного заголовка для использования с другими исходными файлами, теперь необходимо включить объектный файл, созданный вместе с предкомпилированным заголовком.

Например, если скомпилировать файл с именем ЗАГЛУШКи. cpp, чтобы создать предкомпилированный заголовок для использования с другими исходными файлами, необходимо выполнить компоновку с помощью файла ЗАГЛУШКи. obj или получить эту ошибку. В следующих командных строках одна строка используется для создания предкомпилированного заголовка COMMON. pch, который используется с PROG1. cpp и PROG2. cpp в двух и трех строках. Файл ЗАГЛУШЕК. cpp содержит только `#include` строки (те же `#include` строки, что и в PROG1. cpp и PROG2. cpp) и используется только для создания предкомпилированных заголовков. В последней строке параметр ЗАГЛУШЕК. obj должен быть связан в, чтобы избежать LNK2011.

```
cl /c /Yccommon.h stub.cpp
cl /c /Yucommon.h prog1.cpp
cl /c /Yucommon.h prog2.cpp
link /out:prog.exe stub.obj prog1.obj prog2.obj
```
