# <a name="c-runtime-error-r6033"></a>Ошибка времени выполнения C R6033

Попытка использовать код MSIL из этой сборки во время инициализации машинного кода. Это указывает на ошибку в приложении. Скорее всего, это результат вызова функции, скомпилированной с помощью MSIL (/CLR), из собственного конструктора или из DllMain.

> [!NOTE]
> Если при запуске приложения возникло это сообщение об ошибке, работа приложения была завершена из-за внутренней проблемы. Эта ошибка может быть вызвана ошибкой в приложении или ошибкой в надстройке или расширении, которую она использует.
>
> Для устранения этой ошибки попробуйте выполнить следующие действия:
>
> - Используйте страницу **приложения и компоненты** или **программы и компоненты** на **панели управления** , чтобы восстановить или переустановить программу.
> - Используйте страницу **приложения и компоненты** или **программы и компоненты** на **панели управления** , чтобы удалить, восстановить или переустановить расширения или надстройки.
> - Проверьте **Центр обновления Windows** на **панели управления** для обновлений программного обеспечения.
> - Проверьте наличие обновленной версии приложения. Если проблема не исчезнет, обратитесь к поставщику приложения.

**Сведения для программистов**

Это диагностическое сообщение указывает, что инструкции MSIL выполнялись во время блокировки загрузчика. Это может произойти, если скомпилирован собственный C++ с помощью флага/CLR. Используйте флаг/CLR только для модулей, содержащих управляемый код. Дополнительные сведения см. в разделе [Инициализация смешанных сборок](../../dotnet/initialization-of-mixed-assemblies.md).
