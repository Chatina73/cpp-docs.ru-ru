# <a name="linker-tools-warning-lnk4006"></a>Предупреждение средств компоновщика LNK4006

символ уже определен в объекте; Второе определение пропущено

Указанный символ `symbol`, отображаемый в декорированном виде, был определен несколько раз. При возникновении этого предупреждения `symbol` будет добавлено дважды, но будет использоваться только его первая форма.

Это предупреждение можно получить при попытке выполнить слияние двух библиотек импорта в одну.

При перестроении библиотеки времени выполнения C можно проигнорировать это сообщение.

### <a name="to-fix-by-using-the-following-possible-solutions"></a>Возможные способы устранения этой ошибки

1. Заданный `symbol` может быть упакованной функцией, созданной путем компиляции с параметром [/Gy](../../build/reference/gy-enable-function-level-linking.md). Этот символ был добавлен в более чем один файл, но был изменен между компиляциями. Перекомпилировать все файлы, содержащие `symbol`.

1. Заданная `symbol` может быть определена по-разному в двух объектах-членах в разных библиотеках.

1. Абсолютный, возможно, был определен дважды и имеет другое значение в каждом определении.

1. Если при объединении библиотек получено сообщение об ошибке, `symbol` уже существует в библиотеке, в которую добавляется.
