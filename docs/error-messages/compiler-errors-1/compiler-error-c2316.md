# <a name="compiler-error-c2316"></a>Ошибка компилятора C2316

> "*class_type*": не удается перехватить, поскольку деструктор или конструктор копии недоступен или удален

Исключение перехвачено по значению или по ссылке, но конструктор копии, оператор присваивания, или оба были недоступны.

## <a name="remarks"></a>Примечания

Изменения соответствия в Visual Studio 2015 внесенные этой ошибки, которые применимы к инструкциям неправильный catch исключений MFC, производный от `CException`. Так как `CException` имеет унаследованные закрытый конструктор копии, классу и его производные не для простого копирования не может передаваться по значению, а значит и не может быть перехвачены по значению. Выявляйте инструкций, перехваченные исключения MFC по значению, ранее привел к неперехваченных исключений во время выполнения. Теперь компилятор правильно определяет это и сообщает об ошибке C2316. Чтобы устранить эту проблему, мы рекомендуем вам использовать макросы MFC TRY/CATCH, а не создавать собственные обработчики исключений. Если это не подходит для вашего кода, перехватывайте исключения MFC по ссылке вместо этого.

## <a name="example"></a>Пример

В следующем примере возникнет ошибка C2316 и показывает способ ее устранения:

```cpp
// C2316.cpp
// compile with: /EHsc
#include <stdio.h>

struct B
{
public:
    B() {}
    // Delete the following line to resolve.
private:
    // copy constructor
    B(const B&) {}
};

void f(const B&)
{
}

int main()
{
    try
    {
        B aB;
        f(aB);
    }
    catch (B b)    // C2316
    {
        printf_s("Caught an exception!\n");
    }
}
```
