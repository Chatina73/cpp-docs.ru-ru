# <a name="compiler-error-c2316"></a>Ошибка компилятора C2316

> "*class_type*": невозможно перехватить, так как деструктор или конструктор копий недоступны или удалены

Исключение перехвачено по значению или по ссылке, но конструктор копий, оператор присваивания или и то, и другое были недоступны.

## <a name="remarks"></a>Комментарии

Изменения соответствия в Visual Studio 2015 приводят к неправильным операторам catch исключений MFC, производным от `CException` . Поскольку `CException` имеет наследуемый закрытый конструктор копии, класс и его производные классы не копируются и не могут передаваться по значению, что также означает, что они не могут быть перехвачены по значению. Операторы catch, которые ранее перехватывают исключения MFC по значению, привели к неперехваченным исключениям во время выполнения. Теперь компилятор правильно определяет эту ситуацию и сообщает об ошибке C2316. Чтобы устранить эту проблему, рекомендуется использовать макросы TRY/CATCH MFC вместо написания собственных обработчиков исключений. Если это не подходит для кода, следует перехватывать исключения MFC по ссылке.

## <a name="example"></a>Пример

Следующий пример приводит к возникновению ошибки C2316 и демонстрирует способ исправления:

```cpp
// C2316.cpp
// compile with: /EHsc
#include <stdio.h>

struct B
{
public:
    B() {}
    // Delete the following line to resolve.
private:
    // copy constructor
    B(const B&) {}
};

void f(const B&)
{
}

int main()
{
    try
    {
        B aB;
        f(aB);
    }
    catch (B b)    // C2316
    {
        printf_s("Caught an exception!\n");
    }
}
```
