# <a name="compiler-error-c2512"></a>Ошибка компилятора C2512

> "*идентификатор*": нет подходящего конструктора по умолчанию доступны

Объект *конструктор по умолчанию*, конструктор, который не требует аргументов, не доступен для указанного класса, структуры или объединения. Компилятор предоставляет конструктор по умолчанию, только в том случае, если указаны без пользовательских конструкторов.

Если вы хотите разрешить вашего класса без параметров (например, в качестве элементов в массиве) предоставляют конструктор, принимающий отличный от void параметр, необходимо также указать конструктор по умолчанию. Конструктор по умолчанию может быть конструктором со значениями по умолчанию для всех параметров.

## <a name="example"></a>Пример

Вероятная причина ошибки C2512 — при определении класса или структуры конструктор, который принимает аргументы, с последующей попыткой для объявления экземпляра этого класса или структуры без аргументов. Например `struct B` ниже объявляет конструктор, который требует `char *` аргументом, но не тот, который не принимает аргументы. В `main`, объявляется экземпляр B, но аргумент не указан. Компилятор создает ошибки C2512, поскольку не удается найти конструктор по умолчанию для б.

```cpp
// C2512.cpp
// Compile with: cl /W4 c2512.cpp
// C2512 expected
struct B {
   B (char *) {}
   // Uncomment the following line to fix.
   // B() {}
};

int main() {
   B b;   // C2512 - This requires a default constructor
}
```

Эту проблему можно устранить путем определения конструктора по умолчанию для структуры или класса, такие как `B() {}`, или конструктор, где все аргументы имеют значения по умолчанию, такие как `B (char * = nullptr) {}`.
