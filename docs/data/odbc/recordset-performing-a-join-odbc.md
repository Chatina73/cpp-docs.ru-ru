# <a name="recordset-performing-a-join-odbc"></a>Набор записей. Объединение (ODBC)

Этот раздел относится к классам ODBC библиотеки MFC.

## <a name="what-a-join-is"></a>Что такое соединение

Операция JOIN, Общая задача доступа к данным, позволяет работать с данными из нескольких таблиц с помощью одного объекта Recordset. При соединении двух или более таблиц создается набор записей, который может содержать столбцы из каждой таблицы, но будет представлен в виде одной таблицы в приложении. Иногда соединение использует все столбцы из всех таблиц, но иногда предложение SQL **SELECT** в соединении использует только некоторые столбцы из каждой таблицы. Классы базы данных поддерживают соединение только для чтения, но не подключаются к обновляемым объединениям.

Чтобы выбрать записи, содержащие столбцы из соединяемых таблиц, вам потребуются следующие элементы:

- Список таблиц, содержащий имена всех соединяемых таблиц.

- Список столбцов, содержащий имена всех участвующих столбцов. Столбцы с одинаковым именем, но из разных таблиц определяются по имени таблицы.

- Фильтр (предложение **WHERE** SQL), указывающий столбцы, к которым присоединены таблицы. Этот фильтр имеет вид "Table1. Кэйкол = Table2. Кэйкол" и фактически выполняет соединение.

Можно объединить более двух таблиц таким же образом путем сравнения нескольких пар столбцов, каждой пары, соединяемой ключевым словом SQL **и**.
