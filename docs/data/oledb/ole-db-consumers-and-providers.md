# <a name="ole-db-consumers-and-providers"></a>Получатели и поставщики OLE DB

Архитектура OLE DB использует модель потребителей и поставщиков. Потребитель выполняет запросы к данным. Поставщик отвечает на эти запросы, помещая данные в табличный формат и возвращая их потребителю. Любой вызов, который может сделать потребитель, должен быть реализован в поставщике.

Технически определяется, что потребитель — это любой код системы или приложения (не обязательно компонент OLE DB), который обращается к данным через интерфейсы OLE DB. Интерфейсы реализуются в поставщике. Таким образом, поставщик — это любой программный компонент, реализующий интерфейсы OLE DB для инкапсуляции доступа к данным и предоставления их другим объектам (то есть потребителям).

Для ролей потребитель вызывает методы в интерфейсах OLE DB. Поставщик OLE DB реализует необходимые интерфейсы OLE DB.

OLE DB не имеет смысла использовать клиент и сервер, так как эти роли не всегда имеют смысл, особенно в n-уровневой ситуации. Поскольку потребитель может быть компонентом на уровне, который обслуживает другой компонент, для его вызова клиентский компонент будет запутанным. Кроме того, поставщик иногда работает так же, как драйвер базы данных, чем сервер.
