# <a name="cc-properties-linux-c"></a>Свойства C/C++ (Linux)

::: moniker range="vs-2015"

Поддержка Linux, доступные в Visual Studio 2017 и более поздних версий.

::: moniker-end

::: moniker range=">=vs-2017"

## <a name="general"></a>Общие

Свойство. | Описание | Варианты
--- | ---| ---
Дополнительные каталоги включаемых файлов | Указывает один или несколько каталогов для добавления в путь включаемых файлов. Используйте точку с запятой для разделения нескольких каталогов. (-Я\[путь]).
Формат отладочной информации | Определяет тип отладочной информации, создаваемой компилятором. | **Нет** — отладочная информация не создается для ускорения компиляции.<br/>**Минимальные сведения об отладке** — создание минимальных сведений об отладке.<br/>**Полная отладочная информация (DWARF2)** — создание отладочной информации DWARF2.<br/>
Имя объектного файла | Указывает имя, переопределяющее имя объектного файла по умолчанию. Он может быть именем файла или каталога. (-o [name]).
Уровень предупреждений | Выбирает, как строго компилятор должен воспринимать ошибки в коде необходимо.  Добавить другие флаги непосредственно в **Дополнительные параметры**. (/w, /Weverything). | **Отключить все предупреждения** — отключает все предупреждения компилятора.<br/>**Включить все предупреждения** -включает все предупреждения, включая те, которые по умолчанию отключены.<br/>
Обрабатывать предупреждения как ошибки | Интерпретирует все предупреждения компилятора как ошибки. Для нового проекта возможно, лучше всего использовать /Werror во всех компиляциях. Устраните все предупреждения, чтобы обеспечить находимых ошибок кода жестких для поиска.
Дополнительные предупреждения C | Определяет набор дополнительных предупреждений.
Дополнительные предупреждения C++ | Определяет набор дополнительных предупреждений.
Включить подробный режим | Если включен режим подробного протоколирования, выводит Дополнительные сведения для диагностики сборки.
Компилятор C | Задает программу, вызываемую при компиляции файлов исходного кода C, или путь к компилятору C в удаленной системе.
Компилятор C++ | Задает программу, вызываемую при компиляции файлов исходного кода C++, или путь к компилятору C++ в удаленной системе.
Время ожидания компиляции | Время ожидания удаленной компиляции (в миллисекундах).
Копировать объектные файлы | Указывает, следует ли копировать скомпилированные объектные файлы с удаленной системы на локальный компьютер.

## <a name="optimization"></a>Оптимизация

Свойство. | Описание | Варианты
--- | ---| ---
Оптимизация | Указывает уровень оптимизации для приложения. | **Настраиваемый** — настраиваемая оптимизация.<br/>**Отключено** — отключить оптимизацию.<br/>**Минимизировать размер** — оптимизировать по размеру.<br/>**Максимизировать скорость** — оптимизировать по скорости.<br/>**Полная оптимизация** — дорогостоящие оптимизации.
Строгое присвоение псевдонима | Предполагается, что строгие правила присвоения псевдонимов.  Объект одного типа никогда не предполагается, что один и тот же адрес как объект другого типа.
Развернуть циклы | Unrolls циклов, чтобы приложение работало быстрее благодаря уменьшению числа выполняемых ветвей, за счет увеличения размера кода.
Оптимизация времени компоновки | Включает межпроцедурные оптимизации по разрешения оптимизатору просматривать объектные файлы в приложении.
Опустить указатель на фреймы | Отменяет создание указателей на фреймы в стеке вызовов.
Нет общих блоков | Выделяет даже неинициализированный глобальные переменные в разделе данных объектного файла, а не создавать их в виде общих блоков.

## <a name="preprocessor"></a>Препроцессор

Свойство. | Описание | Варианты
--- | ---| ---
Определения препроцессора | Определяет символы предобработки для исходного файла. (-D)
Отменить определения препроцессора | Задает одну или несколько отмен определений препроцессора.  (-U \[макрос])
Отменить все определения препроцессора | Отменяет определение всех определенных ранее значений препроцессора.  (-undef)
Показывать включаемые файлы | Создает список включаемых файлов с выводом компилятора.  (-H)

## <a name="code-generation"></a>Создание кода

Свойство. | Описание | Варианты
--- | ---| ---
Позиционно-независимый код | Создает независимый от позиции код (PIC) для использования в общей библиотеке.
Статические элементы являются потокобезопасными | Создает дополнительный код для использования процедур, указанных в C++ ABI для потокобезопасной инициализации локальных статических элементов. | **Не** — отключение потокобезопасных.<br/>**Да** — Включение потокобезопасных.
Оптимизация операций с плавающей запятой | Включает оптимизацию с плавающей запятой, послабления требований к соответствию спецификации IEEE 754.
Встроенные методы скрыты | Если этот параметр включен, вне строки копии встроенных методов объявляются `private extern`.
Символы скрыты по умолчанию | Все символы объявляются `private extern` Кроме явным образом помеченных для экспорта с помощью `__attribute` макрос.
Включить исключения C++ | Указывает модель обработки исключений, используемую компилятором. | **Нет** — отключение обработки исключений.<br/>**Да** — включение обработки исключений.
