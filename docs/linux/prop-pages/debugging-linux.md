# <a name="c-debugging-properties-linux-c"></a>Свойства отладки C++ (Linux C++)

::: moniker range="vs-2015"

Поддержка Linux, доступные в Visual Studio 2017 и более поздних версий.

::: moniker-end

::: moniker range=">=vs-2017"

Свойство. | Описание | Варианты
--- | ---| ---
Удаленный отладочный компьютер | **Visual Studio 2019 версии 16.1**: Задает компьютер, чтобы отладить программу на. Может отличаться от компьютер удаленной сборки, которое было указано на [Общие](general-linux.md) страницы.
Команда перед запуском | Запускает команду, которая выполняется в оболочке, прежде чем отладчик, который может использоваться для изменения среды отладки.
Программа | Полный путь в удаленной системе для отлаживаемой программы. Если оставить пустым или без изменений, по умолчанию используется текущий выходной каталог проекта.
Аргументы командной строки программы | Аргументы командной строки, передаваемые в отлаживаемую программу.
Рабочий каталог | Рабочий каталог удаленного приложения. По умолчанию это основной каталог пользователя.
Дополнительные команды отладчика | Дополнительные `gdb` команды перед запуском отладки отладчиком.
Номер порта отладчика | Номер порта для связи отладчика с удаленным отладчиком. Порт не должен использоваться локально. Это значение должно быть положительное и от 1 до 65535. Если не указан, будет использоваться номер свободного порта.
Номер порта удаленного отладчика | Номер порта, на котором удаленным сервером отладчика `gdbserver` ведет прослушивание на удаленной системе. Порт не должен использоваться в удаленной системе. Это значение должно быть положительное и от 1 до 65535. Если не указан, будет использоваться номер свободного порта, начиная с 4444.
Режим отладки | Указывает, как отладчик взаимодействует с `gdb`. В *gdb режим*, отладчик диски `gdb` через оболочку в удаленной системе. В *режиме gdbserver*, `gdb` работает локально и подключается к `gdbserver` удаленно. | **gdbserver**<br/>**gdb**
Дополнительные пути поиска символов | Дополнительный путь поиска для символов отладки (solib-search-path).
Отладка дочерних процессов | Указывает, следует ли включить отладку дочерних процессов.
Включить качественную печать в Python | Включить качественную печать значений выражений. Поддерживается только в режиме отладки gdb.
Файл визуализации | Файл собственной визуализации по умолчанию (NATVIS), содержащий директивы визуализации для типов SLT. Другие natvis-файлы, принадлежащие к текущему решению загружаются автоматически.
Сопоставление дополнительных путей к исходным файлам | Дополнительные эквивалентные пути, которые отладчик будет использовать для сопоставления имен исходных файлов Windows с именами исходных файлов Linux. Формат: "\<windows-path>=\<linux-path>;...". Имя исходного файла в каталоге Windows указывается, как если бы он находится в той же относительной позиции в пути Linux. Файлы, обнаруженные в локальном проекте, дополнительное сопоставление не требуется.

::: moniker-end
